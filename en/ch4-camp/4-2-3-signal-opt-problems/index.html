    2.3 信号·优化类真题¶

    难度 ⭐⭐⭐⭐☆ · 前置：3.4 凸优化基础、4.5 音频模型、1.3 限时解题策略

    本页目标

    • 看懂"非典型题"的三大套路：信号处理题、带约束优化题、多任务/复合题，并对每一类有一套可复用的下手框架
    • 学会把一道陌生的信号题"翻译"成你熟悉的机器学习任务，而不是被它的物理外壳吓住
    • 能区分"该上深度学习"和"该上一个干净的数学优化"，不在小题上浪费大算力
    • 用 3 个自编练习示例，把上面的方法论各走一遍

    重要声明：本页所有题目均为自编练习示例，不是官方真题

    我并不掌握 IOAI / IAIO 的真实历年题目，所以这一页不会给你任何"官方原题"。下面出现的每一道题，都是我为了讲清方法而自己编的练习示例，已逐一标注。真正的官方真题请以官方渠道为准： IOAI 官方网站（链接待核实补全） · IAIO 官方网站（链接待核实补全）。等官方真题入库后，会在 1.3 历年真题库 统一收录。

    为什么单独拎出"信号·优化"这一类¶

    你在 Round 1、Round 2 练的题，大多有个清晰的标签：这是图像分类、那是文本生成。可一旦进了集训营和国际赛，出题人最爱干的事，恰恰是把标签撕掉。他们会给你一段心电图、一串传感器读数、一个"在预算约束下最大化收益"的任务，表面上跟你练过的全不一样——这就是我们这一节要专门对付的"非典型题"。

    它们之所以让人慌，不是因为更难，而是因为陌生。但你只要掌握一个心法就够了：任何陌生题，都是某个你学过的套路换了层皮。 心电图也好、振动信号也好，剥掉物理外壳，本质都是"一段随时间变化的数字"，也就是一维信号；而"在约束下求最优"这种题，剥掉商业故事，本质就是 3.4 凸优化基础 里那套带约束的最优化。所以这一节我们不背题，只练"剥皮"——把陌生翻译成熟悉。

    信号题的通用下手框架：先变换，再建模¶

    信号题里最常见的是一维信号，比如音频、心电、加速度计读数。它和图像最大的区别是：图像的关键信息藏在"空间排布"里，而信号的关键信息常常藏在"频率"里。

    举个最直观的例子。一段心跳信号，你盯着它的原始波形（时域）可能看不出名堂，但只要把它换算成"由哪些频率成分组成"（频域），心率、杂音这些特征立刻就清楚了。这个"从时域换到频域"的工具，就是傅里叶变换（Fourier Transform）。所以面对信号题，第一反应不该是直接喂给神经网络，而是先问一句：我要不要先做个变换，把信息摆到更好看的地方？

    一套稳妥的下手顺序是这样的：

    1. 画出来看一眼。 先把原始信号画成波形图，再画出它的频谱，心里有个数：信息在时域明显还是频域明显？
    2. 变换到合适的域。 频率特征重要就做 FFT；如果信号的特征"随时间在变"（比如一段语音里音高起伏），就用短时傅里叶变换，得到一张"时间 × 频率"的图，也就是 4.5 音频模型 里讲过的频谱图（spectrogram）。
    3. 挑模型。 信号短、特征清楚，传统机器学习（提几个频域特征 + 一个分类器）往往又快又准；信号长、模式复杂，就把频谱图当成图像，交给一维卷积或 CNN。

    下面这道自编示例，把这套框架走一遍。

    自编示例 1：传感器信号分类（非官方真题）

    题目（自编）：给你一批工业电机的振动信号，每条是 1 秒、采样率 1000 Hz 的一维数组（1000 个数）。已知"正常电机"和"轴承故障电机"的区别在于：故障电机会在某个特定频率上出现异常的能量峰值。请训练一个分类器区分两者。

    思路：故障特征明明白白写着"某个频率上能量异常"——这是频域信息，所以别傻乎乎把 1000 个原始点直接喂模型，先做 FFT 把它变成频谱，特征一下子就突出了。

    下面这段代码，就是把"先变换、再用频域特征训练一个简单分类器"这条路走通。先从信号里提取频域特征：

    import numpy as np

def fft_features(signal, fs=1000):
    """把一条一维信号变成一组频域特征。"""
    # rfft：实数信号的快速傅里叶变换，得到每个频率成分的幅度
    spectrum = np.abs(np.fft.rfft(signal))
    freqs = np.fft.rfftfreq(len(signal), d=1 / fs)   # 每个成分对应的频率(Hz)

    # 不直接用整条频谱(维度太高)，而是提几个有物理含义的统计量
    peak_freq = freqs[np.argmax(spectrum)]           # 能量最强的频率
    total_energy = spectrum.sum()                    # 总能量
    high_band = spectrum[freqs > 200].sum()          # 高频段能量(故障常在这)
    return np.array([peak_freq, total_energy, high_band])

# 把每条信号都转成 3 维特征向量
X = np.array([fft_features(sig) for sig in raw_signals])   # 形状 (样本数, 3)

    把上面这一步想明白很关键：我们没有改变数据里的信息，只是把它"摆到了更好分的位置"。原来 1000 维、看不出差别的原始波形，被压成了 3 个有物理含义的数。这就是信号题的精髓——好的变换，胜过更大的模型。 接下来用一个最朴素的分类器就能收尾：

    from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score

clf = RandomForestClassifier(n_estimators=100, random_state=0)
scores = cross_val_score(clf, X, y, cv=5)            # 5 折交叉验证
print(f"准确率: {scores.mean():.3f}")                # 频域特征好用的话，会很高

    你会发现，一个只有 3 维输入的随机森林，可能就把这题做到了很高的准确率。这正是想提醒你的：在集训营里碰到信号题，先别急着搬深度学习这门大炮，一个干净的频域变换加传统模型，往往用更少的时间和算力就解决了问题——这一点在 1.3 限时解题策略 里也会反复强调。

    带约束优化题：把"业务故事"翻译成数学¶

    第二大类非典型题，长得完全不像机器学习题。它会给你一个故事：用有限的预算买广告、让总收益最大；或者用有限的算力分配任务、让总延迟最小。这类题考的不是训练模型，而是你能不能把一段文字翻译成一个标准的优化问题，再调一个现成的求解器把它解掉。

    翻译的套路是固定的，永远问自己三句话：

    • 我能调的旋钮是什么？ 这些就是决策变量。
    • 我想让什么最大或最小？ 这就是目标函数。
    • 有哪些不能违反的规矩？ 这些就是约束条件。

    把这三样写清楚，一道应用题就变成了一个数学问题。下面这道自编示例演示整个翻译过程。

    自编示例 2：算力预算下的任务分配（非官方真题）

    题目（自编）：你有 100 个 GPU 小时的预算，要在 3 个模型上分配。第 \(i\) 个模型分到 \(x_i\) 小时，预计能带来 \(v_i \cdot \log(1 + x_i)\) 的收益（投入越多收益越高，但边际递减）。已知 \(v=(5,3,8)\)。问每个模型各分多少小时，总收益最大？

    把它翻译成数学。决策变量是三个分配时长 \(x_1,x_2,x_3\)；目标是最大化总收益；规矩是"加起来不超过 100，且都不为负"。写成标准形式就是：

    \[ \max_{x}\ \sum_{i=1}^{3} v_i\,\log(1+x_i) \quad\text{s.t.}\quad \sum_i x_i \le 100,\ \ x_i \ge 0. \]

    这里有个好消息要送给你：每个 \(\log(1+x_i)\) 都是凹函数，目标是凹函数之和、约束又都是线性的，所以这是一个凸优化问题。凸的好处在 3.4 凸优化基础 讲过——局部最优就是全局最优，求解器一定能找到真正的最好解，不会卡在半路。我们直接调 SciPy 来解，不用自己手写梯度下降：

    import numpy as np
from scipy.optimize import minimize

v = np.array([5.0, 3.0, 8.0])

# minimize 只会"最小化"，所以给目标函数取负号，等价于最大化收益
def neg_reward(x):
    return -np.sum(v * np.log1p(x))      # log1p(x) = log(1+x)，数值更稳

cons = {"type": "ineq", "fun": lambda x: 100 - x.sum()}   # 100 - Σx ≥ 0
bounds = [(0, None)] * 3                                   # 每个 x_i ≥ 0

res = minimize(neg_reward, x0=np.full(3, 33.3),
               bounds=bounds, constraints=cons)
print("最优分配:", np.round(res.x, 1))
print("最大总收益:", round(-res.fun, 3))   # 记得再把负号取回来

    代码跑完你会看到，求解器把更多的小时数分给了 \(v\) 最大的第三个模型——这符合直觉：收益系数高的地方多投入。但更值得你记住的是这套通用流程：先把文字翻译成"变量 / 目标 / 约束"，再判断它凸不凸，最后交给求解器。整个过程一行神经网络都没有，却是国际赛里实打实会考的一类题。

    多任务复合题：拆成子问题，逐个击破¶

    第三类是最唬人的——复合题。它会把好几个能力点缝在一道大题里：可能要你先处理一段信号，再用处理结果去做一个预测，最后还在某个约束下挑出最优方案。题面又臭又长，很多同学读到一半就慌了。

    对付它其实只有一招，但极其管用：别想着一口吃成胖子，把大题切成几个独立的小题。 你先快速通读全题，画一张"数据怎么从输入流到输出"的流程图，把它拆成几个首尾相接的模块；然后每个模块都是你练过的标准套路，逐个解决，再拼起来。

    下面这道自编示例就是个三段式复合题。

    自编示例 3：信号 + 预测 + 约束的三段复合题（非官方真题）

    题目（自编）：给你 50 台机器各自的振动信号，要求：(1) 从每条信号判断这台机器"未来一周故障的概率"；(2) 你只有维修 10 台机器的人手，请挑出最该修的 10 台，使"避免的总损失最大"，已知每台机器一旦故障会造成的损失各不相同。

    读完别急着写代码，先把它拆成清清楚楚的三步，画成一张流程图：

    图 2.3：复合题的解法就是把它拆成首尾相接的三个标准模块，逐个击破再拼起来。

    看清这张图你就踏实了：第①步是我们刚练过的信号题（FFT 提特征）；第②步是 Round 1 就学过的分类预测（输出故障概率）；第③步是一个带约束的选择优化——在"最多修 10 台"的约束下，挑出让"概率 × 损失"之和最大的那 10 台。第③步看着像优化，其实因为每台机器只有"修 / 不修"两种状态、且互不影响，最优策略简单到出人意料：

    import numpy as np

# probs：第②步算出的 50 台故障概率；losses：每台故障的损失
expected_loss = probs * losses        # 每台机器的"期望损失"

# 在只能修 10 台的约束下，修期望损失最大的那几台，避免的损失就最大
top10 = np.argsort(expected_loss)[-10:]   # 取期望损失最大的 10 个下标
print("应优先维修的机器编号:", np.sort(top10))

    这里藏着一个值得点出来的道理：不是每个"优化"都需要请出求解器。 因为每台机器独立、约束只是简单的"最多选 10 个"，按"期望损失"从大到小排序、取前 10 就是最优解，一行 argsort 搞定。能用排序解决的，就别上 SciPy；能用简单数学解决的，就别上深度学习——先判断问题的结构，再选最轻的武器，这是复合题给你上的最重要一课。

    容易踩的坑¶

    • 拿到信号就直接喂神经网络。 信号题十有八九先做个变换（FFT / 频谱图）才好做，跳过这一步硬训，既慢又学不好。先画频谱看一眼，再决定模型。
    • 忘了 minimize 只会最小化。 SciPy 的 minimize 永远在求最小值，想最大化收益就得给目标函数取负号，最后把结果的负号再取回来。少取一次，答案符号就反了。
    • 没判断凸性就乱用求解器。 如果问题不是凸的，求解器可能卡在局部最优。先按 3.4 凸优化基础 的方法确认目标和约束的凸性，心里才有底。
    • 复合题想一口吃完。 不拆模块、试图一个大模型端到端解决，往往中途就乱了。务必先画数据流程图，把大题切成独立小题。
    • 小题上大炮。 能排序解决的别上优化器，能传统机器学习解决的别上深度学习。在限时赛里，算力和时间都是稀缺资源（见 1.3 限时解题策略）。

    它在后面会怎么用到¶

    • 信号变频谱这套思路，直接承接 4.5 音频模型——语音、音乐本质都是一维信号，频谱图就是它们进 CNN 的门票。
    • 带约束优化的数学骨架，全部来自 3.4 凸优化基础；这里只是把它套进了应用题的外壳。
    • "拆模块、逐个击破"的复合题打法，会在 3. 团队挑战 Team Challenge 里再次派上用场——团队赛的大题，正适合按模块分工。
    • 真正的国际赛非典型题，等官方真题入库后见 1.3 历年真题库；本节练的是通用方法论，方法吃透了，换什么外壳都不怕。

    练习¶

    基础练习
    1. 自己造两类一维信号：一类是纯 50 Hz 正弦波加噪声，一类是 50 Hz + 120 Hz 叠加加噪声。用本页的 fft_features 提特征，看看 peak_freq 和 high_band 这两个特征能不能把两类分开。
    2. 把自编示例 2 的预算从 100 改成 200，重新跑一遍 minimize，观察最优分配怎么变。再想一想：为什么收益系数 \(v\) 最大的模型总是分到最多时长？
    进阶练习
    1. 给自编示例 2 加一条新约束："第一个模型至少要分到 20 小时"。用 SciPy 的 bounds 或额外的 constraints 把它表达出来，重新求解，并解释新约束让总收益降了多少、为什么。
    2. 把自编示例 3 的第③步改难一点：现在不是"最多修 10 台"，而是"维修总成本不超过预算 B，每台维修成本各不相同"。这时简单排序还成立吗？想清楚它为什么变成了一个 0/1 背包问题，并查一查可以用什么方法求解。

    小结¶

    这一节的核心就一句话：非典型题不是新知识，而是把学过的套路换了层皮，你的工作是剥皮。信号题先变换再建模，约束优化题先翻译成"变量/目标/约束"再交给求解器，复合题先拆成模块再逐个击破——三套框架背后，都是你早已练过的基本功。再记住一条贯穿全节的纪律：先判断问题结构，再选最轻的武器，别在小题上浪费大算力。

    想系统补一下信号处理的直觉，可以看 The Scientist and Engineer's Guide to DSP（免费在线书）；优化求解器的用法详见 SciPy optimize 官方文档。
