    2.3 激活函数与损失函数¶

    难度 ⭐⭐⭐☆☆ · 前置：2.2 梯度下降与反向传播、3.2 微积分与梯度

    读完这一节，你会

    • 说清 ReLU、Sigmoid、Tanh、Softmax 各自长什么样、值域多少、适合放在网络的哪里
    • 真正理解"为什么 ReLU 利于梯度流"，而 Sigmoid/Tanh 容易让梯度消失
    • 分得清 MSE、交叉熵、稀疏交叉熵，知道回归用哪个、分类用哪个，以及为什么
    • 会从零（NumPy）实现这些函数，也会用 PyTorch 的现成版本，并懂得 softmax 为什么要"减最大值"

    一个网络的两个开关：激活函数和损失函数¶

    上一节我们把感知机和 MLP 搭了起来，也手推了反向传播。但你可能没太注意：一个神经网络能不能学好，很大程度上取决于两个看似不起眼的"配件"——激活函数和损失函数。

    它们的分工很清晰。激活函数装在网络的每一层之间，负责给模型注入"非线性"，让它能拟合弯弯绕绕的复杂关系；而损失函数装在网络的最末端，负责把"模型预测得有多离谱"量化成一个数字，好让梯度下降知道该往哪个方向改。

    换句话说，激活函数决定了网络"能表达什么"，损失函数决定了网络"朝什么目标去学"。这一节我们就把这两类配件挨个拆开看——它们种类不多，但每一个都有自己适合的位置，选错了，模型很可能怎么调都学不动。

    为什么非要有激活函数：没有它，深层网络等于白搭¶

    先回答一个最该想明白的问题：神经网络的每一层做的是 \(\mathbf{z}=W\mathbf{x}+\mathbf{b}\)，这本身就是个线性变换。那如果我们把很多层直接摞起来，会怎么样？

    答案有点出人意料：会原地踏步。 因为线性变换套线性变换，结果还是线性变换——两层 \(W_2(W_1\mathbf{x})\) 完全可以写成一个等效的单层 \((W_2W_1)\mathbf{x}\)。你堆一百层，本质上和一层没有任何区别，照样只能画一条直线、一个平面，连"异或"这种简单的非线性都学不会。

    所以我们必须在每层线性变换之后，插入一个非线性的激活函数 \(a=\phi(z)\)，把直线"掰弯"。正是这一道道弯，让多层网络获得了拟合任意复杂函数的能力。记住这句话：激活函数是深度学习里"深度"二字之所以有意义的根本原因。

    ReLU、Sigmoid、Tanh：三个最常见的弯¶

    我们先看三个最常用的激活函数。它们都是"输入一个数、输出一个数"的简单函数，区别在于"怎么掰这个弯"。

    ReLU（修正线性单元，Rectified Linear Unit）是目前隐藏层里的默认首选，它简单到让人不敢相信：

    \[\text{ReLU}(z)=\max(0,\,z).\]

    意思就是"负数全压成 0，正数原样放行"。它的导数也同样干脆：\(z>0\) 时导数是 1，\(z<0\) 时导数是 0。

    Sigmoid（S 形函数）把任意实数压进 \((0,1)\) 区间，长得像一个被拉平的 S：

    \[\sigma(z)=\frac{1}{1+e^{-z}}.\]

    因为输出落在 0 到 1 之间，它天然适合表示"概率"，所以二分类的输出层常用它。它有一个很漂亮的求导性质 \(\sigma'=\sigma(1-\sigma)\)（这个结论我们在 3.2 微积分与梯度 的练习里推过）。

    Tanh（双曲正切）和 Sigmoid 形状几乎一样，只是把输出范围撑到了 \((-1,1)\)，而且关于原点对称：

    \[\tanh(z)=\frac{e^{z}-e^{-z}}{e^{z}+e^{-z}}=2\sigma(2z)-1.\]

    因为它的输出有正有负、均值接近 0，训练时往往比 Sigmoid 收敛得稍快一些，所以在 ReLU 流行之前，Tanh 一度是隐藏层的主力。

    下面这张图把三条曲线放在一起，你一眼就能看出它们"弯"的方式有多不一样：

    图 2.3-1：三种激活函数的形状。ReLU 是一道折线，Sigmoid 与 Tanh 是两端"压平"的 S 形——而正是这"压平"埋下了梯度消失的隐患。

    为什么 ReLU 利于梯度流：从那条"压平"的尾巴说起¶

    现在来回答这一节最核心的问题：既然 Sigmoid 和 Tanh 早就有了，为什么现在大家几乎清一色用 ReLU 当隐藏层激活？关键就藏在反向传播里。

    回忆一下 2.2 梯度下降与反向传播：梯度是一层一层乘回去的，每经过一个激活函数，就要乘上它的导数。所以激活函数的导数有多大，直接决定了梯度还能不能"活着"传到前面的层。

    我们来看 Sigmoid 的导数 \(\sigma'=\sigma(1-\sigma)\)。它的最大值出现在 \(z=0\) 处，也只有 \(0.25\)；一旦 \(z\) 稍微大一点或小一点，落到那条"压平的尾巴"上，导数就迅速趋近于 0。这意味着什么？每经过一层 Sigmoid，梯度最多只剩四分之一，往往还远不到。 网络一深，这些不到 1 的小数连乘起来，梯度就指数级地缩小，到了前几层几乎归零——前面的参数根本得不到有效更新。这个现象就叫梯度消失（vanishing gradient）。

    ReLU 则巧妙地绕开了这个陷阱。在正半轴上，它的导数恒等于 1——梯度乘上 1，既不放大也不缩小，原原本本地往前传。于是不管网络多深，只要神经元处在激活状态，梯度就能畅通无阻地流回最前面。这就是"ReLU 利于梯度流"的真正含义，也是它能撑起深层网络训练的原因。

    当然 ReLU 也不是完美的。它在负半轴导数为 0，如果某个神经元长期落在负区，它的梯度就一直是 0、再也学不动了，这种现象叫神经元死亡（dying ReLU）。为缓解它，人们又造出了 Leaky ReLU（负半轴给一个很小的斜率，比如 \(0.01z\)）等变体。但作为入门，你先牢牢记住 ReLU 这个默认选项就够了。

    一句话记住选择

    隐藏层默认用 ReLU；二分类输出层用 Sigmoid；多分类输出层用 Softmax（下面就讲）。Tanh 现在主要在一些特定结构（如 RNN）里还能见到。

    Softmax：把一排分数变成一组概率¶

    前面三个激活函数都是"一个数进、一个数出"。但多分类问题不一样——比如手写数字识别，网络最后会吐出 10 个数，分别对应 0 到 9 的"得分"。我们想把这 10 个数变成一组概率：每个都在 0 到 1 之间，而且加起来正好等于 1。干这件事的就是 Softmax。

    它的定义是：对第 \(i\) 个分数 \(z_i\)，

    \[\text{softmax}(z)_i=\frac{e^{z_i}}{\sum_{j} e^{z_j}}.\]

    直觉上，它先用指数函数 \(e^{z}\) 把每个分数都变成正数（同时放大了它们之间的差距，让大的更大），再除以总和做归一化，于是结果就成了一组合法的概率分布。得分最高的那一类，概率也最大。

    这里有一个竞赛里非常实用的小细节，务必记住：算 Softmax 之前，要先把每个 \(z_i\) 都减去这一排里的最大值。 为什么？因为指数函数涨得飞快，\(z_i\) 稍微大一点（比如 1000），\(e^{z_i}\) 就会直接溢出成 inf，整个计算崩掉。而我们有一个免费的数学事实：给所有 \(z_i\) 同时减去一个常数 \(c\)，softmax 的结果完全不变——

    \[\frac{e^{z_i-c}}{\sum_j e^{z_j-c}}=\frac{e^{z_i}e^{-c}}{\sum_j e^{z_j}e^{-c}}=\frac{e^{z_i}}{\sum_j e^{z_j}}.\]

    既然结果不变，我们就取 \(c=\max_j z_j\)，这样减完之后最大的那个数变成 0、其余都是负数，指数值全部落在 \((0,1]\) 之间，再也不会溢出。这个小技巧能帮你在比赛里省下莫名其妙的 nan/inf 报错。

    MSE、交叉熵、稀疏交叉熵：损失函数该怎么选¶

    激活函数讲完，我们转到网络的另一端——损失函数。它的任务是把"预测和真实答案差多少"压成一个数。选哪个损失，主要看你做的是回归还是分类。

    均方误差（MSE，Mean Squared Error）用于回归，也就是预测一个连续的数值（房价、温度）：

    \[L_{\text{MSE}}=\frac{1}{n}\sum_{i=1}^{n}(\hat{y}_i-y_i)^2.\]

    预测值和真实值相减再平方——平方是为了让正负误差都变成正的、且差得越远惩罚越重。这个我们在 1.2 线性回归 里已经用过了。

    但分类问题用 MSE 就不太合适了。分类的输出是"概率"，我们更想衡量"预测的概率分布和真实答案差多远"。这时该用交叉熵（Cross-Entropy）。对于多分类，真实标签通常写成独热（one-hot）形式，即正确类别那一位是 1、其余是 0，交叉熵是：

    \[L_{\text{CE}}=-\sum_{i} y_i \log \hat{y}_i,\]

    其中 \(\hat{y}_i\) 是 softmax 给出的预测概率。因为 \(y\) 是独热的，求和里其实只剩正确类别那一项 \(-\log \hat{y}_{\text{正确}}\)。它的含义很直白：模型给正确答案的概率越高，\(\log\) 越接近 0，损失越小；越是把正确答案预测得没把握，损失越大。 这种"答对得分高、答错重罚"的特性，让它的梯度比 MSE 在分类上更健康、收敛更快。

    那稀疏交叉熵（Sparse Cross-Entropy）又是什么？其实它和交叉熵算的是完全一样的东西，区别只在于标签的存放格式。普通交叉熵要求标签是独热向量 [0,0,1,0]；而稀疏版本允许你直接用类别的整数编号 2 当标签，省去手动做独热的麻烦。当类别非常多（比如几万个词的语言模型）时，存整数比存一长串 0/1 向量省内存得多。所以记住：稀疏交叉熵 = 交叉熵 + "标签用整数下标而非独热"，数学上等价，只是用起来更省事。

    动手实现：从零写 + 库版对照¶

    道理讲完，我们动手把它们写出来，再和 PyTorch 的官方版本对一遍，确认理解无误。先用 NumPy 从零实现这几个函数——注意 softmax 里那一行"减最大值"，正是上面强调的数值稳定技巧：

    import numpy as np

def relu(z):
    return np.maximum(0, z)                    # 负数压 0，正数放行

def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-z))            # 压进 (0,1)

def softmax(z):
    z = z - np.max(z, axis=-1, keepdims=True)  # 关键：先减最大值，防溢出
    e = np.exp(z)
    return e / np.sum(e, axis=-1, keepdims=True)

def cross_entropy(probs, labels):
    """probs: softmax 输出 (n, C)；labels: 整数下标 (n,)，即稀疏形式。"""
    n = probs.shape[0]
    # 取每个样本"正确类别"对应的预测概率，加 1e-9 防止 log(0)
    correct = probs[np.arange(n), labels]
    return -np.mean(np.log(correct + 1e-9))

    这里有个值得一提的写法：cross_entropy 直接用整数 labels 去 probs 里"按下标取正确类别的概率"，根本没构造独热向量——这正是稀疏交叉熵省事的地方。

    接着造几条数据跑一下，看看损失是不是"预测得准就小、预测得离谱就大"：

    logits = np.array([[2.0, 1.0, 0.1],     # 第 0 类得分最高
                   [0.5, 0.2, 3.0]])     # 第 2 类得分最高
labels = np.array([0, 2])               # 真实答案：第 0 类、第 2 类（全对）

probs = softmax(logits)
print(probs.sum(axis=1))                # [1. 1.]，每行概率和为 1
print(round(cross_entropy(probs, labels), 4))   # 损失较小，因为预测对了

    你会看到每一行概率都正好加到 1，且因为两个样本都"押对了"，损失很小。要是你把 labels 改成押错的类别，再跑一遍，损失会明显变大——这正说明交叉熵在"奖对罚错"。

    最后看库版。PyTorch 已经把这些封装好了，而且做了一件聪明事：它的 CrossEntropyLoss 内部已经包含了 softmax，所以你只需把没过 softmax 的原始分数（叫 logits）直接喂给它，标签也直接用整数——它本质上就是稀疏交叉熵：

    import torch
import torch.nn.functional as F

logits = torch.tensor([[2.0, 1.0, 0.1],
                       [0.5, 0.2, 3.0]])
labels = torch.tensor([0, 2])           # 整数标签，无需独热

print(F.relu(torch.tensor([-1.0, 2.0])))   # tensor([0., 2.])
print(F.softmax(logits, dim=1))            # 和我们手写的 softmax 一致
print(F.cross_entropy(logits, labels))     # 注意：传 logits，不是 softmax 后的概率！

    可以看到 F.softmax 的输出和我们 NumPy 版本对得上，证明实现没错。唯一要特别小心的是 F.cross_entropy：它要的是 logits，不是 softmax 之后的概率——如果你自己先做了一遍 softmax 再喂给它，等于做了两次 softmax，结果就全错了。这是新手最常踩的坑，下一节会再提醒。

    容易踩的坑¶

    • softmax 忘了减最大值：分数稍大就 exp 溢出成 inf，整个损失变 nan。从零实现时一定先 z - z.max()，这是数值稳定的命根子。
    • 给 PyTorch 的交叉熵重复做了 softmax：F.cross_entropy 和 nn.CrossEntropyLoss 内部自带 softmax，你要喂 logits。先手动 softmax 再传进去就是做了两次，梯度全乱。
    • log(0) 变成 -inf：自己写交叉熵时，正确类别概率可能是 0，log 会炸。加一个极小量 1e-9，或干脆用框架自带的稳定实现。
    • 隐藏层用了 Sigmoid 还抱怨学不动：深层网络隐藏层用 Sigmoid/Tanh 极易梯度消失。隐藏层请默认上 ReLU，把 Sigmoid 留给二分类输出层。
    • 回归任务套了交叉熵、分类任务套了 MSE：选错损失会让训练事倍功半。记牢：连续值预测用 MSE，类别预测用交叉熵。
    • 独热标签喂给了稀疏交叉熵：稀疏版要的是整数下标 2，不是独热 [0,0,1,0]。两者用错会直接报维度错误。

    这一节的两个配件，是你后面搭任何网络都要反复用到的：

    • 下一节 2.4 PyTorch 基础 会用 nn.ReLU 和 nn.CrossEntropyLoss 把它们组装进真正的训练循环里。
    • 2.5 优化与正则化 讲怎么配合优化器，缓解梯度消失/爆炸，让训练更稳。
    • 3.2 图像分类实战 里的 CNN 分类器，输出层就是 softmax + 交叉熵这套标准组合。
    • 它的数学根基在 3.2 微积分与梯度（激活函数求导）和 3.3 概率与统计（交叉熵其实来自信息论里的"分布距离"）。

    练习¶

    基础练习
    1. 不用任何循环，用 NumPy 把一个向量里所有负数变成 0——你已经实现了 ReLU。再画出它和 Sigmoid 的曲线对比一下。
    2. 手算 Sigmoid 在 \(z=0\) 处的导数值，验证它确实等于 \(0.25\)；再算 \(z=5\) 处的导数，体会"尾巴变平、导数趋零"是怎么导致梯度消失的。
    3. 给定一排分数 [1000, 999, 1001]，分别用"不减最大值"和"减最大值"两种方式算 softmax，看看前者会不会溢出、两者结果是否一致。
    进阶练习
    1. 用链式法则手推：当输出层是 softmax、损失是交叉熵时，损失对 logits 的梯度竟然简化成了一个极其干净的形式 \(\hat{y}-y\)（预测概率减真实独热）。算出来你会惊讶它这么简洁——这正是这对组合广受欢迎的原因之一。
    2. 自己写一个稀疏交叉熵，再写一个普通（独热）交叉熵，用同一批数据验证两者数值完全相等。
    3. 把一个三层 MLP 的隐藏层激活分别换成 Sigmoid 和 ReLU，在同一数据上各训练几十轮，对比训练损失下降的速度，亲眼看看梯度消失的影响。

    小结¶

    • 激活函数给网络注入非线性，没有它，再深的网络也等于一层；隐藏层默认 ReLU，二分类输出用 Sigmoid，多分类输出用 Softmax。
    • ReLU 利于梯度流，因为正半轴导数恒为 1，梯度乘 1 不衰减；而 Sigmoid/Tanh 两端"压平"导致导数趋零，深层连乘后梯度消失。
    • Softmax 把一排分数变成概率，实现时务必先减最大值防溢出——结果不变，却能躲开 inf/nan。
    • 损失看任务选：回归用 MSE，分类用交叉熵；稀疏交叉熵只是交叉熵的"整数标签"版本，数学上等价、用起来更省事。

    想把这些函数的形状和梯度看得更直观，强烈推荐去 PyTorch 官方文档的 nn 模块 边查边动手；想搞懂交叉熵背后的信息论直觉，可看 3Blue1Brown 关于熵与 KL 散度的科普视频，它用动画把"两个分布差多远"讲得很透。
