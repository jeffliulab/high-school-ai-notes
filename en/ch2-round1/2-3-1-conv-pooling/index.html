    3.1 卷积与池化¶

    难度 ⭐⭐⭐☆☆ · 前置：2.1 感知机与 MLP、3.1 线性代数

    读完这一节，你会

    • 说清楚卷积到底在做什么：一个小窗口在图上滑动、不断做加权求和
    • 会手算一个 3×3 卷积核在 5×5 图上的输出，并理解核（kernel）、步长（stride）、填充（padding）各自的作用
    • 理解最大池化（max pooling）怎么缩小特征图，以及感受野（receptive field）为什么会越叠越大
    • 能用 NumPy 从零写一遍卷积，再用 PyTorch 的 nn.Conv2d 对照，确认两者结果一致

    为什么处理图像不能直接用全连接¶

    在上一章我们已经会搭多层感知机（MLP）了，它把输入拉成一长条向量，然后层层做矩阵乘法。那一个很自然的问题是：图像不也是一堆数字吗？我直接把它拉直，喂给 MLP 不就行了？

    道理上可以，但实践中会立刻撞上两堵墙。

    第一堵墙是参数爆炸。 一张普通的彩色图，比如 \(224\times224\times3\)，拉直就是十五万个数。哪怕第一层只接一千个神经元，光这一层的权重就有一亿五千万个。参数这么多，不仅算不动，还极容易过拟合。

    第二堵墙是它把"位置关系"全丢了。 图像里相邻的像素是高度相关的——一只猫的耳朵，是由挨在一起的一小片像素拼出来的。可一旦你把图拉成一长条，原本上下相邻的两个像素，在向量里可能隔得很远，MLP 根本不知道它们"本来是邻居"。更糟的是，同样一只猫，出现在图片左上角还是右下角，对 MLP 来说是两组完全不同的输入，它得分别重新学一遍。

    卷积神经网络（Convolutional Neural Network，简称 CNN）就是冲着这两个问题来的。它的核心想法非常朴素：用一个很小的窗口，在整张图上滑来滑去，每滑到一处就做一次局部的加权求和。 这样一来，参数少了（一个小窗口反复用），位置关系也保住了（窗口始终看的是相邻的一小片）。这个"滑动窗口做加权求和"的操作，就叫卷积（convolution）。

    卷积：一个小窗口在图上滑动¶

    我们先把卷积这件事的画面建立起来，公式随后就好懂了。

    想象你手里有一张图，还有一个小小的方块，比如 \(3\times3\) 的，上面也填着九个数字。我们把这个小方块叫卷积核（kernel），有时也叫滤波器（filter）。卷积要做的，就是把这个核盖在图的左上角，让核里的九个数和它正下方盖住的九个像素一一相乘、再全部加起来，得到一个数；然后把核向右挪一格，再算一个数；挪到头了就换行，从下一行接着挪。所有算出来的数拼在一起，就是一张新的图，我们叫它特征图（feature map）。

    下面这张图把"盖住—相乘相加—挪一格"这个过程画了出来，你可以顺着箭头走一遍：

    图 3.1-1：卷积核盖住输入的一小片，做一次相乘相加，得到特征图里的一个数；核向右、向下逐格滑动，就填满整张特征图。

    写成数学也不难。设输入图是 \(X\)，核是 \(K\)（大小 \(k\times k\)），那么特征图上第 \((i,j)\) 个位置的值就是：

    \[ Y_{i,j}=\sum_{u=0}^{k-1}\sum_{v=0}^{k-1} X_{i+u,\,j+v}\cdot K_{u,v}. \]

    别被两个求和号吓到，它说的就是上面那句大白话：把核里的每个数 \(K_{u,v}\)，和它对应盖住的像素 \(X_{i+u,j+v}\) 相乘，全部加起来。

    这里有一个很值得点破的联系：卷积其实是全连接层的一种"特殊参数化"。全连接层里，每个输出都连着全部输入，权重各不相同；而卷积层里，每个输出只连着局部的一小片输入（这叫局部连接），而且不管滑到哪一处，用的都是同一个核（这叫权重共享）。正是这两条约束，把参数量从上亿砍到了几十个——一个 \(3\times3\) 的核才九个数。

    手算一遍：3×3 核在 5×5 图上¶

    光看公式容易飘，我们老老实实手算一个，把它钉死在脑子里。

    设输入是一张 \(5\times5\) 的图，卷积核是一个专门用来"找竖直边缘"的 \(3\times3\) 核：

    \[ X=\begin{bmatrix} 1&1&1&0&0\\ 1&1&1&0&0\\ 1&1&1&0&0\\ 1&1&1&0&0\\ 1&1&1&0&0 \end{bmatrix},\qquad K=\begin{bmatrix} 1&0&-1\\ 1&0&-1\\ 1&0&-1 \end{bmatrix}. \]

    注意看这张图：左边三列是 1、右边两列是 0，正中间那条"由亮变暗"的竖线，就是一条竖直边缘。我们来看卷积能不能把它找出来。

    先算特征图左上角那个数。把核盖在 \(X\) 的左上角，框住的是前三行、前三列那个全是 1 的小块：

    \[ (1\cdot1+0\cdot1+(-1)\cdot1)\times 3\text{ 行}= (1-1)+(1-1)+(1-1)=0. \]

    结果是 0。这很合理：这一片全是 1、没有明暗变化，核也就"什么都没探到"。

    再把核向右挪一格，让它正好骑在亮暗交界上，框住的是第 1\~3 列和第 2\~4 列之间那块，左两列是 1、右一列是 0：

    \[ (1\cdot1+0\cdot1+(-1)\cdot0)\times 3\text{ 行}=(1-0)\times3=3. \]

    冒出来一个 3——核在这里探到了一条强烈的边缘！再往右挪到最后一格，框住的是第 3\~5 列，也就是"1、0、0"那块，左一列是 1、右一列是 0：

    \[ (1\cdot1+0\cdot0+(-1)\cdot0)\times 3\text{ 行}=(1-0)\times3=3. \]

    还是 3——因为这一格里，左边那列 1 和右边那列 0 之间，仍然横着我们要找的那条亮暗分界。把整张特征图都按这个方法算完，会得到：

    \[ Y=\begin{bmatrix} 0&3&3\\ 0&3&3\\ 0&3&3 \end{bmatrix}. \]

    你看，特征图左边一列是 0、右边两列都是 3。0 出现在原图"全亮"的纯色区，3 则出现在亮暗交界附近——只要核的视野里还罩着那条由 1 跌到 0 的分界，它就会报出一个大值。这条"高亮的竖带"准确地标出了原图里边缘所在的位置。这就是卷积的魔法：一个精心设计（或者训练学出来）的核，能自动在图上把某种特定的花纹挑出来。

    顺带把一个尺寸规律记下来。\(5\times5\) 的图、\(3\times3\) 的核，输出是 \(3\times3\)。一般地，如果输入边长是 \(n\)、核边长是 \(k\)，那输出边长就是：

    \[ \text{输出} = n - k + 1. \]

    道理很直观：核不能滑出图外，所以它能放的位置就比图少了 \(k-1\) 个。这也解释了一个让人有点不爽的副作用——每做一次卷积，图就会缩小一圈。

    步长和填充：控制窗口怎么滑、图缩多少¶

    上面我们默认核是一格一格地挪。但挪几格、要不要给图镶个边，其实都是可以调的，这就引出两个常用旋钮。

    步长（stride），指的是核每次滑动跨几格。前面我们用的是步长 1。如果把步长设成 2，核就一次跳两格，相当于隔一个位置才算一次——好处是输出尺寸直接减半、计算量大降，代价是分辨率变粗。步长 2 的卷积，常被用来替代池化，起到"缩小特征图"的作用。

    填充（padding），是在图的四周补一圈 0，再做卷积。它解决的正是上面那个"图越卷越小"的烦恼。比如给 \(5\times5\) 的图镶上一圈 0，它就变成 \(7\times7\)，再用 \(3\times3\) 的核去卷，输出又变回 \(5\times5\)——尺寸保住了。这种"补一圈让输入输出一样大"的填充，有个专门的名字叫 same padding；什么都不补、任由图缩小的，叫 valid padding。

    下面这张图把补零前后画在一起，你能直观看到那圈灰色的 0 是怎么把图"撑大"的：

    图 3.1-2：same padding 在图四周补一圈 0，把 5×5 撑成 7×7，再用 3×3 核卷出来又是 5×5——尺寸不掉。

    把这两个旋钮和核大小放在一起，就有了卷积输出尺寸的通用公式（设填充为 \(p\)、步长为 \(s\)）：

    \[ \text{输出} = \left\lfloor \frac{n + 2p - k}{s} \right\rfloor + 1. \]

    代进去验证一下：\(n=5,k=3,p=0,s=1\) 给出 \(3\)；加上 same padding 让 \(p=1\)，就给出 \(5\)。和我们前面手推的完全对得上。

    最大池化：把特征图"抽稀"¶

    卷积之后，我们通常还想再做一件事——池化（pooling），让特征图变小一点。

    为什么要变小？一来是减少后面的计算量，二来是带来一点"位置上的宽容"：哪怕物体在图里挪了一两个像素，池化后的结果也基本不变，这种性质叫平移不变性，对识别任务很有用。

    最常用的是最大池化（max pooling）。它的做法极其简单：把特征图切成一块块不重叠的小窗口（比如 \(2\times2\)），每一块里只保留最大的那个数，其余的扔掉。直觉上，最大值代表"这一小片里，那个特征被激活得最强的程度"——我们只关心"这附近有没有探到边缘/花纹"，至于具体在哪个像素上，不那么重要。

    举个例子，对下面这张 \(4\times4\) 的特征图做 \(2\times2\)、步长 2 的最大池化：

    \[ \begin{bmatrix} 1&3&2&4\\ 5&6&1&2\\ 0&1&8&3\\ 2&4&1&0 \end{bmatrix} \xrightarrow{\;2\times2\text{ max}\;} \begin{bmatrix} 6&4\\ 4&8 \end{bmatrix}. \]

    左上那块 \(\begin{smallmatrix}1&3\\5&6\end{smallmatrix}\) 里最大是 6，右上那块最大是 4，左下是 4，右下是 8——四个窗口各留一个最大值，\(4\times4\) 就被抽成了 \(2\times2\)。除了最大池化，还有平均池化（average pooling），把"取最大"换成"取平均"，思路完全一样，只是它保留的是这一片的平均强度。

    感受野：为什么层叠起来就能"看懂大图"¶

    最后讲一个让 CNN 真正强大的概念——感受野（receptive field）。

    它问的是这样一个问题：特征图上的某一个数，到底是被原图上多大一块区域决定的？ 对第一层 \(3\times3\) 卷积来说，输出图上一个点，显然只由原图上对应的 \(3\times3\) 区域算出来，所以它的感受野就是 \(3\times3\)。

    有意思的事情发生在你叠第二层的时候。第二层的一个输出点，看的是第一层特征图上的 \(3\times3\)；而第一层那 \(3\times3\) 里的每个点，又各自来自原图的 \(3\times3\)。一层层往回推，第二层这个点最终"看到"的原图范围就扩大到了 \(5\times5\)。网络越深，顶层每个神经元能"望见"的原图范围就越大。 这就是为什么浅层卷积只能看到边缘、纹理这种局部细节，而深层卷积却能拼出眼睛、轮子乃至整张脸这样的大尺度结构——感受野随着深度一层层长大了。

    这给了我们一个很重要的设计启发：与其用一个又大又笨的核去一口吃下大范围，不如把若干个小核叠起来。 两个 \(3\times3\) 卷积叠在一起，感受野就有 \(5\times5\)，但参数更少、还多了一层非线性，效果往往更好。这正是现代 CNN 普遍偏爱 \(3\times3\) 小核的原因。

    动手实现：从零写一遍，再和 PyTorch 对照¶

    道理讲透了，我们来动手把卷积写出来，验证它和我们手算的一致。先用 NumPy 从零实现，把上面那个二重求和老老实实翻译成代码：

    import numpy as np

def conv2d(X, K):
    """最朴素的 2D 卷积：步长 1、无填充（valid）。"""
    n, _ = X.shape                 # 输入边长（这里假设是方阵）
    k, _ = K.shape                 # 核边长
    out = n - k + 1                # 输出边长 = n - k + 1
    Y = np.zeros((out, out))
    for i in range(out):           # 遍历输出的每一行
        for j in range(out):       # 遍历输出的每一列
            patch = X[i:i+k, j:j+k]    # 核当前盖住的那一小片
            Y[i, j] = np.sum(patch * K)  # 相乘再求和：就是卷积的定义
    return Y

# 用前面手算的那张 5×5 图和竖直边缘核来验证
X = np.array([[1, 1, 1, 0, 0]] * 5, dtype=float)
K = np.array([[1, 0, -1],
              [1, 0, -1],
              [1, 0, -1]], dtype=float)

print(conv2d(X, K))
# [[0. 3. 3.]
#  [0. 3. 3.]
#  [0. 3. 3.]]

    输出正是我们手推的那张"左列为 0、右两列为 3"的特征图，说明代码和定义对上了。注意核心就一行 np.sum(patch * K)——它把核盖住的那片像素和核逐元素相乘再加总，和数学公式里那两个求和号是一回事。

    接着换成 PyTorch 的库版。真实训练里我们不会自己写循环，而是用 nn.Conv2d，它在底层做了大量优化、还能自动求梯度。需要留意的是，PyTorch 要求输入是四维张量 (batch, channel, height, width)，所以我们得先把那张二维图"撑"成四维：

    import torch
import torch.nn.functional as F

# 把 X 和 K 都变成 (batch=1, channel=1, H, W) 的四维张量
X_t = torch.tensor(X, dtype=torch.float32).reshape(1, 1, 5, 5)
K_t = torch.tensor(K, dtype=torch.float32).reshape(1, 1, 3, 3)

Y_t = F.conv2d(X_t, K_t)         # 步长 1、无填充，和我们的 conv2d 一致
print(Y_t.squeeze())             # 去掉多余的维度再打印
# tensor([[0., 3., 3.],
#         [0., 3., 3.],
#         [0., 3., 3.]])

    结果和 NumPy 版、和我们手算的三方完全吻合。

    一个常被忽视的细节：卷积还是互相关

    严格地说，数学教材里的"卷积"会先把核翻转 180° 再做相乘相加；而深度学习框架里的 Conv2d 并不翻转，做的其实是"互相关（cross-correlation）"。但因为核是训练学出来的，翻不翻转只影响它学成什么样、不影响最终能力，所以整个领域就约定俗成地把它也叫"卷积"了。你知道有这回事即可，写代码时完全不用操心。

    最后顺手提一句池化的库版，同样是一行：

    feat = torch.tensor([[1, 3, 2, 4],
                     [5, 6, 1, 2],
                     [0, 1, 8, 3],
                     [2, 4, 1, 0]], dtype=torch.float32).reshape(1, 1, 4, 4)
print(F.max_pool2d(feat, kernel_size=2).squeeze())
# tensor([[6., 4.],
#         [4., 8.]])

    和我们前面手算的 \(\begin{smallmatrix}6&4\\4&8\end{smallmatrix}\) 一模一样。

    容易踩的坑¶

    • 输出尺寸算错：用错公式是最常见的 bug。牢记 \(\lfloor (n+2p-k)/s\rfloor+1\)，搭网络前先把每层的尺寸纸上算一遍，免得到了全连接层才发现维度对不上。
    • 忘了 batch 和 channel 维度：PyTorch 的卷积/池化只吃四维张量 (N, C, H, W)。把单张灰度图直接喂进去会报维度错，必须先 reshape(1, 1, H, W)。
    • 混淆 * 和卷积：patch * K 是逐元素相乘，还要再 np.sum 求和才构成一次卷积。只乘不加，你得到的不是一个数而是一整块，结果就全错了。
    • 步长和池化重复缩图：如果卷积已经用了 stride 2，又紧跟一个 stride 2 的池化，特征图会一下缩到原来的四分之一，很容易缩没了。两个"缩图"手段别无意识地叠加。
    • same padding 不总能整除：核是偶数、或步长大于 1 时，"补多少才刚好不掉尺寸"可能补不整齐，PyTorch 里直接写 padding='same' 更省心。

    它在后面会怎么用到¶

    这一节是整个 CNN 的地基，卷积和池化就是搭图像模型的两块积木：

    • 下一节 3.2 图像分类实战 会把卷积、池化、全连接层堆成一个完整的网络，真刀真枪地训练它在 CIFAR-10 上识别图片。
    • 等你不想从零训练时，会去 3.3 迁移学习与预训练编码器 直接借用别人训好的卷积层——那些层学到的边缘、纹理特征，正是本节讲的核在干的事。
    • 卷积"局部连接 + 权重共享"的思想是上一章 2.1 感知机与 MLP 的特例化；而它能被训练，靠的依然是 2.2 梯度下降与反向传播 那套链式法则。
    • 到了 Round 2，卷积还会撑起更复杂的任务，比如 目标检测 和 图像分割，但内核还是这一节这个滑动窗口。

    练习¶

    基础练习
    1. 把本节那个"竖直边缘核"转置一下，变成 \(\begin{bmatrix}1&1&1\\0&0&0\\-1&-1&-1\end{bmatrix}\)，用它对同一张 5×5 图做卷积。算一算输出是什么，并解释为什么这个核探不到竖直边缘（提示：它专找的是水平边缘）。
    2. 一张 \(32\times32\) 的图，用 \(5\times5\) 的核、步长 1、无填充做一次卷积，输出是多大？如果想让输出仍是 \(32\times32\)，该补几圈 0？
    进阶练习
    1. 给本节的 conv2d 加上 stride 和 padding 两个参数，让它支持任意步长和填充，再用 F.conv2d 设相同参数对照，确认两者输出完全一致。
    2. 自己写一个 max_pool2d(X, k)，对 \(2\times2\) 窗口取最大值，并和 PyTorch 的结果比对。然后想一想：如果把"取最大"改成"取平均"，在一张有很多噪点的图上，哪种池化更不容易被噪点带偏？

    小结¶

    • 卷积就是一个小核在图上滑动、每处做相乘相加，它靠局部连接和权重共享，把全连接的海量参数压到了几十个。
    • 输出尺寸记一个公式 \(\lfloor (n+2p-k)/s\rfloor+1\)：核与步长让图变小，填充把它撑回来。
    • 最大池化取窗口内最大值来抽稀特征图，换来更小的计算量和一点平移不变性。
    • 感受野随深度增长，所以叠很多 \(3\times3\) 小核，既省参数又能看懂大尺度结构——这是现代 CNN 的设计基石。

    想把卷积的动态过程看得更直观，强烈推荐 CS231n 的卷积可视化讲义，它用动图把核滑动、补零、步长讲得一目了然；想深究感受野的计算，可参考 A guide to convolution arithmetic for deep learning。
