    1.3.3 概率与统计¶

    难度 ⭐⭐☆☆☆ · 前置：1.3.2 微积分与梯度

    读完这一节，你会

    • 理解随机变量、期望、方差，认识伯努利和高斯两个核心分布
    • 会用贝叶斯公式做"由果推因"的推断
    • 明白极大似然估计是什么，以及它如何变成我们常用的损失函数

    AI 为什么离不开概率¶

    机器学习从头到尾都在和不确定性打交道。数据里有噪声，预测也从来不是百分百确定——一个好的模型不会武断地说"这是猫"，而是说"这是猫的概率为 0.92"。那个 0.92，就是概率在起作用。

    更进一步说，训练模型时我们其实一直在问一个问题：什么样的参数，最有可能产生我手上看到的这批数据？ 这个朴素的问题，正是统计里"极大似然估计"的核心思想，也是大半个机器学习的出发点。所以，把概率统计这门语言掌握好，你才能看懂损失函数背后的逻辑，而不只是机械地调用它们。

    随机变量、期望、方差：先立定义¶

    我们先把三个最基本的量定义清楚。

    随机变量就是一个"取值带有随机性"的量，比如抛一次硬币的结果、一张图被预测成猫的概率。它的期望 \(\mathbb{E}[X]\) 是"长期平均下来的取值"，对离散情况就是每个取值乘以它的概率再求和：\(\mathbb{E}[X]=\sum_x x\,p(x)\)。

    光有平均还不够，我们还想知道数据"散不散"。方差 \(\mathrm{Var}(X)=\mathbb{E}[(X-\mathbb{E}X)^2]\) 衡量的就是取值围绕平均值的波动大小：方差越大，数据越分散；方差越小，数据越集中。期望和方差合起来，就能粗略刻画一个随机变量的"性格"。

    两个最该认识的分布¶

    实际中分布有很多种，但对入门来说，先死死记住下面这两个就够用了：

    分布 用在哪 关键参数
    伯努利分布 \(\mathrm{Bern}(p)\) 描述二分类标签（是/否、0/1） 取 1 的概率 \(p\)
    高斯分布（正态分布）\(\mathcal{N}(\mu,\sigma^2)\) 描述连续噪声、初始化权重 均值 \(\mu\)、方差 \(\sigma^2\)

    伯努利分布你可以理解成"一次抛硬币"，只有两种结果；它天然适合表示二分类问题的答案。高斯分布就是那条经典的"钟形曲线"，自然界和工程里大量的随机现象都近似服从它，所以神经网络初始化权重、给数据加噪声时都离不开它。

    贝叶斯公式：由果推因¶

    很多时候我们看到的是"结果"，想反推"原因"。贝叶斯公式正是干这个的。

    它的形式是：

    \[P(A\mid B)=\frac{P(B\mid A)\,P(A)}{P(B)}.\]

    换成大白话：在已经观察到 \(B\) 发生的前提下，想知道 \(A\) 发生的概率，可以用"\(A\) 本来的概率"乘上"\(A\) 发生时 \(B\) 有多容易出现"，再做个归一化。垃圾邮件识别、疾病检测这类"看到证据、推断真相"的任务，背后都是它。

    极大似然估计：模型怎么从数据里学参数¶

    现在来到这一节最重要的思想——极大似然估计（MLE）。

    它的定义是：给定一批独立的数据 \(\{x_i\}\)，我们选择那个让"这批数据出现的可能性"最大的参数 \(\theta\)。由于很多概率连乘起来会变成极小的数、不好处理，我们通常取对数，把连乘变成连加：

    \[\hat\theta=\arg\max_\theta \sum_i \log p(x_i;\theta).\]

    为什么要花力气讲它？因为我们天天用的交叉熵损失，本质上就是"最大化对数似然"的另一种写法。理解了 MLE，你就明白了损失函数不是凭空设计出来的，而是有概率含义的。

    例 1：抛硬币的极大似然

    抛了 \(n\) 次硬币，出现 \(k\) 次正面。这批结果出现的似然是 \(L(p)=p^k(1-p)^{n-k}\)。我们想找让 \(L\) 最大的 \(p\)：对 \(\log L\) 求导、令它为零，解出来正是 \(\hat p=\dfrac{k}{n}\)。换句话说，最合理的概率估计，就是观测到的频率——这个朴素的结论，正是从 MLE 严格推出来的。

    动手：采样与估计¶

    我们用代码把"从分布采样，再从样本反推参数"这个统计的核心动作走一遍。

    先从一个均值为 5、标准差为 2 的高斯分布里采一万个样本，再用这些样本反过来估计它的均值和方差：

    import numpy as np
rng = np.random.default_rng(0)

samples = rng.normal(loc=5.0, scale=2.0, size=10000)
print(samples.mean(), samples.var())   # ≈ 5.0, ≈ 4.0

    你会看到估出来的均值接近 5、方差接近 4（也就是 \(2^2\)）——样本越多，估计越准，这正是统计的威力。

    再用代码验证一下刚才的抛硬币结论。我们用一枚"正面概率 0.3"的硬币抛一千次，看看 MLE 估出来的 \(\hat p\) 是不是就是正面频率：

    flips = rng.binomial(n=1, p=0.3, size=1000)
print("MLE p̂ =", flips.mean())          # ≈ 0.3

    结果确实贴近 0.3。这就把"最大似然 = 用频率估概率"这件事，从公式落到了代码上。

    容易踩的坑¶

    • 概率密度不是概率：对连续分布，\(p(x)\) 的值可以大于 1，只有对它做积分才是概率。别被一个"大于 1 的密度值"吓到。
    • 连乘会下溢：很多概率相乘会迅速趋近于 0，超出浮点精度。所以我们永远在对数空间里计算，用 np.log 把连乘变连加。
    • 独立性假设要当心：朴素贝叶斯假设各特征相互独立，现实里常常不成立，但它依然常常好用——别误以为它"严格正确"。
    • 样本太少别轻信：用频率估概率时，样本量小则估计的波动很大，必要时要做平滑处理。

    它在后面会怎么用到¶

    • 交叉熵损失其实就是负的对数似然，是分类任务的标准损失（见 2.1.3 逻辑回归与分类）。
    • 高斯分布用于神经网络的权重初始化，以及给数据加噪声做增强（见 2.2.5 优化与正则化）。
    • 贝叶斯思想贯穿朴素贝叶斯分类器，以及对预测做不确定性估计的各种方法。

    练习¶

    基础练习
    1. 一个袋子里 70% 是红球。已知"取出的是大球"，而大球里红球占一半、全部球里 40% 是大球，用贝叶斯公式求"取出的是红球"的概率 \(P(\text{红}\mid\text{大})\)。
    2. 用 NumPy 从标准正态分布 \(\mathcal{N}(0,1)\) 采样一万次，画出直方图（方法见 1.4.4 数据可视化），亲眼验证它是钟形的。
    进阶练习
    1. 推导高斯分布的极大似然估计，证明 \(\hat\mu\) 恰好是样本均值、\(\hat\sigma^2\) 恰好是样本方差。
    2. 实现一个朴素贝叶斯垃圾邮件分类器（用词袋特征加拉普拉斯平滑），记得在对数空间里累加概率，避免下溢。

    小结¶

    • 概率是度量不确定性的语言，期望、方差、分布是它的基本词汇。
    • 贝叶斯做"由果推因"，极大似然用"最可能产生数据"的原则来选参数。
    • 我们常用的交叉熵损失，就是负对数似然的化身——损失函数背后往往藏着概率。

    想系统补一补，可看 Mathematics for Machine Learning 第 6 章，或可汗学院的概率统计课程。
