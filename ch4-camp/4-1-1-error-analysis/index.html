    1.1 误差分析与调试¶

    难度 ⭐⭐⭐☆☆ · 前置：1.9 模型评估与选择、1.4 分类评估指标

    读完这一节，你会

    • 用一套固定的流程，把"模型不行"这件事拆成数据、容量、优化三类原因，而不是凭感觉乱调
    • 看懂混淆矩阵，知道模型到底错在哪一类、错成了什么样
    • 在 CV 和 NLP 任务里把"错得最离谱的那些样本"挑出来肉眼看一遍，从中读出该补数据、该增强、还是该换架构

    比赛里真正拉开差距的，往往是会不会"看错"¶

    到了集训营这个阶段，模型怎么搭、怎么训，你其实已经会了。可你会发现一个扎心的现象：同样的题、同样的模型，有人 0.82 就卡死，有人却能一路调到 0.91。差的那几个点，几乎从来不是因为他用了什么你不知道的神模型，而是因为他更会分析误差——他知道自己的模型现在错在哪、为什么错、下一步该动哪个旋钮。

    所以这一节我们不学新模型，只学一件事：当验证集分数不理想时，怎么系统地把原因找出来。这是竞赛工程里最值钱、却最少有人认真练的一项能力。我们先建立一套"分诊"的框架，再深入混淆矩阵和可视化两件最实用的工具。

    先分诊：错误到底来自哪三个地方¶

    模型表现不好，原因看起来五花八门，但归根到底逃不出三大类。把它们想成医院里的"分诊台"——病人进来，先粗判是哪个科的问题，再去对症下药，而不是一上来就乱开药。

    第一类，数据问题。 标注有错、类别严重不平衡、训练集和测试集分布不一致、特征里混进了"作弊信息"（数据泄漏）。这类问题最隐蔽，也最致命——因为再强的模型也救不回烂数据，Garbage in, garbage out。

    第二类，模型容量问题。 容量就是模型"能记住多复杂的规律"的能力。容量太小，模型连训练集都拟合不好，这叫欠拟合（underfitting）；容量太大，模型把训练集里的噪声都背了下来，一到新数据就露馅，这叫过拟合（overfitting）。

    第三类，优化问题。 模型本身够强、数据也没毛病，但你没把它训练到位：学习率不合适、训练轮数不够、初始化或归一化没做好，导致损失根本没降下去。

    怎么快速判断是哪一类？只要同时盯住训练误差和验证误差这两个数，绝大多数情况一眼就能分辨：

    图 1.1：随着模型容量增大，训练误差一路下降；验证误差却先降后升，呈 U 形。两条线之间的缝隙就是"泛化差距"——缝隙越大，过拟合越严重。

    照着这张图，分诊规则就清清楚楚了：

    • 训练误差就很高（连训练集都学不好）→ 要么欠拟合（容量太小、特征不够），要么优化没到位（先把模型训练充分再说）。
    • 训练误差很低、验证误差却高（两条线缝隙很大）→ 典型过拟合，该上正则化、加数据、做数据增强了。
    • 两个误差都低，但测试集一交就崩 → 八成是数据问题：训练集和测试集分布不一致，或者训练时偷偷用到了测试阶段拿不到的信息（数据泄漏）。

    记住这套"先看两个误差、再分三类"的流程，你以后面对任何"模型不行"的局面，都不会再瞎调一气了。

    混淆矩阵：别只看一个准确率¶

    分诊告诉你大方向，但要知道模型"具体错在哪一类"，就得请出混淆矩阵（confusion matrix）了。

    为什么不能只看准确率？举个例子：一个判断"是否患病"的模型，假如人群里只有 1% 的人患病，那它哪怕无脑地把所有人都判成"健康"，准确率也有 99%——可它一个病人都没查出来，毫无用处。一个单一的准确率，会把这种致命的偏科完全掩盖掉。

    混淆矩阵的做法很朴素：它把"真实类别"和"预测类别"做成一张表，第 \(i\) 行第 \(j\) 列的数字，表示"真实是第 \(i\) 类、却被模型预测成第 \(j\) 类"的样本有多少个。对角线上是判对的，对角线之外全是各种判错的。一眼扫过去，你就知道模型把谁错认成了谁。

    import numpy as np
from sklearn.metrics import confusion_matrix, classification_report

# y_true 是真实标签，y_pred 是模型预测，这里用一个三分类的例子
y_true = np.array([0, 1, 2, 2, 0, 1, 1, 2, 0, 2])
y_pred = np.array([0, 1, 2, 1, 0, 2, 1, 2, 0, 1])

cm = confusion_matrix(y_true, y_pred)
print(cm)
# 输出（行=真实，列=预测）：
# [[3 0 0]    第 0 类全判对
#  [0 2 1]    有 1 个第 1 类被错判成第 2 类
#  [0 2 2]    有 2 个第 2 类被错判成第 1 类 ← 这里漏得最多

# classification_report 进一步把每一类的精确率/召回率/F1 都列出来
print(classification_report(y_true, y_pred, digits=3))

    看这张矩阵，你立刻能读出关键信息：第 1 类和第 2 类之间互相混淆得很厉害（有样本从 1 错到 2，也有从 2 错到 1），而第 0 类一点问题都没有。这就把"模型不行"这个笼统的抱怨，落实成了一个具体得多的问题——"为什么模型分不清第 1 类和第 2 类？"。

    接下来你就能顺藤摸瓜：是这两类本身长得就像（那也许要更强的特征或更深的网络），还是它们的训练样本太少（那就该针对性地补数据或做增强），又或者标注本身就有混淆。混淆矩阵的价值，就是把一个模糊的分数，变成一个能动手解决的具体问题。

    为了让对角线之外的错误更醒目，竞赛里常把它画成热力图——颜色越深代表数量越多，离群的"亮块"一眼就跳出来：

    from sklearn.metrics import ConfusionMatrixDisplay

# normalize='true' 按行归一化，看的是"每一类里有多大比例被判到了各处"
ConfusionMatrixDisplay.from_predictions(
    y_true, y_pred, normalize='true', cmap='Oranges'
)
# 画出来后，对角线越亮说明该类判得越准；
# 对角线之外哪个格子亮，就说明那对类别最容易被搞混。

    看错样本：把"错得最离谱"的挑出来肉眼过一遍¶

    数字能告诉你"哪两类混了"，但要弄清"为什么混"，最有效的办法朴素到有点反直觉——把模型错得最离谱的样本，一个个调出来用眼睛看。 老手把这叫"error analysis（误差分析）"的体力活，可它常常比再调十次超参都管用。

    怎么定义"错得最离谱"？就是模型又错、又自信的那些样本：它给错误答案打了很高的概率。我们按这个标准把它们排序，挑最靠前的几十个看：

    import torch
import torch.nn.functional as F

# logits 是模型最后一层的原始输出，shape = (N, 类别数)
probs = F.softmax(logits, dim=1)          # 转成概率
conf, pred = probs.max(dim=1)             # 每个样本预测的类别 & 对应的置信度
wrong = pred != labels                    # 找出所有判错的位置

# 在判错的样本里，按置信度从高到低排序——越靠前，错得越"理直气壮"
wrong_idx = torch.where(wrong)[0]
order = conf[wrong_idx].argsort(descending=True)
worst = wrong_idx[order][:32]             # 取最该看的 32 个

for i in worst:
    print(f"真实={labels[i].item()}  预测={pred[i].item()}  置信度={conf[i]:.2f}")

    把这 32 个样本的原始内容找出来看，往往会有"恍然大悟"的时刻：

    • 如果是图像题（CV），就把这些图显示出来。你可能会发现，错的全是模糊图、遮挡图，或者某一种光照——那说明该做数据增强（随机裁剪、调亮度、加噪声），让模型见过这些情况。你也可能发现，错的图其实标注就标错了，那就该去清洗标签。
    • 如果是文本题（NLP），就把这些句子打印出来读。也许错的都是带否定词的句子（"这电影并不难看"被判成差评），或者都是某种夹杂表情、网络黑话的口语——那说明你的特征或分词没抓住这些信号，可能要换更强的预训练编码器，或针对性地补这类样本。

    这一步看似笨，却是连接"指标"和"行动"的桥梁。指标告诉你有问题，看错样本才告诉你问题长什么样、下一步具体该干什么。

    看完之后，到底该动哪个旋钮¶

    把上面三步走完——分诊、读混淆矩阵、看错样本——你手里就有了证据。现在可以对症下药了。下面这张表帮你把"现象"对应到"该做的动作"：

    你观察到的现象 最可能的原因 该先尝试的动作
    训练误差就降不下去 欠拟合 / 优化没到位 加大模型、训久一点、调学习率、加特征
    训练好、验证差，缝隙大 过拟合 加正则（dropout、weight decay）、加数据、做数据增强
    某两类在混淆矩阵里互相混 这两类难分 / 样本不均 针对性补这两类数据、调类别权重、上更强特征
    错样本集中在模糊/遮挡/某种光照 训练分布没覆盖到 数据增强，造出这类样本
    错样本里标注本身就错 数据脏 清洗标签，别让模型去学错误答案
    验证好、测试一交就崩 分布不一致 / 数据泄漏 检查划分方式，排查有没有用到未来信息

    这里有个顺序上的忠告：先怀疑数据，再怀疑优化，最后才考虑换架构。 新手最爱干的事，是模型一不好就立刻去搜"更牛的网络"，结果往往是大力气换来小提升。真正的高手会先花时间把数据和训练流程盘干净——因为换架构是成本最高、收益却最不确定的一步，应该留到最后。

    容易踩的坑¶

    • 只盯着一个总准确率：类别一不平衡，准确率就会骗人。务必同时看混淆矩阵和每一类的召回率，别让少数类被多数类的高分掩盖。
    • 在测试集上反复调参：你应该在验证集上做误差分析、调超参，测试集只在最后碰一次。否则你等于把测试集偷偷"学"进了模型，分数虚高、一到真实场景就垮。
    • 数据泄漏没排查：如果某个特征好得不像话（单它一个就能拿满分），先别高兴，八成是它泄漏了答案信息（比如用了只有标注完成后才存在的字段）。竞赛里这种坑能让你前功尽弃。
    • 看了错样本却不行动：误差分析的目的是导出下一步动作。看完一堆错图却还按原计划盲调，那就白看了。
    • 一次改太多东西：同时改了数据、又改了模型、又调了学习率，分数变了你也不知道是哪一步的功劳。一次只动一个变量，才能把因果关系理清。

    它在后面会怎么用到¶

    误差分析不是一节孤立的内容，它是贯穿整个集训营和竞赛实战的"总开关"：

    • 分诊出"该调参"之后，怎么系统地管理一堆实验、不把自己搞乱，见 1.2 调参与实验管理。
    • 比赛限时只有几小时，误差分析得做得又快又准，时间分配的策略见 1.3 限时解题策略。
    • 在 2.1 CV 类真题 和 2.2 NLP 类真题 里，你会反复用到本节的"看错样本→定位→数据增强/换编码器"这套打法。
    • 混淆矩阵和各类指标的定义，回头可复习 1.4 分类评估指标；过拟合/欠拟合与验证策略的根基在 1.9 模型评估与选择。
    • "该不该做数据增强"在视觉任务里有专门的工具箱，见 3.3 数据增强与自监督。

    练习¶

    基础练习
    1. 给你一个二分类的混淆矩阵 \(\begin{bmatrix}90 & 10\\ 30 & 5\end{bmatrix}\)（行=真实，列=预测，正类是第 1 行）。手算它的准确率、正类的精确率和召回率，并说一句话：这个模型最大的毛病是什么？（提示：正类的召回率低得吓人。）
    2. 用 sklearn 在一个三分类数据集上训练任意一个模型，画出它的混淆矩阵热力图，找出最容易被互相混淆的那一对类别。
    进阶练习
    1. 拿一个图像分类模型，写代码把"错得最自信"的前 16 张图显示出来，逐张判断：错因是模糊、是遮挡、还是标注本身错了？据此提出一条具体的改进动作（比如某种数据增强），实施后看验证分数是否提升。
    2. 故意制造一次数据泄漏：在训练特征里偷偷塞进一个由标签直接算出来的列。观察训练/验证分数飙到接近满分，然后把这列去掉，体会分数"打回原形"的感觉——这正是你在真题里要警惕的陷阱。

    小结¶

    一句话记住这节：当模型不行时，别急着换模型，先用"两个误差分诊 + 混淆矩阵定位 + 看错样本定因"这三步把问题找准，再对着证据动那一个最该动的旋钮。 顺序永远是先数据、再优化、最后架构。

    想把误差分析的方法论建得更扎实，强烈推荐读 Andrew Ng 的 Machine Learning Yearning——这本小册子几乎整本都在讲"怎么科学地诊断和改进一个模型"，是竞赛和工程里都极实用的功夫。
