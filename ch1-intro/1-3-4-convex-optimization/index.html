    3.4 凸优化基础¶

    难度 ⭐⭐⭐☆☆ · 前置：3.2 微积分与梯度

    读完这一节，你会

    • 分得清凸函数和非凸函数，明白它们对训练意味着什么
    • 写得出梯度下降的更新公式，理解学习率为什么是头号旋钮
    • 能用 NumPy 实现一维梯度下降，并亲手复现"学习率过大就发散"

    训练，就是在找损失的最低点¶

    我们已经反复说过，"训练模型"等价于"把损失函数降到最小"。但这件事好不好办，很大程度上取决于损失函数长什么样。

    如果损失函数是一个标准的"碗"，那就太省心了——不管你从哪里开始往下走，最后都会滑到同一个、也是唯一的谷底。可如果损失函数坑坑洼洼、有好多个小坑，那你下山时就可能停在某个小坑里，误以为到底了，其实旁边还有更深的谷。前者我们叫凸，后者叫非凸。理解这两者的区别，你才会明白后面调参时为什么学习率这么关键。

    凸函数和非凸函数：先给定义，再看图¶

    我们先把定义给出来。一个函数是凸函数，指的是：对它图像上任意两点连一条线段，这条线段永远不会落到函数曲线的下方。写成公式就是，对任意两点和任意 \(\lambda\in[0,1]\)，

    \[f(\lambda x+(1-\lambda)y)\le \lambda f(x)+(1-\lambda)f(y).\]

    这个定义不必死记，记住它带来的那个绝妙性质就好：凸函数的任何一个局部最低点，一定就是全局最低点。 也就是说，只要你能往下走，就不怕走错地方。非凸函数则没有这个保证，下面这张图把两者的差别画了出来：

    图 3.4：凸函数只有一个全局最低点；非凸函数可能让你停在某个局部的小坑里。

    梯度下降：怎么一步步走下去¶

    知道了要往低处走，具体怎么走？答案就是上一节那个"下山"动作的数学版——梯度下降。它的更新公式是：

    \[\theta_{t+1}=\theta_t-\eta\,\nabla f(\theta_t).\]

    读一遍：新的参数 = 旧的参数 − 学习率 × 当前的梯度。梯度告诉你"哪个方向上坡最陡"，前面加个负号就掉头朝下坡走；而那个 \(\eta\)，叫学习率，决定你每一步迈多大。

    学习率这个旋钮特别关键。它太小，你会走得极慢，半天到不了谷底；它太大，你又会一脚迈过谷底，在两边来回横跳，甚至越跳越远、彻底发散。 后面我们会用代码亲眼看到这两种情形。

    例 1：手推一步梯度下降

    设 \(f(x)=x^2\)，它的梯度是 \(\nabla f=2x\)。我们从 \(x_0=4\) 出发，取学习率 \(\eta=0.1\)。第一步：\(x_1=4-0.1\times8=3.2\)；第二步：\(x_2=3.2-0.1\times6.4=2.56\)……你会看到它稳稳地、一步步逼近最低点 \(0\)。

    动手：实现梯度下降，并让学习率"作妖"¶

    我们把上面的更新公式写成一个通用函数。它接收"如何算梯度"、起点、学习率和步数，然后老老实实地一步步更新：

    import numpy as np

def gradient_descent(grad, x0, lr, steps):
    x = x0
    traj = [x]
    for _ in range(steps):
        x = x - lr * grad(x)      # 这就是 θ ← θ - η ∇f
        traj.append(x)
    return np.array(traj)

grad = lambda x: 2 * x            # f(x) = x^2 的梯度

    先用一个合适的学习率跑跑看，它应该稳稳收敛到 0：

    print(gradient_descent(grad, 4.0, lr=0.1, steps=5))
# [4. 3.2 2.56 2.048 1.6384 1.31072]，正一步步逼近 0

    现在我们故意把学习率调得过大，看会发生什么：

    print(gradient_descent(grad, 4.0, lr=1.1, steps=5))
# [4. -4.8 5.76 -6.912 8.2944 -9.95328]，数值越跳越大、彻底发散

    注意第二组结果：它在正负之间来回横跳，而且幅度越来越大。这就是"学习率过大导致发散"的真实样子。把这两段亲手跑一遍，你对学习率的体会会比看任何文字都深。

    容易踩的坑¶

    • 学习率是第一调参对象：模型一发散，先别怀疑别的，第一反应是把学习率调小（比如除以 10）；如果收敛太慢，再适当调大。
    • 非凸不等于练不了：神经网络的损失确实是非凸的，但实践证明，配上好的初始化和优化器，梯度下降照样能找到足够好的解。别因为"非凸"两个字就放弃。
    • 特征没标准化会害你：当各个特征的数值范围差很多时，损失面会变成"又长又窄"的山谷，梯度下降在里面来回横跳、收敛极慢。所以建模前往往要先做标准化（见 4.3 Pandas 与数据处理）。
    • 真正的障碍常是鞍点：在高维空间里，比"局部最低"更常见的卡点其实是鞍点（一个方向是低谷、另一个方向是山脊）。

    它在后面会怎么用到¶

    • 所有训练都靠它：从线性回归到 Transformer，参数更新用的都是 \(\theta\leftarrow\theta-\eta\nabla f\)（见 2.5 优化与正则化）。
    • 凸模型有保证：线性回归、逻辑回归、SVM 的损失都是凸的，所以梯度下降一定能走到全局最优（见 1.2 线性回归）。

    练习¶

    基础练习
    1. 对 \(f(x)=x^2\)，把学习率改成 \(\eta=0.5\)，从 \(x_0=4\) 手算 3 步，看它是不是收敛得更快了。
    2. 改一下 gradient_descent，让它每一步都记录并打印 \(f(x)\) 的值，验证当学习率合适时，损失是单调下降的。
    进阶练习
    1. 把函数换成二维的 \(f(x,y)=x^2+10y^2\)（这是个"病态"的窄山谷），观察梯度下降在 \(y\) 方向疯狂横跳；再把坐标缩放一下重试，体会标准化的作用。
    2. 实现带动量的梯度下降：\(v\leftarrow\beta v+\nabla f\)，\(\theta\leftarrow\theta-\eta v\)。和普通版本比一比，看收敛快了多少。

    小结¶

    • 凸函数像个碗，局部最低就是全局最低；非凸函数则有很多坑。
    • 训练的统一更新公式是 \(\theta\leftarrow\theta-\eta\nabla f\)，其中学习率 \(\eta\) 是头号旋钮。
    • 发散先调小学习率，横跳先做特征标准化——这两条经验能帮你省下大量调试时间。

    想深入凸优化，可参考 Mathematics for Machine Learning 第 7 章；更进阶的可读 Boyd 的《Convex Optimization》。
