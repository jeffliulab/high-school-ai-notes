    3.1 线性代数¶

    难度 ⭐⭐☆☆☆ · 前置：高中向量与矩阵基础

    读完这一节，你会

    • 学会用"向量是数据、矩阵是变换"的眼光看待数据和模型
    • 真正理解矩阵乘法在做什么，并会手算转置、逆、特征值
    • 知道这些运算在 NumPy 里怎么写，以及它们各自对应 AI 里的什么

    为什么线性代数是 AI 的语言¶

    很多人一听"线性代数"就头大，但其实它是整个 AI 领域里最值钱的数学工具，没有之一。原因很简单：机器学习里的数据和模型，几乎都是用向量和矩阵来表示的。

    想象你在描述一朵花，记下了它的花瓣长、花瓣宽、花萼长、花萼宽四个数字。把这四个数排成一列，就得到一个向量——它代表了一个数据样本。如果你有一千朵花，就是一千个这样的向量，把它们摞起来，就成了一个一千行、四列的矩阵。

    那模型又是什么呢？神经网络的每一层，做的事情本质上就是"拿一个矩阵去乘这个数据向量"。换句话说，模型在不停地对数据做线性变换。下面这张图就画出了这件事：

    图 3.1：矩阵 A 把向量 v 变成了 Av——旋转加缩放。神经网络的每一层都在重复做这件事。

    把这个画面记在心里，后面学线性回归、PCA、神经网络时，你会发现它们都不过是这张图的变体。

    先把语言统一：向量与矩阵¶

    我们先把几个最基本的对象定义清楚，再逐一解释。

    向量就是一列数，记作 \(\mathbf{x}\in\mathbb{R}^n\)，意思是它由 \(n\) 个实数组成。矩阵是一个数表，记作 \(A\in\mathbb{R}^{m\times n}\)，表示它有 \(m\) 行、\(n\) 列。

    为了更直观，你可以这样对应：一个向量就是一条数据（一朵花的四个测量值），一个矩阵就是一整张数据表（很多朵花排在一起），或者是模型里的一组参数。它们不是抽象符号，而是你天天打交道的"数据"和"模型"本身。

    矩阵乘法：最该吃透的一个运算¶

    如果说整节只能记住一个运算，那一定是矩阵乘法。

    定义：两个矩阵相乘 \(C = AB\)，结果里第 \(i\) 行第 \(j\) 列的元素是

    \[C_{ij}=\sum_{k} A_{ik}B_{kj}.\]

    它要求 \(A\) 的列数等于 \(B\) 的行数，否则没法相乘。

    光看公式有点干，我们换个更有画面感的理解：\(A\mathbf{x}\) 其实是在用向量 \(\mathbf{x}\) 的每个分量，对矩阵 \(A\) 的各列做加权求和。 你给每一列分配一个权重，再把它们加起来，就得到了结果。神经网络里"输入乘权重"算的就是这个。

    例 1：算一个矩阵乘向量

    设 \(A=\begin{bmatrix}1&2\\3&4\end{bmatrix}\)，\(\mathbf{x}=\begin{bmatrix}1\\1\end{bmatrix}\)。按定义，结果的第一个数是第一行点乘 \(\mathbf{x}\)，第二个数是第二行点乘 \(\mathbf{x}\)： $\(A\mathbf{x}=\begin{bmatrix}1\cdot1+2\cdot1\\3\cdot1+4\cdot1\end{bmatrix}=\begin{bmatrix}3\\7\end{bmatrix}.\)$ 动手算一遍，比看十遍公式都管用。

    转置、逆、特征值、SVD：各是什么、有什么用¶

    除了乘法，还有四个名词你会反复遇到。我们先给出简洁定义，再说它们好在哪。

    转置 \(A^\top\)：把矩阵的行和列互换，即 \((A^\top)_{ij}=A_{ji}\)。它常用来调整数据的摆放方向，好让两个矩阵能对得上、乘得起来。

    逆 \(A^{-1}\)：满足 \(A^{-1}A=I\) 的那个矩阵（只有方阵且"不退化"时才存在）。你可以把它理解成"撤销操作"——如果 \(A\) 把向量做了某种变换，\(A^{-1}\) 就把它变回去。线性回归的解析解里就会用到它。

    特征值与特征向量：如果一个向量被矩阵作用后，方向不变、只是被拉长或压短了，即满足 \(A\mathbf{v}=\lambda\mathbf{v}\)，那么 \(\mathbf{v}\) 就叫特征向量，对应的伸缩倍数 \(\lambda\) 叫特征值。它回答的问题是："这个变换里，哪些方向是'稳定'的？"

    奇异值分解（SVD）：任何一个矩阵都能被拆成 \(A=U\Sigma V^\top\) 三部分，其中 \(\Sigma\) 对角线上的数（奇异值）告诉你"在每个方向上数据被拉伸了多少"。后面学 PCA 降维，靠的就是它。

    这四个概念现在不用全背下来，知道"它们各自回答什么问题"就够了，等用到时再回来查。

    动手算一算¶

    光说不练假把式。NumPy 把上面这些运算都封装好了，我们一步步来。

    先做最常用的矩阵乘向量。注意：在 NumPy 里，矩阵乘法要用 @，而不是 *——后者是逐个元素相乘，完全是另一回事：

    import numpy as np

A = np.array([[1., 2.],
              [3., 4.]])
x = np.array([1., 1.])

print(A @ x)     # [3. 7.]，和我们手算的一模一样

    输出正是例 1 里算出的 [3, 7]，说明我们对了。

    接着看转置和逆，它们各自一个函数就能搞定：

    print(A.T)               # 转置：行列互换
print(np.linalg.inv(A))  # 逆矩阵

    最后是特征值和 SVD。它们看着高深，其实也是一行调用的事：

    vals, vecs = np.linalg.eig(A)   # 特征值与特征向量
U, s, Vt = np.linalg.svd(A)     # SVD：s 就是奇异值（已按从大到小排好）
print(vals)
print(s)

    你不需要现在就理解输出的每个数字，重要的是知道：这些"听起来很难"的运算，在代码里往往只是一行。难的从来不是调用，而是理解它们在解决什么问题——而那，正是这一节真正想教给你的。

    容易踩的坑¶

    • @ 和 * 搞混：A @ x 是矩阵乘法，A * x 是逐元素乘（还会触发广播），两者结果天差地别。这是新手最常见的 bug 来源。
    • 形状对不上：(m, n) @ (n,) 合法，但 (m, n) @ (m,) 会报 shapes not aligned。报错时第一反应是 print(A.shape, x.shape) 看看维度。
    • 能不求逆就不求逆：要解方程 \(A\mathbf{x}=\mathbf{b}\)，请用 np.linalg.solve(A, b)，它比先求逆再相乘又快又稳。
    • 整数数组的除法：np.array([[1, 2]]) 是整数类型，做除法时可能被悄悄截断。需要小数就显式写成 1.。

    它在后面会怎么用到¶

    这一节不是为了学数学而学数学，它马上就会派上用场：

    • 线性回归的解析解 \(\hat{\mathbf{w}}=(X^\top X)^{-1}X^\top\mathbf{y}\)，整个就是一串矩阵运算（见 1.2 线性回归）。
    • 神经网络的每一层都是 \(\mathbf{y}=W\mathbf{x}+\mathbf{b}\)，核心还是矩阵乘向量（见 2.1 感知机与 MLP）。
    • PCA 降维靠的就是对数据做特征分解或 SVD（见 1.8 无监督学习）。

    练习¶

    基础练习
    1. 手算 \(\begin{bmatrix}2&0\\0&3\end{bmatrix}\begin{bmatrix}1\\1\end{bmatrix}\)，并想一想：这个矩阵对向量做了什么几何操作？（提示：两个方向各被拉伸了不同倍数。）
    2. 随机生成两个矩阵 \(A\)、\(B\)，用 NumPy 验证 \((AB)^\top = B^\top A^\top\) 这条常用性质。
    进阶练习
    1. 自己写一个不调用 @ 的 matmul(A, B)（三重循环实现），再和 A @ B 对比结果是否一致、耗时差多少。你会亲身感受到向量化的威力。
    2. 找一张 \(50\times50\) 的灰度图，对它做 SVD，只保留前 \(k\) 个最大的奇异值来重建图像，看看 \(k=5\) 和 \(k=20\) 时清晰度差多少。这就是"低秩压缩"，也是 PCA 的雏形。

    小结¶

    • 数据是向量，模型是矩阵变换——记住这句话，半本深度学习都会变得直观。
    • 矩阵乘法是一切运算的核心，\(A\mathbf{x}\) 就是对 \(A\) 的列做加权求和。
    • 转置、逆、特征值、SVD 各回答一个问题；用到时查表即可，不必死记。
    • 代码里，用 @ 做乘法、用 solve 解方程，别滥用求逆。

    想把几何直觉建得更牢，强烈推荐看 3Blue1Brown 的《线性代数的本质》系列——它用动画把"矩阵在做什么"讲得无比清楚。更系统的推导可参考 Mathematics for Machine Learning 的第 2–4 章。
