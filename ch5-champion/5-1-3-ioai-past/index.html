    1.3 历年真题库¶

    难度 ⭐⭐⭐☆☆ · 前置：1.1 赛制详解、1.2 规则与工具、A2 真题与数据集

    本页目标

    • 知道 IOAI 历年题大致考过哪些领域、长什么样，心里有一张"地图"
    • 拿到一道陌生的开放题时，有一套可复用的解题方法论，而不是干瞪眼
    • 会用 2 道自编练习把方法论真的跑一遍，并明确区分"自编示例"和"官方真题"
    • 知道去哪里找真正的官方题面，以及该配什么数据集来练手

    先说清楚：这一页不替你刷题，它替你建地图¶

    到了 Champion 这一关，你面对的已经不再是有标准答案的选择题，而是 IOAI 这种"给你一个数据集和一个任务，限时几个小时，分数按你模型的表现来算"的开放赛题。这类题最折磨人的地方不是某个知识点不会，而是你不知道从哪儿下手——题目往往只给一句话的任务描述、一份数据、一个评测脚本，剩下的全靠你自己拆解。

    所以这一页的定位很明确：它不是一本"真题答案册"，而是一张地图加一套方法论。地图告诉你 IOAI 历年大概在哪些领域出题、每类题长什么样、难在哪；方法论告诉你拿到任意一道陌生题，第一步、第二步、第三步该怎么走。把这两样东西吃透，比你背下某一年的具体解法有用得多——因为竞赛题年年换，但拆题的套路是通用的。

    重要声明：本页不含官方真题

    我手里没有经过核实的 IOAI / IAIO 官方真题题面，因此本页不会给出任何具体年份的官方题目并谎称那是真题。下面所有带"练习示例"字样的题，都是我为了演示方法论而自编的，不是官方真题，难度和风格只是模仿。真正的官方题面，请走本页末尾的官方来源链接去获取（链接待核实补全）。请不要把自编示例当成"押题"。

    IOAI 历年题大致考些什么：按领域归类¶

    虽然我不能给你具体题面，但 IOAI 的考查范围是公开的（依据其官方 Syllabus），历年题基本就在这几个领域里打转。下面这张表帮你建立"领域感"——拿到题先判断它属于哪一类，你的方法论和工具箱就自动对上号了。

    领域 典型任务长什么样 常考难点 对应本教程章节
    计算机视觉（CV） 给一批图，做分类 / 检测 / 分割 / 检索 数据少、域偏移、要会用预训练编码器 3.4 视觉-文本编码器 CLIP、3.1 目标检测
    自然语言（NLP） 文本分类、信息抽取、给定 LLM 做提示工程 标注噪声、长文本、调用大模型而非自己训 4.4 预训练大模型、4.1 文本分类与词嵌入
    多模态 / 表征 图文配对、跨模态检索、嵌入对齐 把不同模态映到同一空间，相似度怎么算 1.2 嵌入表示、3.4 视觉-文本编码器 CLIP
    信号 / 优化 / 经典 ML 表格数据、时序、需要手工特征或调参 特征工程、过拟合、没有 GPU 也能做 1.10 进阶特征工程、1.6 集成方法
    鲁棒性 / 对抗 / 伦理 对抗样本、数据偏见、模型可解释性 题目"反着考"，要你攻击或诊断模型 3.3 数据增强与自监督

    你会发现，IOAI 几乎不考"从零手写一个 Transformer"这种纯实现题——它默认你会用现成的预训练模型和库，真正比的是你怎么"用好"它们：选哪个模型、怎么微调、怎么处理脏数据、怎么在限时内把分数榨上去。这一点和 Round 1、Round 2 的笔试很不一样，越早适应越好。

    拿到一道开放题，按这五步走¶

    下面这套流程是我建议你每道题都机械地走一遍的。竞赛现场时间紧、容易慌，有一个固定流程能让你不至于一上来就埋头瞎试。

    第一步，读懂评测指标，而不是读懂题目。 这是最反直觉、也最重要的一点。开放题最终按一个分数排名，那个分数怎么算，直接决定你该优化什么。比如指标是准确率（accuracy），那类别不平衡时你可能被带偏；如果是 F1 或 AUC，策略又完全不同。先去翻评测脚本，搞清楚分数公式，再回头看任务。

    第二步，跑通一个"最笨的基线"。 别想着一上来就上最强模型。先用最简单的方法（比如逻辑回归、或者直接调一个预训练模型零样本推理）拿到一个分数，把整条流水线——读数据、训练 / 推理、生成提交文件——先打通。有了这个基线，后面每个改进才有得比较。

    第三步，做误差分析，找最大的那块短板。 看模型到底错在哪：是某一类全错？是图片模糊的那些错？还是文本长的那些错？把错误归归类，你就知道力气该往哪使（详见 1.1 误差分析与调试）。

    第四步，针对短板做一个改进，再测一次。 一次只改一个东西——换模型、加数据增强、调一个超参——然后立刻看分数动没动。同时改五样东西，最后你根本不知道是哪一样起了作用。

    第五步，留出时间稳收尾。 比赛结束前留足时间，确认提交文件格式对、没有崩、分数是真的。每年都有人因为提交格式错了而前功尽弃。

    把这五步连起来就是一句话口诀：先懂指标，再立基线，看错在哪，一次改一处，留时间收尾。

    自编练习示例（非官方真题）：CV 方向¶

    再次提醒

    下面这道题是我自编的，用来演示上面的方法论怎么落地，不是 IOAI 官方真题。

    示例题目：给你 800 张某种植物叶片的照片，分属 10 个品种，但每个品种只标注了 30 张，剩下的没标。要求在一个隐藏测试集上做品种分类，评测指标是宏平均 F1（macro-F1，即每一类的 F1 取平均，不偏向多数类）。

    我们按流程走一遍思路。第一步看指标：是 macro-F1 而不是 accuracy，这意味着每一类同等重要，小类不能放弃。第二步立基线：数据这么少，从零训 CNN 必然过拟合，所以基线就该是"拿一个预训练编码器抽特征，再接一个简单分类器"。下面这段代码就是这个基线的核心：

    import torch, timm
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import f1_score

# 第一步：用一个在 ImageNet 上预训练好的编码器，把每张图变成一个特征向量。
#         我们只用它"抽特征"，不微调——数据太少，微调容易过拟合。
encoder = timm.create_model("resnet50", pretrained=True, num_classes=0)
encoder.eval()

@torch.no_grad()
def extract(images):          # images: 形状 (N, 3, 224, 224) 的张量
    return encoder(images).cpu().numpy()   # 得到 (N, 2048) 的特征矩阵

X_train = extract(train_images)   # 把带标注的训练图抽成特征
X_val   = extract(val_images)

# 第二步：在固定的特征上训一个轻量分类器。class_weight="balanced"
#         是专门为 macro-F1 准备的——它让小类的样本在损失里更"值钱"。
clf = LogisticRegression(max_iter=1000, class_weight="balanced")
clf.fit(X_train, y_train)

pred = clf.predict(X_val)
print("macro-F1:", f1_score(y_val, pred, average="macro"))   # 盯住的就是这个数

    这段代码跑完，你会得到第一个 macro-F1 分数。注意 class_weight="balanced" 这一处——它直接回应了"指标是 macro-F1"这个判断，是"先懂指标再动手"的具体体现。接下来第三、四步该做什么？做误差分析，看是哪几个品种 F1 最低，然后只针对这几类加数据增强或找更多无标注图做半监督，再测一次。这就是方法论落地的样子。

    自编练习示例（非官方真题）：NLP 方向¶

    同样是自编示例，非官方真题。

    示例题目：给你 2000 条用户评论和一个可调用的预训练大语言模型（LLM），要求把评论分成"投诉 / 咨询 / 好评"三类，评测指标是准确率。训练标注只有 200 条，其余 1800 条没标。

    这题的关键判断是：数据少 + 给了现成 LLM，那就别傻乎乎地拿 200 条去微调一个模型，直接用 LLM 做零样本或少样本分类往往更省事、分数更高。思路框架是这样的：

    # 思路：把分类任务写成一段提示词（prompt），让 LLM 直接输出类别。
# 这就是"少样本"做法——在提示里塞几个带答案的例子，模型照着学。
def build_prompt(review):
    examples = (
        "评论：东西收到就坏了，要求退款。\n类别：投诉\n\n"
        "评论：请问什么时候发货？\n类别：咨询\n\n"
        "评论：质量很好，下次还来。\n类别：好评\n\n"
    )
    return examples + f"评论：{review}\n类别："

# 把每条评论套进提示、调用模型、取输出里的类别词。
# （这里 call_llm 代表赛方提供的模型接口，现场会给你具体调用方式。）
preds = [call_llm(build_prompt(r)).strip() for r in reviews]

    这里没有"训练"这一步，活儿全在怎么把提示词写好：给的例子要覆盖三类、要典型，必要时还能把那 200 条标注挑几条最有代表性的塞进提示里。它和 CV 那道题表面差很远，但骨架完全一样——先看指标定方向，再立一个最省力的基线，然后再想怎么改进（比如对模型拿不准的评论再补一轮提示）。这正是方法论"通用"的意思。

    容易踩的坑¶

    • 把自编示例当押题。本页的题是用来练方法论的，不代表 IOAI 真会这么考。真题请走官方链接，别拿我编的题去"赌考点"。
    • 不读评测脚本就开干。最常见的翻车方式：优化了半天准确率，结果人家按 F1 排名。指标永远先看。
    • 一上来就堆最大的模型。没有基线做对照，你根本不知道改进有没有用，还可能在限时里把时间全烧在调一个跑不动的大模型上。
    • 忘了提交格式。分数再高，提交文件列名错了、顺序乱了，照样零分。收尾一定留时间核对。
    • 数据泄漏。用全部数据（含测试集）去算标准化统计量或做特征筛选，等于偷看答案，本地分虚高、线上崩盘（同 4.3 Pandas 与数据处理 里讲的那个坑）。

    它在后面会怎么用到¶

    这一页搭的是"拆题地图 + 方法论"。真正把它练熟，你需要两样配套的东西。一是题面和数据：官方真题去本页末尾的链接找，配套数据集和下载方式整理在 A2 真题与数据集，建议你照着那里的清单把环境和数据先备齐。二是限时演练：方法论只有在计时压力下反复跑才会变成肌肉记忆，所以接下来去 3.1 全真模拟 做整场限时模拟，并把 1.3 限时解题策略 一起看了。想了解 IAIO 在题型上和 IOAI 的差异，再翻 2.2 差异化备考。

    练习¶

    基础练习
    1. 找一道你之前做过的 Round 2 综合题，只用本页的五步流程重新走一遍：写下它的评测指标、你的最笨基线、误差分析结论。不用真跑，先把"拆题"这步练熟。
    2. 把上面 CV 示例里的 class_weight="balanced" 去掉再跑一次，对比 macro-F1 的变化，亲眼看看"指标决定策略"是什么意思。
    进阶练习
    1. 给 NLP 示例那道题设计一个完整改进方案：先做一轮零样本，找出模型最拿不准的评论（输出含糊或置信度低的），再设计第二轮提示专门处理它们，估计这样能把准确率提多少、代价是什么。
    2. 自己照着本页格式，再编一道"信号 / 优化类"的练习示例（明确标注非官方），写清它的指标、基线、和一处可能的改进。能出题，说明你真的把方法论吃进去了。
