    3.1 全真模拟¶

    难度 ⭐⭐⭐⭐☆ · 前置：1.1 IOAI 赛制详解、1.3 限时解题策略、2.1 CV 类真题、2.2 NLP 类真题

    读完这一节，你会

    • 会按 IOAI / IAIO 的真实节奏，给自己安排一场 4–6 小时的限时全真模拟
    • 拿到一套覆盖 CV、NLP、机器学习理论三个方向的练习题，知道每类题该怎么开局
    • 学会用一张固定的复盘模板，把每次模拟里的失分点系统地榨干、转成下次的提分项

    为什么到了这个阶段，最该练的是"模拟"而不是"刷新知识点"¶

    到了 Champion 这一关，你大概率已经把考纲上的知识点过了好几遍——CNN 会写、Transformer 会调、评估指标背得滚瓜烂熟。可你会发现，真到了赛场上，知识够用，时间却永远不够用。这正是国际赛和平时学习最大的区别：IOAI / IAIO 的题往往是开放式的，给你一个数据集、一个分数目标，让你在几个小时里从零搭出一个能跑、能拿分的方案。考的不只是"你会不会"，更是"你能不能在限定时间里把会的东西落地"。

    所以这一节不再教新的模型，而是教你怎么给自己造一场和正赛尽量像的模拟考。打个比方：你已经学会了游泳的每一个动作，现在要做的是跳进真正的泳池、掐着秒表游一趟，看看哪个动作在实战里会变形。只有这样，你才能在真正上场前，把"读题慌、调参乱、最后没时间提交"这些坑都提前踩一遍。

    重要声明：本页练习题均为自编

    我并不掌握 IOAI / IAIO 的官方真题原文，本页给出的所有题目都是为了练手而自编的示例，不是、也不冒充任何一届的官方真题。它们的作用是帮你熟悉"题目长什么样、该怎么开局"，难度和形式向正赛靠拢，但具体内容请只当作练习。真正的历年题请以官方渠道为准（见本页末的链接）。

    先把"全真"两个字落实：一场模拟该怎么搭¶

    很多人所谓的"模拟"，其实是开着浏览器、随时查文档、想歇就歇地做了道题——这练的是手感，但练不出抗压。真正的全真模拟，关键在于把约束条件也一起复刻。下面这张表帮你把一场模拟需要固定下来的要素一次对齐：

    要素 正赛大致情况 你模拟时该怎么做
    时长 单场约 3–4 小时，多场累计 一次性掐表 4–6 小时，中途不暂停
    算力 提供统一的 GPU 环境（如 Colab/受限实例） 固定用一个 GPU 会话，别中途换更强的机器
    可用资料 通常允许查官方文档、不许搜答案/不许联网问人 只开框架文档，关掉搜索引擎和聊天工具
    提交方式 在线评测，按分数排名，可多次提交 自己留一个测试集当"评测机"，只在结束前看分
    干扰 现场紧张、设备陌生 手机静音收起，一次坐满全程

    把这几条钉死之后，你的房间就成了一个"小赛场"。最容易被忽略、却最该坚持的一条是"中途不暂停"——因为正赛里没有暂停键，你一旦养成"卡住了就停下歇会儿"的习惯，到了考场上那种连续高压会让你瞬间露怯。

    下面给一个 5 小时单场的时间骨架，你可以按自己的节奏微调，但比例尽量别动：

    00:00–00:25  通读全部题目，判断难度与分值，排好做题顺序（先易后难、先高性价比）
00:25–01:10  搭最简单的可提交基线（baseline），先确保"有分可拿"
01:10–03:30  主攻：针对每道题逐步迭代模型与特征，每跑通一版就记一下分
03:30–04:30  调优：挑收益最大的一两道题深挖，做数据增强 / 调参 / 集成
04:30–05:00  复查与收尾：检查提交格式、跑一遍完整流程、留 buffer 防意外

    这个骨架背后的逻辑，和我们在 1.3 限时解题策略 里讲的"读题 10% / 原型 40% / 调优 40% / 复查 10%"是一脉相承的。请你务必把"先搭一个能提交的最简基线"当成铁律：哪怕它分数很低，也好过你花三小时雕一个完美方案、结果最后因为格式错误一分没拿到。

    自编练习一：CV 方向——细粒度图像分类（练习示例）¶

    我们从一道计算机视觉（computer vision）题开始。这类题在 IOAI 里几乎必考，特点是给你一批图、让你训练一个分类器，按测试集准确率打分。下面这道题是我自编的，请当练习做。

    练习 · CV：小样本鸟类细分类（自编，非官方真题）

    给你一个 8 类鸟的图像数据集，每类只有 200 张训练图、共 1600 张，另有一个不带标签的测试集。要求训练一个分类模型，使测试集 top-1 准确率尽可能高。限制：单 GPU，2 小时内出结果。

    拿到这种"数据少、类别细"的题，先别急着自己从头堆一个大 CNN——数据这么少，从零训只会过拟合。正确的开局是迁移学习：拿一个在大数据集上预训练好的编码器，只在你的小数据上微调。下面这段代码就是一个能直接当基线提交的骨架：

    import torch, torch.nn as nn
from torchvision import models, transforms, datasets
from torch.utils.data import DataLoader

# 1) 数据增强：数据少，就靠翻转/裁剪"造"出更多样本来对抗过拟合
train_tf = transforms.Compose([
    transforms.RandomResizedCrop(224),
    transforms.RandomHorizontalFlip(),
    transforms.ToTensor(),
])
train_ds = datasets.ImageFolder("data/train", transform=train_tf)
train_dl = DataLoader(train_ds, batch_size=32, shuffle=True)

# 2) 取预训练 ResNet18，冻住主干、只换最后一层分类头
net = models.resnet18(weights="IMAGENET1K_V1")
for p in net.parameters():        # 先冻结所有参数
    p.requires_grad = False
net.fc = nn.Linear(net.fc.in_features, 8)   # 换成 8 类的新分类头（默认可训练）

# 3) 只训练这个新头，速度快、又不容易过拟合
opt = torch.optim.Adam(net.fc.parameters(), lr=1e-3)
loss_fn = nn.CrossEntropyLoss()
for x, y in train_dl:
    opt.zero_grad()
    loss = loss_fn(net(x), y)
    loss.backward()
    opt.step()

    这段代码体现的就是国际赛里最常用的提分套路：预训练编码器 + 数据增强。先冻结主干、只训分类头，你能在十几分钟内拿到一个像样的基线；之后若时间够，再"解冻"主干、用很小的学习率整体微调，准确率通常还能再上一个台阶。关于这两件事的细节，可以回看 3.3 迁移学习与预训练编码器 和 3.3 数据增强与自监督。

    自编练习二：NLP 方向——文本分类（练习示例）¶

    第二个方向是自然语言处理（natural language processing，NLP）。它的开局逻辑和 CV 惊人地像——同样是"别从零训，先借一个预训练模型"，只不过这次借的是语言模型。

    练习 · NLP：新闻主题分类（自编，非官方真题）

    给你 5000 条带主题标签的中/英文短新闻（共 6 个主题），训练一个分类器，在测试集上最大化 macro-F1。限制：单 GPU，1.5 小时内出结果。

    这里要特别提醒：题目要的是 macro-F1 而不是准确率。为什么？因为如果某些主题的样本特别少，模型只要全猜多数类就能骗到不低的准确率，但那是没意义的。macro-F1 对每一类一视同仁地算 F1 再取平均，逼着你把小类也照顾好——这正是评估指标在悄悄告诉你"题目在乎什么"。这一点在 1.4 分类评估指标 里讲得很细，做题前最好回去扫一眼。

    开局同样是迁移学习，借一个预训练的文本编码器：

    from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

name = "distilbert-base-multilingual-cased"   # 多语种，中英文都能吃
tok = AutoTokenizer.from_pretrained(name)
clf = AutoModelForSequenceClassification.from_pretrained(name, num_labels=6)

# 把一批文本编码成模型输入，再前向得到 6 类的打分
batch = tok(["欧洲央行宣布加息", "新款显卡发布"],
            padding=True, truncation=True, return_tensors="pt")
logits = clf(**batch).logits          # 形状 [2, 6]，每行是一条新闻对 6 个主题的打分
pred = logits.argmax(dim=-1)          # 取分最高的那一类作为预测

    这段代码本身只是"前向跑通"，真正训练时你要把它接上优化器、按 macro-F1 在验证集上挑最好的 epoch。但它已经说明了 NLP 题的核心打法：选一个合适的预训练模型 + 一个对症的评估指标。题目是多语种就选多语种模型，看重小类就盯 macro-F1——把这两件事想清楚，后面只是迭代。相关原理见 4.1 文本分类与词嵌入 与 4.4 预训练大模型。

    自编练习三：理论方向——一道要"想清楚"的题（练习示例）¶

    国际赛里除了写代码，还会夹杂一些纯靠推理拿分的理论题——它们不一定要你跑模型，而是考你对概念有没有真理解。这种题往往是"性价比之王"：花十分钟想明白，就能稳稳拿一档分。

    练习 · 理论：过拟合与正则化（自编，非官方真题）

    一个模型在训练集上准确率 99%、验证集上只有 62%。请：(a) 用一句话判断它出了什么问题；(b) 列出三种不同思路的缓解办法，并各说一句为什么有效；(c) 如果换成训练集和验证集都只有 60%，你的诊断和对策会怎么变？

    这道题考的不是某个 API，而是你脑子里有没有一张清晰的"诊断地图"。(a) 训练好、验证差，典型的过拟合——模型把训练集背下来了，没学到能泛化的规律。(b) 三种正交的思路：加数据或数据增强（让模型见的样本更多，背不过来）、加正则化（如 L2、dropout，限制模型复杂度）、早停（在验证集变差之前就停，别让它继续钻牛角尖）。(c) 训练和验证都差，那是欠拟合，方向完全反过来——该加大模型、训久一点、或造更有信息量的特征。

    你看，同样是"准确率不行"，过拟合和欠拟合的药方正好相反。做理论题的关键，是先把现象归到正确的类别里，对策自然就出来了。 这套"诊断—对症"的思路，在 1.1 误差分析与调试 里有更完整的展开。

    模拟结束后：一张能反复用的复盘模板¶

    模拟做完，真正长本事的环节才刚开始。很多人考完就把卷子一扔，这等于白练。正确的做法是趁记忆还热，照下面这张模板把这场模拟"解剖"一遍，逼自己把每个失分点都落到纸面上：

    【本场模拟复盘】日期：____  总时长：____  最终得分：____

一、时间去哪了？（对照计划，记实际每段花了多久）
   - 读题/排序：____   基线：____   主攻：____   调优：____   复查：____
   - 哪一段严重超时？为什么？

二、失分点逐条列（每条写：题号 → 丢了多少分 → 根因 → 下次怎么避免）
   1.
   2.
   3.

三、根因归类（把上面的失分打勾归类，看自己最大的短板在哪）
   [ ] 知识没掌握   [ ] 时间没分配好   [ ] 代码/环境卡壳
   [ ] 读错题/做错指标   [ ] 提交格式出错   [ ] 心态崩了

四、下一场要改的"三件事"（只挑最致命的三条，多了执行不了）
   1.   2.   3.

    这张模板的精髓在于第三栏——把零散的失分归到几个固定类别里。连着做三四场模拟、每场都填一遍，你会惊讶地发现自己的失分总是集中在那么一两类上：可能你每次都死在"调优阶段贪心、最后没留时间复查"，也可能你反复"读错评估指标"。短板一旦被量化出来，提分就从"凭感觉努力"变成了"精准补漏"，这比你再多刷十道新题都管用。

    容易踩的坑¶

    • 不搭基线就开始死磕难题：最常见的翻车方式。务必先用半小时搞出一个能提交、有分的最简方案，再去碰难的——这是你的"保底分"。
    • 整场不掐表，想停就停：那不叫模拟，叫练习。正赛没有暂停键，平时不练连续抗压，上场必慌。
    • 看错评估指标：题目要 macro-F1，你却照着 accuracy 调模型，方向从一开始就偏了。读题时一定把"按什么打分"圈出来。
    • 最后五分钟才想起检查提交格式：提交文件的列名、行数、编码格式错一点就可能零分。把"留 buffer 复查格式"写进时间表，别赌运气。
    • 复盘走过场：只在心里"嗯我下次注意"，等于没复盘。必须落到纸面、必须归类、必须只挑三件事去改。

    它在后面会怎么用到¶

    这一节是你冲刺阶段的"主训练场"，往前往后都接得上：

    • 模拟前，先按 1.1 IOAI 赛制详解 和 2.1 IAIO 赛制详解 把目标比赛的真实规则、时长、算力对齐，模拟才"全真"。
    • 模拟时的时间分配，直接套用 1.3 限时解题策略；遇到具体题型，回看 2.1 CV 类真题、2.2 NLP 类真题、2.3 信号·优化类真题。
    • 模拟之后，新题源源不断地出现，要靠 3.2 前沿跟进 持续补充；历年真实题目与数据集的入口，见 A2 真题与数据集。

    练习¶

    基础练习
    1. 给自己排一场 4 小时的单场模拟：从上面三道自编题里任选两道，严格掐表、关搜索、中途不停，做完后把复盘模板完整填一遍。
    2. 只做"基线冲刺"的专项训练——拿 CV 那道题，要求自己 20 分钟内产出一个能提交的最简方案。练的就是开局速度。
    进阶练习
    1. 连做三场模拟，每场都填复盘模板，最后把三张表的"根因归类"汇总，找出你出现频率最高的那一类短板，并写出一个针对性的训练计划。
    2. 给 CV 那道题做一次"消融对比"：分别试"只训分类头""解冻主干微调""加更强数据增强"三种设置，记录各自的验证集准确率，亲手感受每个技巧带来的提分幅度。

    小结¶

    • 到了 Champion 阶段，最该练的是在限定时间里把会的东西落地，而不是再刷一遍知识点。
    • 一场"全真"模拟的关键，是连约束一起复刻：掐表 4–6 小时、固定算力、关搜索、中途不停。
    • 万能开局口诀：先搭能提交的基线保底，再针对高性价比的题深挖；CV 和 NLP 都优先用预训练 + 迁移学习。
    • 模拟之后的复盘，比模拟本身更值钱——用固定模板把失分归类，把短板从"凭感觉"变成"可量化"。
