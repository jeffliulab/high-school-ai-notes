    3.2 前沿跟进¶

    难度 ⭐⭐☆☆☆ · 前置：2.2 Transformer 架构、4.4 预训练大模型、3.4 视觉-文本编码器 CLIP

    本页目标

    • 想清楚到了 IOAI / IAIO 这一层，为什么"跟前沿"值得花一点时间，又为什么不能花太多
    • 拿到一份按方向分好类、标好优先级的阅读清单，知道每篇该看什么、看到什么程度
    • 学会一套低成本的"跟进习惯"，每周一两个小时就能把边际优势吃到手

    为什么到了国际赛这一层，才该谈"前沿"¶

    先说一句实在话：前面整套考纲——经典机器学习、反向传播、CNN、Transformer、CLIP、扩散模型——才是你分数的大头。真正能决定你在 Round 1、Round 2、集训营里走多远的，是这些基本功有没有练扎实，而不是你读过多少篇刚出炉的论文。所以如果你还没把 考纲总览 里的东西过完，请先回去补，这一页可以等。

    那为什么到了 IOAI / IAIO 还要专门留一节讲前沿？因为国际赛和前面几关有个本质区别：它给你的题，常常是一个没有标准答案的开放任务——给一个数据集、一个评测指标，让你在几个小时里把分数刷到尽量高。这种题拼的不只是"你会不会某个算法"，而是"你知不知道现在解这类问题最趁手的工具是什么"。举个例子，同样是小样本图像分类，知道"先拿一个预训练好的视觉编码器抽特征、再在上面接个轻量分类头"的人，往往比从零搭网络的人快好几倍、还更稳。这点"我见过、我知道现在大家都这么做"的信息差，就是前沿跟进能给你的边际优势。

    所以这一节的定位很清楚：它是锦上添花，不是雪中送炭。我们的目标是用最少的时间，换来"临场不会被一个新名词吓住、知道哪条路最可能 work"的底气。

    哪些方向值得盯，哪些可以略过¶

    前沿浩如烟海，但和竞赛真正相关的就那么几块。下面这张表帮你对号入座——左边是方向，中间是它对应你已经学过的哪一块，右边是该花多少注意力。

    方向 接的是哪门基本功 优先级 一句话理由
    指令微调的大语言模型（instruction-tuned LLM） 4.4 预训练大模型 高 NLP 类开放题里，"会用、会提示、会轻量微调"一个现成 LLM 几乎是标配
    新一代视觉骨干（ViT 及其变体、纯视觉自监督） 2.2 Transformer、3.3 数据增强与自监督 高 CV 类题越来越多直接拿大视觉模型抽特征，不再从头训 CNN
    视觉-语言多模态（CLIP 一脉及其后续） 3.4 CLIP 中 零样本分类、图文检索这类题的现成利器
    高效微调技巧（LoRA、adapter、提示学习） 1.3 模型微调 中 GPU 受限的赛场上，"小代价改大模型"特别实用
    扩散模型的新采样/条件控制 3.5 生成模型 低 出现概率低，了解概念即可，别投入太多

    记住一个判断标准：凡是能让你"少训练、多复用现成模型"的进展，优先级就高——因为赛场上算力和时间都紧，谁能站在巨人肩膀上谁就快。反过来，那些需要大规模算力才能复现、和评测任务又不直接挂钩的炫技工作，了解个名字就够了。

    一份能照着看的阅读清单¶

    光说"去跟前沿"等于没说，所以我把它落到具体的入口上。下面按"先看什么、再看什么"的顺序排好，你不必全看，挑和你弱项相关的切进去即可。

    第一档 · 打底，必看（讲清楚一个架构在干嘛）

    • Attention Is All You Need（Transformer 原论文）：所有现代大模型的源头。配合 2.2 Transformer 一起看，你只需要看懂"注意力怎么算、为什么要多头"。
    • An Image is Worth 16x16 Words（ViT）：把图像切成小块当 token 喂给 Transformer。理解了它，你就懂了为什么新视觉骨干都长一个样。
    • Learning Transferable Visual Models from Natural Language Supervision（CLIP）：图文对齐的奠基作，零样本分类的来由都在这。

    第二档 · 提效，按需看（讲清楚"怎么省着用大模型"）

    • LoRA / adapter 这一类参数高效微调的综述或博客：核心思想是只训一小撮新参数、冻住大模型主体，这正是 GPU 受限赛场的救命技巧。
    • 指令微调（instruction tuning）相关的综述：搞清楚"为什么同一个底座模型，调过指令的就更听话、更会做任务"。

    第三档 · 拓宽视野，闲时翻（不必深究）

    • 一两个高质量的"年度进展回顾"长文，帮你建立"现在大家都在卷什么"的全局感。
    • 顶会（NeurIPS、ICML、CVPR、ACL）的 best paper 标题列表——只看标题和摘要，知道有这么个东西、解决什么问题就行，绝不陷进细节。

    看清单时守住一条纪律：第一遍只追求"它解决什么问题、大致怎么解决的"，公式和实验细节一律先跳过。 竞赛要的是"我知道有这把锤子、知道它砸哪种钉子"，不是"我能从头造一把锤子"。

    把跟进变成一个低成本的习惯¶

    前沿最怕的不是看不懂，而是"一时兴起猛看三天，然后再也不碰"。比起冲刺，更管用的是把它做成一个轻量、可持续的固定动作。给你一套省力的办法：

    下面这段不是要你跑的程序，而是把"一周一次的跟进流程"用伪代码写清楚，照着做就行：

    # 每周固定花 60~90 分钟做这件事，别贪多
weekly_routine = [
    "1. 扫一遍订阅的几个来源（顶会摘要 / 一两个可靠的解读博客），只看标题",
    "2. 挑出 1~2 篇和我的弱项方向相关的，读摘要 + 看图",
    "3. 用三句话写下：它解决什么问题 / 核心思路 / 能不能用在竞赛题上",
    "4. 把这三句话记进自己的速查笔记，下次模拟赛遇到类似题先翻它",
]

for week in range(till_competition):
    do(weekly_routine)   # 关键是"每周都做一点"，而不是赛前突击

    这套流程的精髓就藏在最后那句注释里：积累靠的是频率，不是强度。 每周三句话的笔记，几个月攒下来就是一份属于你自己的、紧贴你薄弱环节的前沿小抄。等到 3.1 全真模拟 里碰到一道开放题，你脑子里能立刻蹦出"这题像那篇博客讲的零样本做法"，这一秒钟的反应，就是几个月习惯换来的边际优势。

    最后提醒一句心态：保持好奇，但别被前沿牵着走焦虑。看到一堆没听过的新词是常态，连研究者都不可能全懂。你要做的不是追平所有进展，而是在你已经扎实的基本功之上，多备几把趁手的新工具。

    容易踩的坑¶

    • 本末倒置：把大量时间砸在读新论文上，却把基本功（考纲 里的核心算法）荒废了。前沿是加分项，基本功才是底盘，顺序不能反。
    • 追细节追到出不来：第一遍就抠某篇论文的每个公式，结果一周只读了半篇。竞赛要的是广度上的"我知道有这东西"，不是深度上的"我能复现"。
    • 只收藏不消化：链接存了一堆，从没用三句话总结过。没经过你自己语言转述的信息，临场根本调不出来。
    • 盲信赛场不一定能装的工具：注意 IOAI / IAIO 对联网、对可用库有限制（见 1.2 规则与工具）。你跟进的方法要确认"赛场环境里真的能用"，否则纸上谈兵。

    它在后面会怎么用到¶

    这一节攒下的"前沿小抄"，最直接的用武之地就是 3.1 全真模拟 和正式比赛里的开放题——当一道题没有标准解法、要你自己选路线时，你见过的现成方案越多，起步就越快越稳。具体的赛制限制和可用工具，对照 1.2 规则与工具 与 2.2 差异化备考 一起看；想系统补来源和资料，去 A1 资料中心。

    练习¶

    基础练习
    1. 从上面的阅读清单里挑一篇"第一档"的论文，只读摘要和图，用三句话写下：它解决什么问题、核心思路是什么、能不能用在某类竞赛题上。
    2. 给自己定一个每周 60 分钟的"前沿时段"，连续坚持两周，把每次的三句话笔记攒成一页速查表。
    进阶练习
    1. 找一道你做过的开放式模拟题，回头想一想：如果当时知道某个现成的预训练模型或微调技巧，你会怎么改解法？把改进思路写下来。
    2. 针对你最弱的一个方向（CV / NLP / 生成模型任选其一），整理出 3 个"赛场上确认能用"的现成方案，标清各自适合什么题型。

    小结¶

    一句话总结这一节：前沿跟进是站在扎实基本功之上、用每周一点点时间去攒"现成趁手工具"的习惯，目的是临场少走弯路、多一分底气，而不是去追平所有最新进展。

    想找靠谱的来源，可以从 Papers with Code 按任务排行榜倒推该用哪些方法，配合各顶会的 best paper 列表建立全局感——记住，看它们时第一遍只看标题和摘要就好。
