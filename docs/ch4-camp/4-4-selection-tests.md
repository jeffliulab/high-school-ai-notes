# 4. 选拔测试准备

到了集训营这一步，你已经把经典机器学习、深度学习、计算机视觉、自然语言处理这些大块知识都过了一遍，也练过限时解题和误差分析。剩下的最后一关，是**选拔测试**——它决定谁能代表队伍走得更远。这一节不教新知识点，而是帮你把已有的本事，在一场高压、陌生、信息不全的考试里稳稳地交付出来。

## 选拔测试和前面的轮次到底有什么不一样

你可能会想：前面已经考过 Round 1、Round 2 了，选拔测试不就是再考一遍吗？还真不是。常规赛轮考的是"你会不会某个知识点"，题目大多有明确的套路；而选拔测试更像一场**综合实战**——它要看的是当题目超出你的准备范围时，你还能不能靠工程方法把分数啃下来。

换句话说，前面的考试在测"你学过的东西掌握得牢不牢"，选拔测试在测"你没学过的东西能不能现场搞定"。这个差别非常关键，它直接决定了你的备考重心：到这一步，再去背更多模型已经收益不大，真正该练的是**迁移和应变**。

## 它大概会考成什么样

每年的具体形式都会变，但选拔测试有一些相当稳定的特征，提前心里有数能省下很多临场的慌乱。下面这张表帮你把它和常规赛轮对照着看：

| 维度 | 常规赛轮（Round 1/2） | 选拔测试 |
| --- | --- | --- |
| 题型 | 偏向单一知识点，套路清晰 | 综合、开放，常给一份陌生数据要你做出可用模型 |
| 时限 | 较宽松，够你慢慢调 | 偏紧，逼你先拿基线分再优化 |
| 评分 | 多按固定答案或固定指标 | 常按**排行榜相对排名**或多档评分量规打分 |
| 计算资源 | 可能限 CPU 或给 GPU | 通常给 GPU，但有时长/额度限制 |
| 难点所在 | 知识点是否掌握 | 面对没见过的任务能否快速搭出工作流 |

要特别留意"评分"这一栏。很多选拔测试不是"答对就满分"，而是**排行榜制**：你的得分取决于你比别人高多少。这意味着策略变了——你不必把模型做到完美，只要稳稳地比大多数人做得好就行。所以先用最朴素的办法拿到一个能提交的基线（baseline），再一点点往上挤，几乎永远是正确的打法。关于这套限时取分的节奏，可以回看 [1.3 限时解题策略](4-1-3-time-strategy.md)。

## 怎么面对"未知的未知"

这是选拔测试最让人发怵的地方：你不知道会考什么，甚至不知道自己有哪些不知道——这就是所谓"未知的未知"（unknown unknowns）。面对它，与其指望押中题，不如练就一套**通用应对流程**。任何陌生的 AI 任务，几乎都能套进下面这四步：

1. **先把任务讲成一句人话。** 输入是什么、输出是什么、用什么指标评分？光是把这三件事写清楚，你就已经把一道吓人的题拆成了一个普通的监督学习问题。
2. **抢一个能提交的基线。** 别上来就想最优解。哪怕是一个随便跑通的小模型、甚至是"全猜众数"，先把提交流程打通，确认评分系统认你的格式。有了基线，你的心态会立刻稳下来。
3. **看数据和错误，找改进方向。** 这时候 [1.1 误差分析与调试](4-1-1-error-analysis.md) 的功夫就用上了——模型错在哪类样本上，往往直接告诉你下一步该补什么。
4. **在时限内迭代，留出提交缓冲。** 每改一版就提交一次、记一次分，别把所有改动堆到最后一把梭。临交卷前十分钟一定停手，保证手里有一个**能交、有分**的版本。

你会发现，这套流程根本不依赖"考前押中了什么"。它依赖的是你把基本功练成了肌肉记忆——这才是选拔测试真正在筛选的东西。

## 把状态稳住：压力管理

知识到位了，最后能不能发挥出来，往往卡在心态上。考场上最常见的崩盘，不是因为不会，而是因为**一道难题卡住后开始慌，慌了之后连会做的题也做砸了**。给你几条很实在的建议：

- **开局先扫一遍全卷，挑软柿子捏。** 先把有把握的分拿到手，既攒了分，也给自己攒了信心。难题往后放，别在第一道硬骨头上耗到天昏地暗。
- **给每道题设一个"止损线"。** 比如"这道题超过 30 分钟还没头绪，就先存一个基线答案，去做别的"。卡住不可怕，可怕的是不肯放手。
- **卡壳时，做一次深呼吸再动手。** 这听起来像废话，但生理上它真的有用——紧张时大脑供氧下降，判断力跟着掉，一次缓慢的深呼吸能把你从"瞎试"拉回"思考"。
- **把"我不会"换成"我先试个最简单的"。** 行动是焦虑最好的解药。哪怕方向不对，动起来本身就会让你冷静。
- **考前别熬夜抱佛脚。** 睡饱、吃好、提前到场，这些"非技术因素"对发挥的影响，常常比你临时多看的那个模型大得多。

记住一句话送给你：**慌是会传染的，但稳也是。** 你越是按流程一步步走，越能从混乱里把节奏夺回来。

## 往届选拔大概是什么风格

虽然题目年年不同，但风格上有迹可循，提前感受一下能降低陌生感：

- **给一份"半干净"的真实数据集**，让你从读数据、清洗、建模到提交跑完整条流水线——考的是工程闭环，不是某个孤立算法。
- **跨领域综合题**：比如一道题里既要做图像处理又要做文本理解，逼你把不同模块拼起来。
- **资源受限的优化题**：给定算力或时间上限，看谁能在约束下把指标做得更高，本质是在考你的取舍判断。
- **开放式探索题**：问题描述故意留白，需要你自己定义清楚"什么算做好了"，再动手——这最考验前面说的第一步"把任务讲成人话"。

想找贴近这种风格的练手材料，去看 [真题与数据集](../appendix/a2-datasets.md) 里整理的往届题与公开数据集，把上面那套四步流程反复跑几遍。

## 容易踩的坑

- **死磕一道题，把时间全搭进去。** 选拔测试是相对排名，一道题再难也只是一道题，弃掉它去拿别处的分，整体往往更划算。
- **不留提交缓冲，最后一版没交上。** 见过太多人改到最后一秒，结果格式错了或没来得及提交，前面的努力全白费。务必留出余量。
- **追求完美模型而忽略基线。** 没有基线就没有底分，也没有改进的参照系。先有"能交的"，再谈"更好的"。
- **被排行榜实时分数带跑节奏。** 看到别人分高就乱改方向，是大忌。盯住你自己的误差分析，按计划迭代。

## 它在后面会怎么用到

练好选拔测试这套应变能力，你面对的下一站就是真正的国际赛场。无论是 IOAI 还是 IAIO，比的都是同一种东西：在陌生、高压、限时的环境里把工程流程稳稳跑完。具体的国际赛制和往届题，可以继续看 [Champion · 国际赛](../ch5-champion/index.md) 这一关，那里会把全真模拟（[3.1 全真模拟](../ch5-champion/5-3-1-full-mock.md)）和赛制细节讲透。本节练的心态与流程，正是那一关的地基。

## 练习

??? note "基础练习"
    1. 找一个你完全没做过领域的 Kaggle 入门赛，给自己卡 90 分钟，逼自己只完成一件事：跑出一个能提交的基线，并在排行榜上看到分数。重点不是分高，是把"读题 → 基线 → 提交"这条路打通。
    2. 给自己写一张"考场流程卡"：开局做什么、卡住怎么办、临交卷前十分钟做什么。一句话一条，越短越好，下次模拟时贴在手边照着做。

??? note "进阶练习"
    1. 找一道跨领域综合题（比如图文结合的任务），严格按本节的四步流程做一遍，并在每一步用一两句话记下你的判断和取舍。事后回看，找出哪一步最拖你后腿。
    2. 模拟一次"排行榜制"：和同学各自做同一道题，约定按相对排名打分，体会一下"不必完美、只需领先"的策略差异，看它和你平时"追求满分"的做法有什么不同。

## 小结

- 选拔测试考的不是"你学过什么"，而是"没学过的你能不能现场搞定"——重心在迁移和应变，不在多背模型。
- 面对未知，靠的是一套通用流程：讲清任务 → 抢下基线 → 误差分析 → 留缓冲迭代，而不是押题。
- 它常按排行榜相对排名评分，所以"先稳住基线、再小步领先"几乎永远对。
- 心态决定发挥：扫全卷挑软柿子、给难题设止损线、卡壳先深呼吸、考前睡饱。慌会传染，稳也会。
