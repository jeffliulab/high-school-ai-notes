## 为什么 CV 题值得单独练一套方法论

你可能会想：CV 的知识点我在 Round 2 都学过了，目标检测、分割、CLIP 都会，为什么还要单独练"真题"？

因为**会一个模型，和会用这个模型去解一道陌生题，是两码事**。集训营和国际赛的 CV 题，很少直接告诉你"请训练一个 ResNet 做分类"。它更可能给你一份长相奇怪的数据、一个含糊的指标、一块算力有限的 GPU，然后让你自己判断：这到底是个什么任务？该从零训练还是微调？时间只够跑三轮，我把宝押在哪？

换句话说，CV 真题考的是**工程判断力**——在信息不全、资源受限的情况下，快速做出一连串还不错的取舍。这种能力背不出来，只能靠一套清晰的流程反复演练。下面就是这套流程。

## 解一道 CV 题的五步流程

不管题目多陌生，我们都按同样的五步走。把它记成肌肉记忆，考场上就不会慌。

**第一步，读题——先弄清"这是什么任务"。** 拿到题先别碰代码，先回答三个问题：模型的**输入**是什么（单张图？一对图？图加文字？），**输出**是什么（一个类别？一堆框？一张掩码？），**评分指标**是什么（准确率？mAP？IoU？）。这三个问题的答案，直接决定了你属于下面哪一类任务：

| 输出长什么样 | 任务类型 | 常用指标 | 第一反应选的模型 |
| --- | --- | --- | --- |
| 整张图一个标签 | 图像分类 | Accuracy / F1 | 预训练 ResNet/ViT + 微调 |
| 图里若干个框 + 类别 | 目标检测 | mAP | 预训练 YOLO / Faster R-CNN |
| 每个像素一个标签 | 语义分割 | mIoU / Dice | U-Net / DeepLab |
| 只给几张样例就要认新类 | 少样本分类 | Accuracy | CLIP 零样本 / 特征 + KNN |

**第二步，选模型——默认"站在巨人肩膀上"。** 这是 CV 题最关键的取舍，我们下一节专门展开。一句话原则先记住：**除非数据量极大或任务极特殊，否则永远优先用预训练模型做迁移学习，而不是从零训练。**

**第三步，实现——先跑通一个最小可用版本（baseline）。** 别一上来就追求高分。先用最朴素的配置把整条链路跑通：能读数据、能训练、能在验证集上出一个分数。有了这个 baseline，后面所有改动才有参照。

**第四步，调试——用误差分析驱动改进。** baseline 分数不理想很正常。这时候**不要瞎调参**，而要去看模型到底错在哪（这正是 [1.1 误差分析与调试](4-1-1-error-analysis.md) 教的）：是欠拟合还是过拟合？错的样本有没有共同点？是不是某一类特别差？看清病因再下药。

**第五步，解读结果——把分数翻译成人话。** 最后别只报一个数字。要能说清"我的模型为什么是这个水平、它在什么情况下会失灵"。国际赛常有书面或口头答辩，解读能力本身就是分。

## 为什么 CV 题几乎总是先选"迁移学习"

第二步是最容易做错的一步，单独说透。

设想一道题只给你 2000 张图做分类。如果你从零训练一个深度网络，参数有几百万个，而数据只有 2000 张——这就像让一个完全没见过世界的小孩，只看 2000 张图就学会认猫认狗，他几乎一定会"死记硬背"这 2000 张，换张新图就懵，也就是严重**过拟合**。

迁移学习的思路完全不同：我们拿一个**已经在上百万张图片上预训练好**的编码器（比如在 ImageNet 上训练过的 ResNet），它早就学会了"边缘、纹理、形状"这些通用视觉特征。我们只需要把它最后一层换成适配本题的分类头，再用这 2000 张图**微调**一下。相当于请来一位已经见多识广的视觉专家，只让他适应一下新任务，而不是从婴儿教起。

所以记住这条几乎不会错的默认规则：**数据少就冻住预训练主干、只训练分类头；数据中等就解冻顶部几层一起微调；只有数据极多、或图片和自然图像差异极大（比如医学影像、卫星图）时，才考虑从零训练。** 迁移学习的细节见 [3.3 迁移学习与预训练编码器](../ch2-round1/2-3-3-transfer-learning.md)。

下面我们用三个自编示例，把这套流程走三遍。

## 示例一：一道"伪装"成难题的分类题

!!! example "练习示例 A（自编，非官方真题）"
    **题面**：给你 12 个类别、每类约 300 张的植物叶片照片，训练集 3600 张，要求在测试集上最大化分类准确率。GPU 只能用 2 小时。

**走一遍五步流程。**

读题：输入单张图、输出一个类别、指标是准确率——典型的**图像分类**。数据量小（每类 300 张），算力紧（2 小时）。这两点立刻把方案锁死了。

选模型：数据少 + 时间紧，**别想从零训练**。选一个在 ImageNet 上预训练的 ResNet-18（轻、快），冻住主干，只训练最后的分类头。为什么是 18 而不是更深的 50？因为时间只有 2 小时，小模型迭代快，能多试几轮。

实现 baseline：

```python
import torch, torchvision
from torchvision import transforms

# 预训练模型当"特征提取器"：主干不动，只换最后一层
model = torchvision.models.resnet18(weights="IMAGENET1K_V1")
for p in model.parameters():
    p.requires_grad = False                 # 冻住主干，省时间也防过拟合
model.fc = torch.nn.Linear(model.fc.in_features, 12)  # 换成 12 类的新分类头

# 输入要和预训练时对齐：缩放到 224、用 ImageNet 的均值方差归一化
tf = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])
```

调试：假设 baseline 验证准确率只有 78%，而训练准确率高达 99%——**训练高、验证低，是过拟合的典型信号**。病因清楚了，对症下药：加数据增强（随机翻转、裁剪、颜色抖动），用更多"假数据"逼模型学通用特征而非死记。

```python
tf_train = transforms.Compose([
    transforms.RandomResizedCrop(224),      # 随机裁剪缩放
    transforms.RandomHorizontalFlip(),      # 随机左右翻转
    transforms.ColorJitter(0.2, 0.2, 0.2),  # 颜色轻微扰动
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
])
```

解读结果：加增强后验证准确率升到 88%，训练验证差距缩小——说明过拟合确实被压住了。如果还想再榨一点分，可以解冻主干顶部几层做更深的微调。但要清醒：**时间预算才 2 小时，与其反复调到 89%，不如把这套稳定方案跑完、留出时间检查提交格式。** 这就是限时赛的取舍。

## 示例二：少样本——这次别急着训练

!!! example "练习示例 B（自编，非官方真题）"
    **题面**：给你 5 个全新的商品类别，**每类只有 4 张**参考图，要把一批查询图分到这 5 类里。

读题：又是分类，但数据少到离谱——每类才 4 张。这是**少样本分类（few-shot）**，是个陷阱：你要是套示例一的思路去训练分类头，4 张图根本喂不饱，必然崩。

选模型：数据这么少，**最好的策略往往是干脆不训练**。我们改用 CLIP（一个把图片和文字映射到同一空间的视觉-文本编码器，见 [3.4 视觉-文本编码器 CLIP](../ch3-round2/3-3-4-clip.md)）。思路是：用 CLIP 把那 4×5=20 张参考图都编码成向量，每类取平均当作"该类的中心"；再把查询图也编码成向量，**看它离哪个类中心最近，就判给哪一类**——本质是在 CLIP 特征空间里做最近邻。

实现：

```python
import torch, clip   # OpenAI CLIP
model, preprocess = clip.load("ViT-B/32")

@torch.no_grad()
def embed(images):                          # 把图片编码成单位向量
    feats = model.encode_image(torch.stack([preprocess(im) for im in images]))
    return feats / feats.norm(dim=-1, keepdim=True)

# 每个类别的"中心" = 该类参考图向量的平均
centers = torch.stack([embed(support[c]).mean(0) for c in range(5)])

def classify(query_img):
    q = embed([query_img])                  # 查询图向量
    return (q @ centers.T).argmax().item()  # 余弦相似度最大的类别
```

调试与解读：如果某一类总被认错，先看它的 4 张参考图是不是质量差（背景杂乱、角度刁钻）——**少样本下，参考图的质量直接决定上限**，这比调代码重要得多。CLIP 还有个额外好处：它支持**零样本**，你甚至能直接用类别的文字描述（"a photo of a red mug"）当中心，连那 4 张图都不需要。这道题真正想让你学会的，是**判断"什么时候不该训练"**——这恰恰是新手最反直觉的一课。

## 示例三：分割题，指标会"骗"你

!!! example "练习示例 C（自编，非官方真题）"
    **题面**：道路航拍图，要把"道路"像素从背景里分出来，指标是 IoU（交并比）。道路像素只占全图约 5%。

读题：输出是每个像素一个标签（路 / 非路），这是**语义分割**，首选 U-Net 这类编码器-解码器结构。但有个红灯要立刻警觉：**道路只占 5%，正负样本严重不均衡。**

为什么这是个坑？设想模型偷懒，把整张图全判成"背景"——它的**像素准确率高达 95%**，看着很美，可它一条路都没分出来，IoU 几乎是 0。**这就是为什么这道题不能用准确率、必须用 IoU 做指标**：IoU 只看"预测的路"和"真实的路"重叠多少，模型偷懒立刻原形毕露。

选模型与损失：U-Net 主干没问题，关键在**损失函数**要对得起不均衡。普通交叉熵会被海量背景像素带偏，所以我们换成或叠加 **Dice 损失**，它直接优化重叠度，对小目标更友好：

```python
def dice_loss(pred, target, eps=1e-6):
    # pred 是 sigmoid 后的概率图，target 是 0/1 掩码
    pred, target = pred.flatten(), target.flatten()
    inter = (pred * target).sum()
    return 1 - (2 * inter + eps) / (pred.sum() + target.sum() + eps)
```

调试与解读：训练后如果 IoU 卡在 0.4 上不去，去**看可视化**——把预测掩码叠回原图，常见病因有两种：要么道路边缘糊成一团（解码器分辨率不够，可加跳连或提高输入分辨率），要么把"河流""屋顶边"误判成路（特征混淆，需要更多负样本或更强主干）。**分割题一定要肉眼看输出图，光盯着 IoU 数字你永远不知道它错在哪。** 分割的更多细节见 [3.2 图像分割](../ch3-round2/3-3-2-segmentation.md)。

## 容易踩的坑

- **预处理和预训练对不齐**：用 ImageNet 预训练模型，输入就**必须**用 ImageNet 的均值方差归一化、尺寸也要对上。这一步错了，模型表现会莫名其妙地差，而且很难查。
- **一上来就追高分**：不先跑通 baseline 就猛调参，是限时赛第一大死法。**先有一个能提交的完整方案，再谈优化。**
- **指标看一个数就完事**：尤其是检测和分割，一定要**可视化输出**。数字漂亮不等于模型对，类别不均衡时准确率尤其会骗人（见示例三）。
- **少样本还硬训练**：每类只有几张图却去训练大网络，必然过拟合。该上 CLIP / KNN 时就别死磕训练（见示例二）。
- **忘了时间预算**：GPU 时间是死的。开跑前先估算"一轮要多久、我能跑几轮"，把算力花在刀刃上（限时策略见 [1.3 限时解题策略](4-1-3-time-strategy.md)）。
- **数据泄漏**：增强、归一化的统计量只能从训练集来；验证/测试集的信息不能偷看，否则线下分虚高、线上翻车。

## 练习

??? note "基础练习"
    1. 找一份小型分类数据集（如 Flowers-102 的子集），照示例一走完五步：跑通冻结主干的 baseline，再加数据增强，记录两次的验证准确率，并解释差距来自哪里。
    2. 用 CLIP 对任意 3 个你自定的类别做**零样本**分类（只给文字描述，不给参考图），看准确率有多高，想一想哪类描述写得越具体效果越好。

??? note "进阶练习"
    1. 把示例三的不均衡分割做实：分别用纯交叉熵和 Dice 损失训练同一个 U-Net，对比 IoU 与可视化结果，写清 Dice 在小目标上好在哪。
    2. 给示例一的方案做一次完整的**误差分析**：把验证集里所有错分样本挑出来，按"真实类别"分组统计，找出最难的那一类，并提出一条针对性的改进（如该类专门加增强、或补数据）。

## 小结

- CV 真题考的不是背模型，而是**在受限条件下做工程取舍**——靠"读题→选模型→实现→调试→解读"这套固定流程把不确定性降下来。
- **默认迁移学习**：数据少就冻主干微调分类头，少到极致（few-shot）就干脆用 CLIP 不训练，只有数据极多或图像极特殊才从零训练。
- **永远先跑通 baseline 再优化**，**永远可视化输出而非只看一个分数**，类别不均衡时尤其要选对指标（IoU/Dice 而非准确率）。
- 本节所有题目都是自编练习，**不是官方真题**；真正的历年原题以 IOAI / IAIO 官方为准（见 [1.3 历年真题库](../ch5-champion/5-1-3-ioai-past.md)）。

想找真实题感，除了上面的官方占位链接，还可以去 [A2 真题与数据集](../appendix/a2-datasets.md) 看本站整理的公开数据集与往届题资源。
