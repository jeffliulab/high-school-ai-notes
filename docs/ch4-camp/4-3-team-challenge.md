## 为什么团队赛是另一种比赛

到了集训营这一关，你会遇到一种和前面完全不同的考法：团队挑战。几个人组成一队，在有限时间里共同解决一个开放问题——通常没有标准答案，比的是谁的方案在排行榜上分数更高，或者谁的报告更有说服力。

很多平时单兵作战很强的同学，一进团队赛就懵了。原因很简单：个人赛里你只要管好自己的脑子，而团队赛里，**最大的瓶颈往往不是题难，而是三个人的力气没往一处使**。你写你的模型，他洗他的数据，结果两份代码对不上，最后半小时全花在合并冲突和"你那个变量叫什么来着"上。所以团队赛真正考的，是工程协作能力——这恰恰是 USAAIO 选拔越往后越看重的东西。

这一节我们就把团队赛拆开讲：怎么分工、怎么沟通、遇到分歧怎么办。

## 团队赛的规则长什么样

不同赛事细则不一，但团队挑战一般有这么几个共同点，你心里要先有数：

- **一队通常 3 到 4 人**，提交以队为单位，全队共享同一个最终分数；
- **时间是硬约束**，常见是连续几个小时甚至一整天，中途不重置；
- **算力共享**：往往一队只有一两台 GPU 或一个 Colab 配额，谁都想用，这本身就是个要协调的资源；
- **评分多元**：除了排行榜分数，常常还要交一份方案说明或做一次答辩，所以"做出来"之外还得"讲得清"。

把这几条连起来看，你就明白了：团队赛奖励的不是某个人的灵光一现，而是一支队伍**稳定、协调、不内耗**地把一件事做完的能力。

## 三条线分工：建模、数据、调试

人一多，最忌讳的就是三个人挤在一个屏幕前看同一行代码。比较稳的做法是按"职责线"分工，让每个人有一块自己负责、又互相咬合的地盘。我们一般分成三条线：

| 角色 | 主要职责 | 交付给队友的东西 |
| --- | --- | --- |
| 建模手（Modeling） | 选模型架构、写训练循环、调网络结构 | 一个能跑的 baseline 和清晰的训练接口 |
| 数据手（Data） | 清洗、特征工程、数据增强、划分训练/验证集 | 一份干净、版本固定的数据集 |
| 调试手（Debugging） | 跑实验、记录结果、做误差分析、管理提交 | 一张实验记录表和"下一步调什么"的建议 |

这套分工的关键不在于把人隔开，而在于**接口要约定清楚**。比如数据手交出来的数据，列名、划分方式、文件路径都得固定下来，建模手才不用反复问。它们之间的衔接，正是 [1.1 误差分析与调试](4-1-1-error-analysis.md) 和 [1.2 调参与实验管理](4-1-2-experiment-management.md) 里讲的那套方法在团队场景下的放大版。

要注意，三个角色不是"各干各的"。最理想的节奏是：数据手先搭出能用的数据，建模手立刻跑通一个最糙的 baseline（哪怕分数很低），调试手马上开始记录——**先让整条流水线转起来，再分头优化**。千万别让建模手憋一个完美模型憋两小时，等他抬头时数据还没洗完，那就全完了。

## 赛中怎么沟通才不内耗

沟通这件事，听起来软，其实是团队赛里最该提前定好规则的硬骨头。给你几条经过实战检验的铁律：

1. **建一份共享文档当"大脑"**。所有人随时能看到：当前最好成绩是多少、用的是哪套配置、谁正在做什么、下一步打算试什么。脑子里的想法不写下来，队友就看不见，于是会重复劳动。
2. **代码用 Git 分支，别用聊天框传文件**。每条线在自己的分支上干活，合并前先说一声。最怕的就是两个人同时改 `train.py`，最后谁也不知道哪份是对的。
3. **约定一个"同步节点"**。比如每过一小时停三分钟，各自说一句"我做到哪了、卡在哪"。这点时间换来的是避免方向跑偏，绝对值。
4. **改动接口前先吭一声**。数据手要把标签从 0/1 改成 one-hot，建模手的代码可能就崩了——这种会影响别人的改动，改之前一定先在共享文档或群里说一句。

这份共享文档可以简单到就是一个在线表格，但它必须真的被维护。记住一句话：**没写下来的进展，等于没发生。**

## 走一遍：一个示例场景

光说规则太抽象，我们假设你们三人队伍抽到这样一道题：给一批带噪声的传感器信号，要分成 5 类，4 小时内提交，排行榜按准确率排名。看看一支配合默契的队伍会怎么走。

**第 0 到 20 分钟，对齐。** 三个人先一起读题，把评分标准、数据格式、提交方式弄清楚，然后当场分工：谁建模、谁管数据、谁调试，并在共享文档里写下来。这二十分钟看着像没产出，其实是最值钱的——方向对了，后面才不返工。

**第 20 到 60 分钟，先把流水线打通。** 数据手快速读数据、切出验证集；建模手用一个最简单的模型（比如一个小 MLP）跑通"读数据 → 训练 → 在验证集上算分 → 生成提交文件"的全流程；调试手提交一次，确认排行榜真的能收到分数。这时分数低没关系，关键是**整条链路验证通了**，后面就是不断往上调。

**第 1 到 3 小时，分头爬坡。** 数据手开始做特征和数据增强，建模手换更强的网络、调结构，调试手专心跑实验、记录每次改动带来的分数变化，并据此提建议："上次加了滤波涨了 2 个点，这次试试再调学习率。"这个阶段，那张实验记录表就是全队的指挥棒。

**最后 1 小时，收口。** 停止一切大改动，挑出验证集上最好的那个配置，把提交反复检查一遍（格式对不对、有没有用错文件），如果要交报告就分头写。**最后半小时绝不上新模型**——这是团队赛血的教训，新东西经常带来你来不及发现的 bug。

走完这一遍你会发现，真正决定成绩的，是前 20 分钟的对齐和最后 1 小时的克制，而不是中间谁的模型更花哨。

## 队里吵起来了怎么办

人多了，分歧难免。常见的是两种：一是**方案之争**（"该用 CNN 还是 Transformer"），二是**资源之争**（"GPU 该先给谁用"）。处理它们有个共同原则——**用数据说话，用时间止损**。

方案吵不出结果时，别靠嗓门大决定，而是定一个小实验和一个时限："我们各自跑 20 分钟，谁的验证分高就用谁的。"让排行榜或验证集当裁判，比谁都公正。资源紧张时，就按"谁的实验预期收益大、谁先用"来排队，并写进共享文档，省得反复扯皮。

还有一种隐性冲突最伤队伍：**有人闷头干、不同步**。他可能正埋头重写整个模型，却没人知道。化解办法就是前面说的同步节点——强制每个人定时冒个泡。记住，团队赛里**沉默不是美德，及时暴露问题才是。** 一个卡了一小时不说话的队友，比一个频繁报告坏消息的队友危险得多。

## 容易踩的坑

- **前期不对齐就开干**：省下的 20 分钟，会在后面用几小时的返工还回来。务必先把分工和接口定清楚。
- **最后关头还上新模型**：临交卷换方案，等于把没验证过的代码推上线，翻车概率极高。最后一小时只做收口和检查。
- **共享文档没人维护**：文档建了却不更新，等于没建。它必须实时反映"当前最好成绩 + 谁在做什么"。
- **直接传文件代替 Git**：在聊天框里互传 `train.py`，迟早会搞混版本。老老实实用分支。
- **改接口不打招呼**：悄悄改了数据格式或函数签名，会让队友的代码莫名其妙地崩。影响别人的改动，改前必须通知。

## 它在后面会怎么用到

团队挑战练的是协作，但它每一步又都建立在你的个人功底上：临场分工和取舍靠的是 [1.3 限时解题策略](4-1-3-time-strategy.md)，调试手做误差分析靠 [1.1 误差分析与调试](4-1-1-error-analysis.md)，实验记录表则是 [1.2 调参与实验管理](4-1-2-experiment-management.md) 的直接应用。具体到不同题型，建模手要面对的可能是 [2.1 CV 类真题](4-2-1-cv-problems.md)、[2.2 NLP 类真题](4-2-2-nlp-problems.md) 或 [2.3 信号·优化类真题](4-2-3-signal-opt-problems.md) 里的任意一种。

再往后，团队赛的表现会直接影响 [4. 选拔测试准备](4-4-selection-tests.md)，因为协作能力正是选拔看重的一环；而到了国际赛 [IOAI 赛制详解](../ch5-champion/5-1-1-ioai-format.md)，团队赛更是正式赛制的一部分，这里练的每一招都会真刀真枪地用上。

## 练习

??? note "基础练习"
    1. 给自己的队伍（哪怕只是和两个同学约着练）写一份"团队赛分工模板"：列出三个角色各自的职责、要交付给队友的东西、以及彼此之间约定好的数据/代码接口。
    2. 设计一张"实验记录表"应该包含哪些列（提示：至少要有改了什么、验证分、是否提交、下一步建议），并解释每一列为什么有用。

??? note "进阶练习"
    1. 找两位同学组队，用一个 Kaggle 入门赛做一次 2 小时的模拟团队赛：严格按三条线分工，全程维护一份共享文档，结束后复盘——时间主要花在哪、哪次沟通失误最贵。
    2. 设计一套"分歧仲裁规则"：当队内对模型方案争执不下时，用什么样的小实验、多长的时限、什么指标来快速做决定？把它写成一句队友都认可的明确规则。

## 小结

- 团队赛比的不是单点最强，而是一支队伍稳定、不内耗地把事做完——瓶颈常在协作，不在题目。
- 按建模 / 数据 / 调试三条线分工，先打通流水线再分头爬坡，接口一定要约定清楚。
- 沟通靠一份实时维护的共享文档和定期同步节点；记住"没写下来的进展等于没发生"。
- 冲突用数据和时限来裁决，最忌讳的是有人闷头干不说话——及时暴露问题才是团队美德。

想找一支队伍真刀真枪练手，最好的去处是 [Kaggle](https://www.kaggle.com/competitions) 上允许组队的入门赛，找两三个同学组队打一场，比看十篇攻略都长经验。
