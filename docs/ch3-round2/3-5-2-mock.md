## 为什么模拟和复盘比"再多学一节"更值钱

到了 Round 2 这个阶段，你大概率已经把注意力机制、Transformer、目标检测、CLIP、语言建模这些硬骨头都啃过一遍了。可"学过"和"能在限时、用 GPU、跨多个领域的卷子上拿分"完全是两回事。真正拉开差距的，往往不是谁多懂一个冷门模型，而是谁能在两小时里稳稳地把会做的题全做对、把环境坑全绕过去。

所以这一节我们不再讲新知识，而是带你做一件更划算的事：**做一套综合模拟，然后用模板把每一处失分榨干价值**。一道做错的题，如果只是订正答案就翻篇，那它只值 1 分；但如果你顺着它找到"原来我对交叉熵的输入维度一直没搞清"，那它能帮你避开后面十道同类题——这才是模拟真正的回报。

## 一套贴近 Round 2 的综合模拟（CV + NLP）

下面这套题按 Round 2 的实际形态设计：跨两个领域、有代码、有需要你判断的"为什么"。建议你**先掐表 90 分钟自己做完，再回来看解析**。如果时间紧，也可以只挑一个域深做，但综合卷的价值正在于"切换领域"本身——很多人栽在脑子没从图像切回文本上。

**第一部分 · CV（目标检测）。** 给你一个已经训练好的检测模型在一张图上的原始输出：每个候选框带一个置信度分数，框与框之间有重叠。题目要求你用非极大值抑制（NMS, Non-Maximum Suppression）把重复框去掉。这考的是你是否真懂检测的后处理，而不只是会调库。

```python
import numpy as np

def nms(boxes, scores, iou_thresh=0.5):
    # boxes: (N, 4)，每行是 [x1, y1, x2, y2]；scores: (N,) 置信度
    # 思路：每次留下当前分数最高的框，再把和它重叠太多的框全删掉
    idxs = scores.argsort()[::-1]      # 按分数从高到低排序的下标
    keep = []
    while len(idxs) > 0:
        i = idxs[0]                    # 当前最高分的框，一定保留
        keep.append(i)
        if len(idxs) == 1:
            break
        # 计算它和其余所有框的交并比 IoU
        xx1 = np.maximum(boxes[i, 0], boxes[idxs[1:], 0])
        yy1 = np.maximum(boxes[i, 1], boxes[idxs[1:], 1])
        xx2 = np.minimum(boxes[i, 2], boxes[idxs[1:], 2])
        yy2 = np.minimum(boxes[i, 3], boxes[idxs[1:], 3])
        inter = np.maximum(0, xx2 - xx1) * np.maximum(0, yy2 - yy1)
        area_i = (boxes[i, 2] - boxes[i, 0]) * (boxes[i, 3] - boxes[i, 1])
        area_o = ((boxes[idxs[1:], 2] - boxes[idxs[1:], 0]) *
                  (boxes[idxs[1:], 3] - boxes[idxs[1:], 1]))
        iou = inter / (area_i + area_o - inter)
        # 只保留重叠不超过阈值的框，进入下一轮
        idxs = idxs[1:][iou < iou_thresh]
    return keep
```

读完这段你要能讲清一件事：NMS 的核心就是"贪心地留高分、删重叠"。`iou < iou_thresh` 这一句是灵魂——阈值调高，框留得多（容易重复检测）；调低，框删得狠（容易把挨得近的两个物体当成一个）。考场上如果题目说"同类物体经常被合并成一个框"，你第一反应就该是把这个阈值调高，这背后的检测原理见 [3.1 目标检测](3-3-1-object-detection.md)。

**第二部分 · NLP（文本分类的损失计算）。** 同一张卷子切到文本域：给你一个分类模型最后一层的原始输出（logits）和真实标签，要求你算出这一批的交叉熵损失。这考的是你对"logits、softmax、交叉熵"三者关系的真实掌握。

```python
import torch
import torch.nn.functional as F

# logits: (batch=3, num_classes=4) 的原始分数，没过 softmax
logits = torch.tensor([[2.0, 0.5, 0.1, 0.2],
                       [0.1, 0.3, 3.0, 0.2],
                       [1.0, 1.0, 1.0, 1.2]])
labels = torch.tensor([0, 2, 3])           # 每个样本的正确类别下标

loss = F.cross_entropy(logits, labels)     # 内部已包含 softmax + log + NLL
print(loss.item())
```

这里最容易出错、也最常被考的一点：`F.cross_entropy` 的第一个输入必须是**没过 softmax 的 logits**，标签是**类别下标**（不是独热向量）。很多人手动先做了一次 softmax 再喂进来，等于做了两次，损失就全错了。背后的原理在 [激活函数与损失函数](../ch2-round1/2-2-3-activations-losses.md) 里有详细推导，这里你只要把这个"接口契约"刻进肌肉记忆。

## 一份能反复用的错误分析模板

题做完、分打完，真正的功夫在这一步。我推荐你对**每一道失分题**都过一遍下面这张表，把它当成体检报告——不是为了难过，而是为了精准地知道下一步补哪。

| 失分类型 | 典型表现 | 该补的动作 |
| --- | --- | --- |
| 概念没懂 | 思路从一开始就错，做完也讲不出为什么 | 回到对应知识点那一节重学，别急着刷题 |
| 接口/维度搞错 | 思路对，但代码报 shape / 类型错误 | 把该 API 的输入输出维度抄一遍、记牢 |
| 环境/GPU 问题 | 代码逻辑没问题，是跑不起来或太慢 | 对照下面的"GPU 排障清单" |
| 时间不够 | 会做，但没做到 | 练限时，先捡分高、好拿的题 |
| 粗心 | 全懂，一个符号/下标写反 | 留 5 分钟复查，养成检查 shape 的习惯 |

用法很简单：每错一题，先问自己"这属于哪一行？"。你会很快发现自己的失分**高度集中**在某一两行——有人全是"接口搞错"，那就该专门花一晚上把常用 API 的输入输出过一遍；有人全是"时间不够"，那问题根本不在知识，而在做题顺序和取舍策略，这部分可以接着看 [限时解题策略](../ch4-camp/4-1-3-time-strategy.md)。

## GPU 跑不动？一张排障清单照着走

Round 2 开始用 GPU，很多人第一次在真实环境里被"非知识性"的报错坑掉大段时间。下面这张清单按出现频率从高到低排，遇到问题从上往下对：

- **`CUDA out of memory`（显存炸了）**：最常见。先把 batch size 调小一半试试；推理/验证时记得用 `with torch.no_grad():`，它能省掉一大块用于反向传播的显存；还不行就用混合精度（`torch.cuda.amp`）。
- **模型和数据不在同一个设备上**（`Expected all tensors to be on the same device`）：你大概率把模型 `.to("cuda")` 了，却忘了把这一批数据也 `.to("cuda")`。检查训练循环里每个 batch 有没有搬到 GPU。
- **跑得慢、GPU 占用却很低**：瓶颈往往不在 GPU 而在数据加载。给 `DataLoader` 加上 `num_workers` 和 `pin_memory=True`，让 CPU 提前把下一批数据准备好。
- **`device-side assert triggered` 这种看不懂的报错**：十有八九是标签越界——比如你有 4 个类别，标签里却混进了一个 4（下标该是 0~3）。先去查标签的取值范围。
- **结果每次都不一样、没法复现**：训练前固定随机种子（`torch.manual_seed`、`np.random.seed`），调试期可临时设 `torch.backends.cudnn.deterministic = True`。
- **干脆没用上 GPU**：先 `print(torch.cuda.is_available())`。返回 `False` 就是环境没装对，别再瞎调代码了。

这套清单的来历和更系统的 GPU 训练配置，可以回看 [1.1 GPU 训练](3-1-1-gpu-training.md)；环境本身怎么搭，见 [A4 环境与工具](../appendix/a4-tools.md)。

## 容易踩的坑

- **不掐表的模拟等于白做**：Round 2 真正的敌人之一是时间。平时不限时，考场上一定慌。
- **交叉熵重复做 softmax**：`F.cross_entropy` 自带 softmax，别手动再来一次（这是最高频失分点之一）。
- **只订正答案、不归类失分**：错题不过"错误分析模板"，下次还在同一个坑里栽。
- **把 GPU 报错当成知识题硬磕**：显存、设备不一致这类问题，先查清单，别怀疑自己的算法。
- **跨域切换没"切脑子"**：从图像题跳到文本题，输入形状、损失、评估指标全变了，惯性思维最容易在这里出错。

## 它在后面会怎么用到

这一节是 Round 2 备考的"收口"环节：把前面所有知识点放进一张综合卷里压力测试一遍。模拟里暴露的每一类问题，后面都有专门的地方深挖——失分归类后的系统性误差诊断，见 [1.1 误差分析与调试](../ch4-camp/4-1-1-error-analysis.md)；想找更多真题和可练手的数据集来加练，去 [A2 真题与数据集](../appendix/a2-datasets.md)；做完 Round 2 全真模拟后，下一站就是集训营的 [CV 类真题](../ch4-camp/4-2-1-cv-problems.md) 和 [NLP 类真题](../ch4-camp/4-2-2-nlp-problems.md)，难度和综合度会再上一个台阶。

## 练习

??? note "基础练习"
    1. 给定 5 个候选框和置信度，手动跑一遍上面的 `nms` 函数，并改变 `iou_thresh`（0.3 / 0.7），观察保留下来的框数量怎么变，再用一句话解释原因。
    2. 故意把 `F.cross_entropy` 的输入先做一次 `F.softmax` 再喂进去，对比损失值的差别，亲眼确认"为什么不能做两次"。

??? note "进阶练习"
    1. 找一套你做过的模拟卷，把每道失分题填进"错误分析模板"表格，统计哪一类失分最多，并据此定一个"今晚就补哪一节"的具体计划。
    2. 故意制造一次 `CUDA out of memory`（把 batch size 调到极大），然后照着 GPU 排障清单一项项尝试，记录哪个办法最先把它救活。

## 小结

一句话：**Round 2 拼的不只是会不会，更是能不能在限时、跨域、用 GPU 的真实环境里稳稳拿分**——所以做完模拟一定要用错误分析模板归类失分，再配上一份 GPU 排障清单兜底。想进一步加练，去 [A2 真题与数据集](../appendix/a2-datasets.md) 取材，并参考 [PyTorch 官方 CUDA 语义文档](https://pytorch.org/docs/stable/notes/cuda.html) 把显存和设备这一块吃透。
