## 同一条数据，能画出无数条分界线，哪条最好

我们先回到一个最朴素的问题。假设平面上有两类点，一类是圆圈、一类是叉号，它们之间留着一条明显的空隙。现在要你画一条直线把两类分开——你会发现，能把它们分开的直线有无数条，往左偏一点能分开，往右偏一点也能分开。

那么问题来了：**这无数条线里，哪一条才是"最好"的？** 逻辑回归（见 [1.3 逻辑回归与分类](2-1-3-logistic-regression.md)）会给你一条，但它并不特别在意这条线离两边的点有多远。直觉上我们会觉得，**一条好的分界线，应该离两边的点都尽量远**——因为留的余地越大，将来来一个新点，哪怕它稍微偏一点，也不至于被分错。

支持向量机（Support Vector Machine，简称 SVM）正是把这个"离得越远越好"的直觉变成了数学。它要找的，是那条让两类点之间"空隙"最宽的线。这条空隙的宽度，有个专门的名字叫**间隔（margin）**，而 SVM 干的事，就叫**间隔最大化**。

## 间隔最大化，本质上是一道带约束的凸优化题

现在把直觉翻译成数学。一条分界线在数学上是一个**超平面（hyperplane）**，写成

$$\mathbf{w}^\top\mathbf{x}+b=0.$$

这里 $\mathbf{w}$ 是法向量，决定了线的"朝向"，$b$ 决定了线的位置。对任意一个点 $\mathbf{x}$，$\mathbf{w}^\top\mathbf{x}+b$ 的正负号就告诉我们它落在线的哪一边——这正是我们用来分类的依据。

我们把两类的标签记成 $y_i\in\{+1,-1\}$。如果分类正确，正类点的 $\mathbf{w}^\top\mathbf{x}+b$ 应该是正的、负类是负的，于是不管哪一类，都满足 $y_i(\mathbf{w}^\top\mathbf{x}_i+b)>0$。SVM 更进一步，要求每个点不仅分对，还要离线**足够远**，远到至少满足

$$y_i(\mathbf{w}^\top\mathbf{x}_i+b)\ge 1.$$

可以证明，间隔的宽度恰好等于 $\dfrac{2}{\lVert\mathbf{w}\rVert}$。要让间隔最大，就得让 $\lVert\mathbf{w}\rVert$ 最小。把它写成更顺手的平方形式，SVM 的目标就成了：

$$\min_{\mathbf{w},b}\ \frac{1}{2}\lVert\mathbf{w}\rVert^2 \quad\text{s.t.}\quad y_i(\mathbf{w}^\top\mathbf{x}_i+b)\ge 1,\ \forall i.$$

请你停下来看一眼这个式子的"形状"：要最小化的目标 $\frac{1}{2}\lVert\mathbf{w}\rVert^2$ 是一个标准的"碗形"，也就是凸函数；约束又全是线性不等式。**这恰好就是一道我们在 [3.4 凸优化基础](../ch1-intro/1-3-4-convex-optimization.md) 里见过的带约束的凸优化问题。** 凸问题最大的好处是：它只有一个全局最优解，不会卡在某个局部最低点出不来。所以 SVM 训练出来的那条线，是数学上唯一确定的"最好"的线——这正是它和很多其他模型不一样、也格外让人放心的地方。

## 支持向量：决定分界线的，只是那么几个点

这里有一个非常漂亮、也很反直觉的结论，值得你专门记住。

把上面那道带约束的优化题，用拉格朗日乘子法（这正是 [3.4 凸优化基础](../ch1-intro/1-3-4-convex-optimization.md) 里处理约束的标准武器）转化一下，会得到一个结果：最终的 $\mathbf{w}$ 只由**少数几个恰好落在间隔边界上的点**决定，其余那些离得远远的点，权重全是零，对结果毫无贡献。

这几个"卡在边界上、真正撑起了分界线"的点，就叫**支持向量（support vector）**——这也是整个算法名字的由来。你可以这样体会它的含义：哪怕你把那些离得远的点统统删掉，只留下这几个支持向量，重新训练，得到的还是同一条线。换句话说，**整个模型其实是被边界上那几个最"难分"的点撑起来的**，就像一顶帐篷，真正吃力的只有支撑杆所在的那几个点。

这个性质有个实在的好处：预测时，模型只需要和这几个支持向量比一比，而不必记住全部训练数据，所以又快又省。

## 现实数据没那么干净：软间隔与 C

到目前为止我们都假设两类能被一条直线干净利落地分开。可现实里，数据往往是有噪声的——总有那么几个点跑到了对面的"地盘"，硬要一条直线把它们完全分开，要么做不到，要么会逼出一条扭曲得很离谱、只为迁就个别噪声点的线。

为了解决这个问题，SVM 允许少数点"越界"，但每越一点界就记一笔"罚款"。这种放宽叫**软间隔（soft margin）**。目标函数因此多了一项惩罚：

$$\min_{\mathbf{w},b}\ \frac{1}{2}\lVert\mathbf{w}\rVert^2 + C\sum_i \xi_i,$$

其中 $\xi_i\ge 0$ 衡量第 $i$ 个点越界了多少。这里的 $C$ 是一个你来设定的旋钮，它管的是"对越界有多较真"：

- **$C$ 很大**：罚款很重，模型尽量不让任何点越界，间隔会变窄，容易**过拟合**——把噪声也当真了。
- **$C$ 很小**：罚款很轻，模型愿意牺牲几个点换更宽的间隔，泛化往往更好，但太小又会**欠拟合**。

所以 $C$ 本质上是在"间隔要宽"和"别分错"之间做权衡，是 SVM 最重要的超参数之一。

## 核技巧：直线分不开，就换个空间分

软间隔解决了"有几个噪声点"的问题，但还有一种更棘手的情况：两类数据**本质上就不是一条直线能分开的**。比如一类点聚在中间，另一类把它团团围在外圈——任你怎么画直线，都没法把里圈和外圈分开。

怎么办？SVM 有一个堪称神来之笔的办法。它的思路是：**既然在原来的低维空间里分不开，那就把数据映射到一个更高维的空间，在那里它们也许就能被一个超平面分开了。** 比如上面"里外两圈"的例子，只要给每个点添一个新维度，让外圈的点"抬高"、里圈的点"压低"，立刻就能用一个平面从中间切开。

听起来很美，但有个现实障碍：维度升得太高，计算量会爆炸。**核技巧（kernel trick）的精妙之处就在这里**——它发现，SVM 的整个计算过程其实只用到两个点的**内积**，而不需要真的把每个点都搬到高维空间里去。于是我们可以直接定义一个**核函数（kernel）** $K(\mathbf{x}_i,\mathbf{x}_j)$，让它一步算出"两个点在高维空间里的内积"，省掉了显式升维这一整个昂贵步骤。

最常用的核就是**高斯核**，也叫 RBF 核：

$$K(\mathbf{x}_i,\mathbf{x}_j)=\exp\!\big(-\gamma\,\lVert\mathbf{x}_i-\mathbf{x}_j\rVert^2\big).$$

它衡量的是两个点有多"近"：离得越近，值越接近 1；离得越远，值越接近 0。这里的 $\gamma$ 是另一个关键旋钮——$\gamma$ 越大，模型越关注"非常近"的邻居，决策边界会变得很弯曲、很贴合训练点，从而容易过拟合；$\gamma$ 越小，边界越平滑。所以用 RBF 核时，**$C$ 和 $\gamma$ 要一起调**，这一点我们在 [1.9 模型评估与选择](2-1-9-model-evaluation.md) 里会用网格搜索系统地处理。

## 上手：用 scikit-learn 训练一个 SVM

道理讲完了，我们动手跑一遍。SVM 的对偶问题求解涉及二次规划，自己从零写既繁琐又容易出数值问题，所以这一节我们直接用 scikit-learn 的 `SVC`（Support Vector Classifier），把精力放在"理解每个参数在干什么"上。

先在一个线性可分的玩具数据上看看线性 SVM 长什么样：

```python
import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_blobs

# 造两团明显分开的点
X, y = make_blobs(n_samples=60, centers=2, cluster_std=1.1, random_state=6)

clf = SVC(kernel="linear", C=1.0)   # 线性核、软间隔强度 C=1
clf.fit(X, y)

# 模型学到的法向量 w 和偏置 b
print("w =", clf.coef_[0])
print("b =", clf.intercept_[0])
# 支持向量：撑起这条线的那几个点
print("支持向量的个数：", len(clf.support_vectors_))
```

跑完你会看到，`支持向量的个数`通常只有两三个——这正印证了前面那个结论：**几十个点里，真正决定分界线的就那么几个。** `clf.support_vectors_` 里存的就是这几个点的坐标。

接着换上 RBF 核，去对付直线分不开的数据，并亲眼感受 `gamma` 的威力：

```python
from sklearn.datasets import make_circles

# 里外两圈，线性核必败
Xc, yc = make_circles(n_samples=200, factor=0.4, noise=0.08, random_state=0)

for g in [0.1, 1.0, 10.0]:
    model = SVC(kernel="rbf", C=1.0, gamma=g)
    model.fit(Xc, yc)
    acc = model.score(Xc, yc)            # 这里只看训练集，纯为演示 gamma 的影响
    print(f"gamma={g:>4}  训练准确率={acc:.3f}  支持向量数={len(model.support_)}")
```

你会观察到：`gamma` 太小时模型太"迟钝"、连训练集都分不太好；`gamma` 太大时训练准确率冲到很高，但支持向量数量也激增——这往往是过拟合的信号，它把每个训练点都背下来了。真正可靠的做法是用交叉验证去挑 `C` 和 `gamma`，而不是只盯着训练准确率（详见 [1.9 模型评估与选择](2-1-9-model-evaluation.md)）。

最后补一句最实用的提醒：**SVM 对特征的量级极其敏感**，因为它算的是距离和内积。所以正式用之前，几乎总要先把特征标准化：

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

# 标准化 + SVM 打包成一条流水线，避免数据泄漏
pipe = make_pipeline(StandardScaler(), SVC(kernel="rbf", C=1.0, gamma="scale"))
pipe.fit(Xc, yc)
print("流水线准确率：", pipe.score(Xc, yc))
```

把 `StandardScaler` 和 `SVC` 用 `make_pipeline` 串起来，标准化的参数只会从训练集学，避免把测试集信息偷偷漏进来——这是个能让你在 Round 1 表格题里少丢分的好习惯。

## 容易踩的坑

- **没做特征标准化**：SVM 靠距离吃饭，如果一个特征是"年龄"（几十）、另一个是"收入"（几万），收入会完全压过年龄。喂数据前先 `StandardScaler`，这几乎是必修动作。
- **`C` 和 `gamma` 只调一个**：用 RBF 核时这两个旋钮是耦合的，单独调任何一个都可能误导你，要用网格搜索一起调。
- **把 `gamma` 调太大当成"准确率高"**：训练准确率随 `gamma` 飙升往往是过拟合，一定要看交叉验证或测试集的表现。
- **在超大数据集上硬跑核 SVM**：核 SVM 的训练复杂度大致随样本数的平方甚至立方增长，几十万条数据就会慢到难以忍受——这时该换线性 SVM（`LinearSVC`）或别的模型。
- **忘了 SVM 默认不输出概率**：`SVC` 默认只给类别。若需要概率，要设 `probability=True`，但它会显著变慢。

    2. 固定 RBF 核，把 `C` 依次取 `0.01, 1, 100`，在同一份带噪声的数据上训练，观察决策边界（可以画散点图辅助），用一句话说清 `C` 变大时边界发生了什么变化。

??? note "进阶练习"
    1. 在 `make_circles` 数据上，用 `GridSearchCV` 对 `C` 和 `gamma` 做一次网格搜索（配 5 折交叉验证），报告最优参数和交叉验证准确率，再和你随手设的参数对比，看提升了多少。
    2. 试着只用 NumPy 实现"线性可分情形下的硬间隔 SVM"：把它写成一个二次规划问题，调用 `scipy.optimize` 或 `cvxopt` 求解，再把解出的 $\mathbf{w},b$ 和 `SVC(kernel="linear")` 的结果对比。这一步能让你彻底搞懂"对偶变量非零的点就是支持向量"。

## 小结

- SVM 找的是**间隔最大**的那条分界线——离两边都尽量远，所以泛化更稳。
- 这条线只由少数几个**支持向量**撑起来，其余点删了也不影响结果。
- 间隔最大化是一道**带约束的凸优化**问题，只有唯一的全局最优解。
- **软间隔（C）**容忍噪声，**核技巧（gamma + 核函数）**让直线分不开的数据变得可分；用 SVM 前务必先**标准化特征**。

想把核方法的几何直觉建得更牢，推荐看 scikit-learn 官方的 [SVM 文档与示例](https://scikit-learn.org/stable/modules/svm.html)，它配了大量可视化；更系统的推导见 [*Mathematics for Machine Learning*](https://mml-book.github.io/) 第 12 章"用 SVM 做分类"。
