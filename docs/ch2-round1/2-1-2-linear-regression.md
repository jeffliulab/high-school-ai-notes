# 1.2 线性回归

> **难度** ⭐⭐☆☆☆ · **前置**：[3.1 线性代数](../ch1-intro/1-3-1-linear-algebra.md)、[3.2 微积分与梯度](../ch1-intro/1-3-2-calculus-gradients.md)

!!! abstract "读完这一节，你会"
    - 用"找一条最贴合数据的直线"的眼光理解线性回归，并说清 MSE 损失为什么长那样
    - 掌握求解它的两条路：一步到位的**正规方程**（闭式解）和一小步一小步逼近的**梯度下降**
    - 能用纯 NumPy 从零把两种解法都写出来，再用 scikit-learn 对一遍答案

## 它到底在解决什么问题

我们从一个最朴素的场景说起。假设你手上有一批二手房的数据：每套房子记了它的面积，以及最后的成交价。现在来了一套新房子，只知道面积，你想预测它大概能卖多少钱——这就是一个典型的**回归（regression）**问题，因为我们要预测的是一个**连续的数值**（价格），而不是"是猫还是狗"那种类别。

最简单也最直观的想法是：**在散点图上画一条直线，让它尽量从所有点的"中间"穿过去。** 以后来了新面积，就顺着这条直线读出对应的价格。线性回归（linear regression）做的就是这件事——它假设输出和输入之间是一条直线（或在多个特征时是一个平面/超平面）的关系：

$$\hat{y} = w_1 x_1 + w_2 x_2 + \dots + w_d x_d + b.$$

这里 $x_1,\dots,x_d$ 是房子的各个特征（面积、楼层、房龄……），$w$ 是每个特征的**权重**（weight，表示它对价格的影响有多大），$b$ 是**偏置**（bias，可以理解成"基础价"）。我们的任务，就是从数据里把这组 $w$ 和 $b$ 给学出来。

别看它简单，线性回归是几乎所有机器学习课程的"第一课"，因为它把后面要反复用到的三样东西——**模型、损失、优化**——以最干净的形式凑齐了。把它吃透，后面的逻辑回归、神经网络都只是它的"加强版"。

## 用一个小技巧把 b 也塞进矩阵

往下推导之前，先做一个会让公式干净很多的约定。那个偏置 $b$ 单独拖在外面有点碍事，我们可以给每个样本人为地添上一个恒等于 $1$ 的"假特征"，再把 $b$ 当成这个假特征的权重。这样一来，$w_1x_1+\dots+w_dx_d+b$ 就能整齐地写成一个向量内积 $\mathbf{w}^\top\mathbf{x}$。

把所有 $n$ 个样本摞起来，每行是一个样本（已经补上了那一列 $1$），就得到一个 $n\times(d{+}1)$ 的**设计矩阵** $X$；把真实价格摞成一列向量 $\mathbf{y}$。于是整个模型可以一口气写成：

$$\hat{\mathbf{y}} = X\mathbf{w}.$$

看，这正是 [3.1 线性代数](../ch1-intro/1-3-1-linear-algebra.md)里讲的"矩阵乘向量"。整个线性回归，本质上就是在这个矩阵运算的框架里找那个最好的 $\mathbf{w}$。

## 用 MSE 衡量"这条线画得好不好"

要让机器自己找出最好的直线，我们得先告诉它一个标准：怎么算一条线"好"还是"不好"。直觉很清楚——**每个点的预测价和真实价差得越小，这条线就越好。**

把"差距"写成数学，最常用的就是**均方误差（Mean Squared Error，MSE）**：

$$L(\mathbf{w}) = \frac{1}{n}\sum_{i=1}^{n}\big(\hat{y}_i - y_i\big)^2 = \frac{1}{n}\lVert X\mathbf{w} - \mathbf{y}\rVert^2.$$

这里 $\hat{y}_i$ 是模型对第 $i$ 套房的预测，$y_i$ 是它的真实价格。两者相减再**平方**，是有讲究的：平方让正负误差都变成正数（不会一正一负互相抵消），而且差得越多、惩罚涨得越狠（差 $2$ 倍误差，惩罚是 $4$ 倍）。所以最小化 MSE，就是在逼模型别犯大错。

## 第一条路：正规方程，一步算出最优解

好消息是：MSE 是一个关于 $\mathbf{w}$ 的**二次函数**，它的图像是一个标准的"碗"（凸函数），只有一个最低点。对于这种函数，我们有办法**直接一步算出**谷底在哪，根本不用慢慢试——这就是**正规方程（normal equation）**。

思路和 [3.2 微积分与梯度](../ch1-intro/1-3-2-calculus-gradients.md)里说的一样：函数在最低点的斜率是零，所以我们对 $\mathbf{w}$ 求梯度、令它等于零。把 $L=\frac{1}{n}\lVert X\mathbf{w}-\mathbf{y}\rVert^2$ 对 $\mathbf{w}$ 求导（这一步用到向量求导，第一次看不懂可以先记结论），得到

$$\nabla_{\mathbf{w}} L = \frac{2}{n}X^\top(X\mathbf{w} - \mathbf{y}).$$

令它等于零，把 $\mathbf{w}$ 孤立到一边，就解出了那个著名的闭式解：

$$\boxed{\;\hat{\mathbf{w}} = (X^\top X)^{-1}X^\top \mathbf{y}\;}$$

你看，它整个就是一串矩阵运算——转置、相乘、求逆，全是 [3.1 线性代数](../ch1-intro/1-3-1-linear-algebra.md)里学过的招式。只要 $X^\top X$ 可逆，这一行公式就直接给出全局最优的 $\mathbf{w}$，没有任何近似。

那它有什么缺点？两个。第一，求逆 $(X^\top X)^{-1}$ 的计算量随特征数量急剧增长，特征上万时会非常慢；第二，当特征之间高度相关时 $X^\top X$ 接近不可逆，求逆会变得数值不稳。所以它适合特征不太多的小问题——而真正的大问题，我们走第二条路。

## 第二条路：梯度下降，一步步走到谷底

当特征太多、正规方程吃不消时，我们改用更通用的办法：**梯度下降（gradient descent）**。它不追求一步到位，而是**从一个随便的起点出发，反复沿着"下坡最快"的方向迈一小步**，慢慢挪到谷底。这正是 [3.2 微积分与梯度](../ch1-intro/1-3-2-calculus-gradients.md)里"蒙眼下山"那个画面。

具体怎么走？我们刚才已经算出了损失对 $\mathbf{w}$ 的梯度，它指向上升最快的方向，那下山就朝它的反方向走。每一轮按这个公式更新参数：

$$\mathbf{w} \leftarrow \mathbf{w} - \eta\,\nabla_{\mathbf{w}}L = \mathbf{w} - \eta\cdot\frac{2}{n}X^\top(X\mathbf{w}-\mathbf{y}).$$

这里的 $\eta$（读作 eta）是**学习率（learning rate）**，也就是每一步迈多大。它是个需要你调的关键旋钮：太小了走得慢、半天到不了谷底；太大了一步迈过头，会在谷底两边来回横跳甚至越跳越远。

那么问题来了——既然正规方程能一步算出精确解，我们为什么还要费劲一步步走？因为梯度下降**几乎对所有模型都适用**：神经网络的损失没有闭式解，唯一的办法就是梯度下降。所以现在用线性回归这个最简单的场景把它练熟，是为后面的深度学习打地基。

## 从零实现：纯 NumPy 把两条路都写一遍

道理讲完，我们动手验证。先用 NumPy 造一批"带噪声的直线"假数据，再分别用正规方程和梯度下降去把那条真实的直线还原出来，看两者答案对不对得上。

第一步，造数据。我们故意设定真实关系是 $y = 3x + 2$，再加一点随机噪声模拟现实：

```python
import numpy as np

rng = np.random.default_rng(0)
n = 200
x = rng.uniform(0, 5, size=n)               # 200 个面积
y = 3 * x + 2 + rng.normal(0, 1, size=n)    # 真实斜率 3、截距 2，加噪声

# 给每个样本补一列 1，把偏置 b 也塞进矩阵
X = np.column_stack([x, np.ones(n)])        # 形状 (200, 2)：第一列是 x，第二列全 1
```

第二步，正规方程。它就是把那行闭式解公式翻译成代码。注意：实战里我们**不直接求逆**，而是用更稳的 `np.linalg.solve` 来解线性方程组——这是 [3.1 线性代数](../ch1-intro/1-3-1-linear-algebra.md)里反复提醒过的好习惯：

```python
# 解 (XᵀX) w = Xᵀy，比先求逆再相乘又快又稳
w_closed = np.linalg.solve(X.T @ X, X.T @ y)
print("正规方程解 [斜率, 截距]:", w_closed)   # ≈ [3.0, 2.0]，成功还原
```

第三步，梯度下降。我们从全零起步，反复套用上面的更新公式，每隔一段打印一次损失，亲眼看着它往下掉：

```python
def mse(X, y, w):
    err = X @ w - y
    return np.mean(err ** 2)

w = np.zeros(2)            # 从一个随便的起点出发
eta = 0.05                # 学习率
for step in range(2000):
    grad = (2 / n) * X.T @ (X @ w - y)   # 损失对 w 的梯度
    w = w - eta * grad                   # 朝下坡方向迈一小步
    if step % 500 == 0:
        print(f"step {step:4d}  loss={mse(X, y, w):.3f}  w={w}")

print("梯度下降解 [斜率, 截距]:", w)       # 同样收敛到 ≈ [3.0, 2.0]
```

你会看到损失从一个很大的值一路下降、最后稳定下来，而 `w` 慢慢逼近 `[3, 2]`——和正规方程算出的几乎一模一样。两条完全不同的路，殊途同归地找到了同一个谷底。这正好印证了前面那句话：MSE 是个只有一个谷底的"碗"。

## 库版对照：scikit-learn 一行搞定

自己从零写一遍是为了**搞懂里面在发生什么**；但真到做题、做项目时，没人会手写这些——成熟的库早就把它打磨得又快又稳了。线性回归在 scikit-learn 里就是一个类，几行就能用：

```python
from sklearn.linear_model import LinearRegression

model = LinearRegression()        # 它内部用的也是闭式解（最小二乘）
model.fit(x.reshape(-1, 1), y)    # 喂入特征和标签，自动学好 w 和 b

print("斜率:", model.coef_)        # ≈ [3.0]
print("截距:", model.intercept_)   # ≈ 2.0
print("预测 x=4 时的价格:", model.predict([[4]]))
```

注意一个小差别：sklearn 默认帮你处理偏置（`fit_intercept=True`），所以你**不用自己补那列 1**，直接喂原始特征即可，斜率在 `coef_` 里、截距在 `intercept_` 里。结果和我们手写的两版完全一致——这正说明我们从零写的没错。

记住一个原则：**理解时从零写，干活时用库。** 库版不是让你偷懒，而是让你把精力省下来花在更难的地方。

## 容易踩的坑

- **忘了给特征做标准化**：如果不同特征的数值范围差得很大（面积是几百、房间数是个位数），梯度下降会收敛得极慢甚至来回震荡。训练前先把特征缩放到同一量级（见 [4.3 Pandas 与数据处理](../ch1-intro/1-4-3-pandas-data.md)），往往只是一行代码的事。
- **学习率没调好**：`eta` 太大时损失不降反升、甚至变成 `nan`；太小时几千步都纹丝不动。调参时先打印每几步的损失，看它是稳稳下降还是在乱跳。
- **直接 `np.linalg.inv` 求逆**：能用 `np.linalg.solve` 就别求逆，前者又慢又容易在矩阵接近奇异时出数值问题。
- **形状对不齐**：`X @ w` 报 `shapes not aligned` 时，第一反应是 `print(X.shape, w.shape)`。sklearn 里单特征还要记得 `x.reshape(-1, 1)`，它要的是二维的 `(n, 1)` 而不是一维。
- **用线性模型硬套非线性数据**：如果数据本身明显是条曲线，再好的直线也拟合不上。这时要么造非线性特征（如 $x^2$），要么换模型——别指望线性回归能学会它学不会的东西。

## 它在后面会怎么用到

线性回归是一颗"种子"，后面很多内容都是从它长出来的：

- 把输出再套一个 Sigmoid 函数、把损失换成交叉熵，线性回归就变成了**逻辑回归**，用来做分类（见 [1.3 逻辑回归与分类](2-1-3-logistic-regression.md)）。
- 这里的"模型—MSE 损失—梯度下降"三件套，正是**感知机和神经网络**训练的雏形，只不过模型从一条直线变成了多层非线性变换（见 [2.1 感知机与 MLP](2-2-1-perceptron-mlp.md)）。
- 给损失加上一个惩罚项（L1/L2 正则）防止过拟合，就得到了 Lasso 和 Ridge 回归（见 [2.5 优化与正则化](2-2-5-optimization-regularization.md)）。
- 梯度下降这套"算梯度、迈一步"的流程，会在 [2.2 梯度下降与反向传播](2-2-2-backprop.md)里被推广到任意复杂的网络。

## 练习

??? note "基础练习"
    1. 用上面造数据的代码，把真实关系改成 $y = -2x + 5$，重新跑正规方程，确认它能还原出斜率 $-2$、截距 $5$。
    2. 把梯度下降的学习率分别设成 `0.001`、`0.05`、`0.5`，各跑一遍并打印损失曲线。观察：哪个收敛太慢？哪个开始震荡甚至发散？
    3. 用 sklearn 训练好模型后，调用 `model.score(X, y)` 看一下 $R^2$ 分数，并查一查这个分数衡量的是什么。

??? note "进阶练习"
    1. 给数据加上第二个特征（比如房间数），把它扩成多元线性回归，分别用从零的正规方程和 sklearn 各解一遍，对比两组 $\mathbf{w}$ 是否一致。
    2. 把全量梯度下降改成**小批量（mini-batch）**：每一步只用随机抽的一小批样本来算梯度。观察损失曲线为什么会变得"抖"，但整体仍在下降——这正是后面训练神经网络的标准做法。
    3. 故意造一组高度共线的特征（让第二列约等于第一列的 2 倍），看正规方程的 $X^\top X$ 求逆会出什么问题，再想想为什么正则化能救它。

## 小结

- 线性回归就是**找一条最贴合数据的直线**，用 **MSE** 衡量"贴得好不好"，平方是为了让误差不抵消、且重罚大错。
- 求解有两条路：**正规方程**一步给出闭式解 $\hat{\mathbf{w}}=(X^\top X)^{-1}X^\top\mathbf{y}$，适合小问题；**梯度下降**一步步逼近，通用且能推广到神经网络。
- 从零写是为了懂原理，干活用 **scikit-learn**；训练前记得标准化特征、调好学习率。
- 想再深入，推荐看 [scikit-learn 官方线性模型文档](https://scikit-learn.org/stable/modules/linear_model.html)，或 Andrew Ng 机器学习课程里讲线性回归与梯度下降的那几节——它把"下山"这件事讲得格外清楚。
