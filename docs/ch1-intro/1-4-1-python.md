## 为什么偏偏是 Python

学 AI 为什么要先学 Python？因为它几乎是这个领域的"普通话"。你后面要用的 NumPy、Pandas、PyTorch，全都是 Python 库；USAAIO 的考试也是在 Python 环境（Colab）里进行的。换句话说，**Python 是你和所有 AI 工具对话的唯一语言**。

好消息是，你不需要成为 Python 专家。我们的目标只有一个——能流畅地把脑子里的想法翻译成代码。Python 的语法非常接近自然语言，正适合快速试验各种模型。下面我们就按"够用就好"的原则，把写 AI 代码会反复用到的那些语法过一遍。

## 变量与基本类型

和很多语言不同，Python 是**动态类型**的：你不需要提前声明变量是什么类型，赋值的时候它自己就知道了。

```python
n = 10           # 整数 int
pi = 3.14        # 小数 float
name = "cat"     # 字符串 str
ok = True        # 布尔值 bool
```

这种"拿来就用"的特性让写代码很快，但也意味着你得自己心里有数：一个变量现在到底装的是数字还是文字。

## 控制流：让程序学会判断和重复

程序要做决策、要重复劳动，靠的就是 `if` 和 `for`。Python 有个鲜明的特点：**它用缩进来划分代码块**，而不是大括号。

```python
for i in range(3):        # 依次取 0、1、2
    if i % 2 == 0:
        print(i, "是偶数")
```

这段代码会把 0 到 2 里的偶数挑出来打印。注意 `if` 下面那行往里缩进了——缩进对不齐，Python 是会直接报错的。

## 函数：把一段逻辑打包

当一段逻辑要反复使用时，就把它装进函数里，起个名字，以后一行调用即可。

```python
def square(x):
    return x * x

print(square(5))          # 25
```

函数让代码不重复、好维护。后面你训练模型时，会把"训练一轮""评估准确率"这类操作都包成函数。

## 容器：列表、字典、元组、集合

数据多了就需要"容器"来装。Python 有四个最常用的，各有各的脾气：

```python
nums = [1, 2, 3]              # 列表 list：有序、可改
d = {"cat": 1, "dog": 2}     # 字典 dict：键 → 值的映射
t = (1, 2)                    # 元组 tuple：有序、但不可改
s = {1, 2, 2, 3}             # 集合 set：自动去重，得到 {1, 2, 3}
```

其中**列表**和**字典**是你用得最多的两个：列表适合按顺序存一串东西，字典适合做"查表"（比如把类别名映射成编号）。

## 推导式：一行写出一个列表

推导式是 Python 特别优雅的一个语法，在 AI 代码里随处可见。它能让你用一行，从一个序列变出另一个序列。

```python
squares = [x * x for x in range(5)]        # [0, 1, 4, 9, 16]
even = [x for x in nums if x % 2 == 0]     # 只留下偶数：[2]
```

读它的时候从中间的 `for` 看起：先遍历 `range(5)`，再把每个 `x` 变成 `x*x`。比起写一个三四行的 `for` 循环，推导式既短又清楚。

## 类：把数据和行为绑在一起

最后是**类**。它把"数据"和"对这些数据的操作"打包成一个整体。你现在可能觉得用处不大，但请一定看懂它——因为 PyTorch 里的每一个模型，都是用类写出来的。

```python
class Counter:
    def __init__(self, start=0):   # 构造函数，创建对象时自动调用
        self.count = start
    def add(self, x):
        self.count += x
        return self.count

c = Counter()
print(c.add(5))        # 5
```

这里 `Counter` 把"当前计数"这个数据和"加一个数"这个行为绑在了一起。等你写 `class Net(nn.Module)` 时，用的就是同一套语法。

## 容易踩的坑

- **可变默认参数**：把默认值写成 `def f(x, acc=[])` 时，那个空列表只会被创建一次，于是多次调用会把结果偷偷累加在一起。正确做法是默认用 `None`，进函数再新建：`def f(x, acc=None): acc = acc or []`。
- **`is` 和 `==` 别混**：`==` 比较的是"值是否相等"，`is` 比较的是"是不是同一个对象"。判断相等永远用 `==`。
- **整除和普通除法**：`5 / 2` 得到 `2.5`，而 `5 // 2` 得到 `2`（整除取整），别用错。
- **赋值不是复制**：`b = a` 只是给同一个列表起了别名，改 `b` 会连 `a` 一起改。想要独立副本，用 `a.copy()`。

## 小结

- Python 动态类型、靠缩进分块、生态强大，是 AI 领域的默认语言。
- **推导式、字典、类**是写模型代码的三件套，尤其类是 PyTorch 的基础。
- 记牢可变默认参数、`is`/`==`、赋值不是复制这三个经典坑，能帮你少掉很多头发。

想再扎实一点，可以过一遍 [Python 官方教程](https://docs.python.org/zh-cn/3/tutorial/) 的前几章。
