## 训练模型，其实就是在下山

整个深度学习，如果用一句话概括它的核心动作，那就是：**沿着梯度的反方向，一小步一小步地把损失"走"到最低点。** 听起来抽象，但它的画面非常朴素。

你可以把"损失函数"想象成一片山地：你所在位置的**高度就是损失的大小**，而你脚下的**坐标就是模型的参数**。我们的目标，是找到这片山地里最低的那个谷底——因为谷底意味着损失最小，也就是模型最准。

问题是，你蒙着眼睛，看不见整座山长什么样，只能感觉脚下哪个方向是下坡。**梯度**就是那个"上坡最陡"的指南针；把它反过来，就是"下坡最快"的方向。朝那个方向迈一小步，损失就降一点，反复迈步，你就慢慢挪到了谷底。

## 导数、偏导、梯度：先给定义，再讲直觉

这三个词是层层递进的，我们先把定义摆出来，再逐一解释。

**导数**衡量的是"函数变化的快慢"，定义为 $f'(x)=\lim_{h\to0}\dfrac{f(x+h)-f(x)}{h}$。直观上，它就是曲线在某一点的**斜率**：斜率越陡，函数变化越剧烈。

当函数有多个自变量时，我们对其中某一个变量求导、把其他变量当常数，得到的就是**偏导数**。把所有偏导数拼成一个向量，就是**梯度**：

$$\nabla f=\Big(\frac{\partial f}{\partial x_1},\dots,\frac{\partial f}{\partial x_n}\Big).$$

那梯度的几何含义是什么？这才是你最该记住的一点：**梯度永远指向函数上升最快的方向**。所以下山时，我们朝它的反方向 $-\nabla f$ 走，就是下降最快的路线。

!!! example "例 1：手算一个梯度"
    设 $f(x,y)=x^2+3y^2$。对 $x$ 求偏导（把 $y$ 看成常数）得 $\dfrac{\partial f}{\partial x}=2x$；同理 $\dfrac{\partial f}{\partial y}=6y$。于是梯度是 $\nabla f=(2x,\,6y)$。在点 $(1,1)$ 处，它等于 $(2,6)$——这说明在这一点，沿 $y$ 方向上升得更陡。

## 链式法则：反向传播的命根子

接下来这条法则，是后面学反向传播时绕不开的核心，务必弄懂。

**链式法则**说的是：如果 $y$ 通过中间量 $u$ 依赖于 $x$，即 $y=g(u)$、$u=h(x)$，那么 $y$ 对 $x$ 的导数等于"一层层导数相乘"：

$$\frac{dy}{dx}=\frac{dy}{du}\cdot\frac{du}{dx}.$$

为什么它这么重要？因为神经网络就是一个"函数套函数套函数"的超级复合函数——输入经过一层层变换才得到输出。要算损失对最前面那些参数的导数，就得用链式法则，从后往前一层层把导数乘回去。这个"从后往前传导数"的过程，就是**反向传播**这个名字的由来。

至于 **Jacobian** 和 **Hessian**，你现在只需知道它们是导数的"高维版本"：前者是向量值函数的一阶导排成的矩阵，后者是二阶偏导排成的矩阵、描述函数的弯曲程度。了解到这个程度就够了，用到时再深入。

## 动手：用代码估计梯度

理解了梯度是什么，我们来用代码把它"看"出来。这里用一个很实用的技巧——**数值梯度**：既然导数是"自变量动一点点、函数变多少"，那我们就真的让每个变量动一个极小的量，看函数的变化，反推出梯度。

先把要研究的函数和数值求梯度的逻辑写出来：

```python
import numpy as np

def f(x):                      # 还是例 1 的 f(x, y) = x^2 + 3y^2
    return x[0]**2 + 3*x[1]**2

def numerical_grad(f, x, eps=1e-5):
    """对每个分量加减一个极小量，用中心差分估计偏导。"""
    g = np.zeros_like(x)
    for i in range(len(x)):
        xp, xm = x.copy(), x.copy()
        xp[i] += eps
        xm[i] -= eps
        g[i] = (f(xp) - f(xm)) / (2 * eps)
    return g
```

然后在点 $(1,1)$ 处跑一下，看它和我们手算的 $(2,6)$ 对不对得上：

```python
x = np.array([1.0, 1.0])
print(numerical_grad(f, x))    # ≈ [2. 6.]
```

结果几乎完全吻合。需要提醒的是，**数值梯度主要用来"验算"**——当你亲手推导并写出解析梯度后，可以用它来检查自己有没有写错（这一步叫 gradient check）。真正训练时，我们不会用这么慢的数值法，而是用 PyTorch 的 `autograd` 自动、精确地算出解析梯度。

## 容易踩的坑

- **`eps` 不能乱取**：太大了估得不准，太小了又会被浮点误差吞掉，一般取 $10^{-4}$ 到 $10^{-6}$ 比较稳。
- **链式法则漏层**：复合函数从外往里逐层相乘，少乘一层结果就错。反向传播写错时，十有八九是这里出了问题。
- **忘了 `.copy()`**：`xp = x` 只是给同一个数组起了个别名，改 `xp` 会连 `x` 一起改。必须用 `x.copy()` 真正复制一份。
- **梯度爆炸或消失**：网络很深时，链式相乘会让梯度变得极大或极小，后面我们会用归一化、梯度裁剪等手段来缓解。

## 它在后面会怎么用到

- **梯度下降**的更新公式 $\theta\leftarrow\theta-\eta\nabla f$，是所有训练过程的引擎（见 [1.3.4 凸优化基础](1-3-4-convex-optimization.md)）。
- **反向传播**用链式法则把梯度逐层回传，是神经网络能学习的根本原因（见 [2.2.2 梯度下降与反向传播](../ch2-round1/2-2-2-backprop.md)）。

    2. 用上面的 `numerical_grad`，验证 $f(x,y)=\sin(x)+xy$ 在点 $(0,1)$ 处的梯度。

??? note "进阶练习"
    1. 用链式法则手推 $\sigma(x)=\dfrac{1}{1+e^{-x}}$ 的导数，并证明一个很漂亮的结论：$\sigma'=\sigma(1-\sigma)$。逻辑回归会直接用到它。
    2. 写一个 gradient check 函数：对比你手写的解析梯度和数值梯度，输出两者的最大误差。这是调试反向传播的必备工具。

## 小结

- 梯度指向上升最快的方向，**它的反方向就是下山方向**——这是训练的引擎。
- **链式法则**把复合函数的导数拆成逐层相乘，是反向传播的数学本体。
- 数值梯度用来验算，解析梯度（由 `autograd` 自动算）用来训练。
