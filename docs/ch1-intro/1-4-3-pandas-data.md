## 为什么要先处理数据

处理数据这件事，对人工智能来说怎么强调都不过分。计算机领域有一句很有名的话：*Garbage in, garbage out.*（喂进去的是垃圾，吐出来的也只能是垃圾。）模型的表现高度依赖数据的质量——只有当你喂给它干净、规整的数据时，才有可能训练出一个可靠的模型。

可现实里的原始数据往往很"脏"：可能有缺失的空格子，可能同一个类别写法五花八门（"NY"、"New York"、"纽约"），不同特征的数值范围也可能差出好几个量级。模型对这些非常敏感，稍有不慎就会学歪，甚至直接报错。所以在真正建模之前，我们必须先给数据做一遍"清洗"——而这，正是 Pandas 最擅长的工作。

## Pandas 的几个核心操作

拿到一份数据，我们一般会关注三件事：

1. 读取数据，看一下都有哪些项目、一共多少条；
2. 看一下数据有没有缺失；
3. 看一下有没有明显的异常值。

我们可以用 Pandas 快速把文件读进来：

```python
import pandas as pd

df = pd.read_csv("data.csv")   # 把 CSV 读成一张表（DataFrame）
```

读进来之后，第一件事是确认数据是否正常。我们可以**先用 `df.head()` 看头几行**——如果数据有严重问题，这一步往往能初步暴露出来，比如某一列整列是空的、某个数字格子里混进了文字、或者列名根本没对齐。

```python
df.head()        # 看前 5 行长什么样
df.info()        # 每列的类型和非空数量
df.describe()    # 数值列的均值、最大最小值等统计
```

`info()` 能帮你一眼看出哪几列有缺失（非空数量比总行数少），`describe()` 则能让异常值现形（比如年龄的最大值是 999，多半是错误数据）。

## 挑选和过滤：只取你要的部分

确认数据没大问题后，我们常常只想用其中的某些行、某些列。Pandas 提供了几种直观的方式：

```python
df["age"]                  # 取一整列（得到一个 Series）
df[["age", "score"]]       # 取多列
df.loc[df["age"] > 18]     # 按条件过滤行：只留下成年人
df.iloc[0:5]               # 按位置取前 5 行
```

记住一个区别：`loc` 是"按标签/条件"选，`iloc` 是"按位置编号"选。条件过滤几乎都用 `loc`。

## 处理缺失值

缺失值是最常见的问题，也是模型最不能容忍的。我们先看每列缺了多少，再决定怎么补：

```python
df.isna().sum()            # 每列缺失值的个数
```

补的策略要看情况：数值列通常用中位数或均值填，类别列用出现最多的值（众数）填；如果是关键的标签缺失，那一行干脆丢掉更稳妥。

```python
df["age"] = df["age"].fillna(df["age"].median())   # 数值列：用中位数补
df = df.dropna(subset=["label"])                   # 标签缺失：丢掉该行
```

## 把数据整理成模型能吃的样子

补完缺失，还差最后两步，数据才能交给模型。我们把整个流程串成一条龙看一遍。

**第一步，标准化数值列。** 前面讲过，特征尺度差太多会拖累训练，所以把每个数值特征都缩放到"零均值、单位方差"：

```python
num = ["age", "income"]
df[num] = (df[num] - df[num].mean()) / df[num].std()
```

**第二步，把文字类别变成数字。** 模型只认数字，不认"北京""上海"这样的文字。独热编码（one-hot）会把一个类别列拆成若干个 0/1 列：

```python
df = pd.get_dummies(df, columns=["city"])
```

**最后，分出特征和标签，转成 NumPy 数组**，就可以喂给模型了：

```python
X = df.drop(columns=["label"]).to_numpy()
y = df["label"].to_numpy()
```

三步走完，一份"脏数据"就变成了干净、全数值、模型能直接读的矩阵。

## 容易踩的坑

- **`SettingWithCopyWarning`**：像 `df[df.a > 0]["b"] = 1` 这样的"链式赋值"可能改不到原表，还会弹警告。正确写法是一步到位：`df.loc[df.a > 0, "b"] = 1`。
- **缺失值不能直接喂模型**：训练前一定先 `isna().sum()` 检查，否则后面的 sklearn 或 PyTorch 要么报错、要么算出一堆 NaN。
- **小心数据泄漏**：标准化用的均值和方差，只能用**训练集**来算，再套用到验证集和测试集；如果用全部数据来算，就等于偷看了答案（详见 [1.1 监督学习工作流](../ch2-round1/2-1-1-supervised-workflow.md)）。
- **`inplace` 的迷惑性**：带 `inplace=True` 的操作是原地修改、返回 `None`，别再把它的返回值赋给变量，否则变量会变成 `None`。

## 它在后面会怎么用到

这一节做的是"基础清洗"，是几乎所有表格类任务（Round 1 里很常见）的第一步：读数据 → 清洗 → 整理特征 → 建模。等你想造出更有信息量的特征时，会接着看 [1.10 进阶特征工程](../ch2-round1/2-1-10-feature-engineering.md)——那里讲的是"高级造特征"，和这里的"基础清洗"正好衔接。

    2. 用 `get_dummies` 把某个类别列做独热编码，对比一下编码前后表的列数变化。

## 小结

- 数据决定模型上限，记住 *Garbage in, garbage out*：先把数据洗干净，再谈训练。
- Pandas 的基本功是：读取、查看、过滤、补缺失、造特征。
- 模型怕缺失、对尺度敏感、只认数字，所以**补缺失 + 标准化 + 独热编码**是三件必做的事。
- 标准化的统计量只能从训练集算，别让测试集的信息泄漏进来。
