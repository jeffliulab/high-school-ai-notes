# 2.2 差异化备考

> **难度** ⭐⭐⭐☆☆ · **前置**：[2.1 赛制详解](5-2-1-iaio-format.md)

!!! abstract "本页目标"
    - 想清楚 IAIO 这种"团队赛"和你之前一路打过来的"个人赛"到底差在哪，为什么不能照搬个人战术
    - 学会三种实用的并行解题分工法，并能根据题目类型当场选一种用上
    - 知道队长该做哪些事、不该越权做哪些事，把沟通开销控制在帮忙而不是添乱的范围内

## 为什么个人备考的那套，到这里要改一改

一路从 Round 1 打到集训营，你练的几乎全是"一个人闷头解题"的本事——审题、写代码、调参、提交，节奏完全由自己掌控。可一旦进了 IAIO 这种以团队为单位计分的赛场，情况就变了：你身边坐着两三个同样优秀的队友，你们共用一份时间、共享一套榜单成绩，最后拿的是一个**团队总分**。

这时候最容易犯的错，恰恰是"每个人都太能干了"。三个高手如果各管各的，很可能三个人都在偷偷重写同一段数据加载代码，或者两个人不约而同地去啃那道最难的题，却没人去捡那些好拿的分。所以团队赛真正比的，往往不是谁单题更强，而是**谁的协作更不浪费**。换句话说，你要补的不是新知识，而是一套"怎么和别人一起高效解题"的打法。

## 并行解题：把题目摊开，而不是排成一队

团队赛最大的红利就是**并行**——本来一个人要串行做完的事，现在可以几个人同时推进。但并行不是"大家一起上"，而是要先把活儿切干净、分明白，否则人多反而互相绊脚。下面三种分工法，你可以按比赛当下的题型来挑。

**第一种，按题目分。** 如果这场比赛是好几道相对独立的题（比如一道 CV、一道 NLP、一道优化），那最简单：一人认领一道，各自从头做到尾。这种分法沟通成本最低，适合开局——先让每个人手里都有活、都能往榜上贡献分数，避免所有人挤在同一道题上空耗。

**第二种，按流程分。** 如果是一道大题、但能拆成清晰的几段（数据清洗 → 建模 → 调参 → 写提交），那就按流水线切：一个人先把数据管线搭好，另一个人接手训练，第三个人专门盯调参和提交格式。它的好处是每段能专精，坏处是**前一段没交货，后一段就得干等**，所以一定要让上游先产出一个"能跑通的最简版本"，下游才好接力。

**第三种，按"探路 + 深耕"分。** 面对一道吃不准的难题，可以让一个人快速试各种思路、做小实验探虚实（探路），另一个人在已被验证有效的方向上把模型和参数往深里做（深耕）。这种分法专治"全队卡在一道题上不知往哪走"的局面。

不管用哪种，记住一条铁律：**分工的前提是先有一个能提交、能拿分的基线（baseline）**。哪怕它很烂，先让榜上有个数，团队心里才不慌，后面所有的并行优化也才有了对比的基准。

## 队长该干什么，又不该抢着干什么

很多队伍误以为队长就是"队里最强的那个，所以难题都归他"。这恰恰是浪费——队长一旦埋头钻进一道题里，他的视野就锁死在那一题上，再没人从全局看整盘棋了。队长真正的价值，是当那个**始终抬着头看全局的人**。

具体来说，队长该做这么几件事：

- **盯总分和时间。** 时刻清楚现在团队总分卡在哪、离截止还剩多久、哪道题的边际收益最高，据此决定"该集中火力还是该分散捡分"。
- **做取舍决策。** 当两个队友为某个方向争执不下、或者一道难题久攻不下时，由队长拍板：是再投入半小时，还是果断止损、转去拿别处更稳的分。团队赛里，**一个及时的"放弃"决定，常常比多写一百行代码更值钱**。
- **管协作的"接口"。** 约定好代码放哪、提交文件谁来合、命名怎么统一，避免队友各写各的最后拼不到一起。

反过来，队长**不该**做的是：把自己当成主力，独占最难的题，或者事事都要过自己的手。那样他既丢了全局视野，又成了全队的瓶颈。一句话——队长管的是"方向盘和油门"，不是去当那个踩得最猛的轮子。

## 沟通是有成本的，别让它吃掉你的时间

团队协作有个反直觉的地方：沟通本身是要花时间的。每一次"等一下，你那段写完了吗""我们要不要换个思路"，都在消耗本可以用来解题的分钟数。人越多，要协调的两两关系就越多，这个开销涨得还挺快。所以高效的队伍不是沟通最多的，而是**沟通最值的**。

几个能立竿见影压低开销的习惯：

- **开局花五分钟对齐，胜过中途乱成一团。** 一坐下别急着写代码，先一起把题扫一遍，定下分工、基线策略和提交约定，然后才各自散开干活。这五分钟买的是后面两小时的不互相打扰。
- **同步要短、要定时，而不是随时打断。** 与其想到啥就喊一嗓子，不如约定"每隔一段时间碰一次进度"。被频繁打断的人是写不出好代码的。
- **共享状态，而不是反复口头问。** 把"谁在做哪题、当前最好成绩多少、提交还剩几次"写在一个所有人都看得见的地方，能省掉大量"诶你那边怎么样了"的来回。
- **冲突就快速试验，别空口争论。** 两种思路谁也说服不了谁时，最快的解法不是辩论，而是各花十分钟跑个小实验，用结果说话。

## 容易踩的坑

- **三个人都去啃最难那道题。** 题目最性感不代表分最好拿。先把容易的、确定能拿的分收进口袋，再谈攻坚。
- **迟迟不出基线。** 一上来就追求完美方案，结果到比赛中段榜上还是零分——永远先用最简单的办法拿到一个可提交结果，再迭代。
- **代码各写各的，最后拼不起来。** 没有提前约定数据格式、文件命名和提交接口，临交卷前才发现合不到一块，时间全耗在对接上。
- **队长当主力。** 队长一头扎进难题，全队就没人看全局、没人管时间和取舍了。
- **把沟通当免费的。** 频繁打断、反复口头确认，看着像在协作，其实在悄悄偷走每个人的解题时间。

## 它在后面会怎么用到

这套团队打法不是凭空来的——它的练兵场就是集训营里的 [3. 团队挑战 Team Challenge](../ch4-camp/4-3-team-challenge.md)，那里你能在低风险环境里把分工、基线和队长决策真正跑一遍。真正上场前，记得把它和 [3.1 全真模拟](5-3-1-full-mock.md) 结合：在模拟里故意按团队节奏走一遍，比赛日才不会手生。而每道题内部具体怎么快速拿分、怎么止损，要靠 [1.3 限时解题策略](../ch4-camp/4-1-3-time-strategy.md) 打底——团队协作管的是"人怎么配合"，限时策略管的是"单题怎么取舍"，两者合在一起才完整。

## 练习

??? note "基础练习"
    1. 假设 IAIO 给你们队三道独立的题（CV / NLP / 优化），队里三个人。写出你们开局五分钟内要敲定的三件事，并说明先出基线为什么比先攻难题重要。
    2. 列出队长在比赛中应该一直盯着的三个全局指标，并解释为什么队长不该独占最难的题。

??? note "进阶练习"
    1. 面对"一道大题、能拆成数据清洗 → 训练 → 调参三段"的场景，设计一套流水线分工方案，并指出其中最容易卡住的环节（提示：上游交货延迟），给出你的应对办法。
    2. 设计一个"共享状态板"的最简模板（哪些字段必须有），说明它具体能消掉哪几类口头沟通，从而省下时间。

## 小结

- 团队赛比的不是单题最强，而是**协作最不浪费**：先出基线，再并行，按题型选分工法。
- 队长是抬头看全局、管时间和取舍的人，不是抢难题的主力。
- 沟通有成本——开局对齐、定时同步、共享状态、用实验代替争论，把每一次沟通都花在刀刃上。

想把团队协作练扎实，最好的办法就是去打 [3. 团队挑战 Team Challenge](../ch4-camp/4-3-team-challenge.md)，在真实对抗里磨合，比读多少文字都管用。
